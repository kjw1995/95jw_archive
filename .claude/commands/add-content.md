## 입력

$ARGUMENTS

축약어 예시: "모던자바 6장", "JPA 12장", "네트워크 37장", "웹프로그래밍 11장", "코틀린 3장"

## 워크플로우

다음 순서를 **반드시** 따라 실행하세요:

### 1단계: 대상 디렉터리 파악

`content/education/CLAUDE.md`의 **디렉토리 등록부**를 읽고, 입력된 축약어로 대상 경로를 결정합니다.

### 2단계: 디렉터리 CLAUDE.md 읽기

대상 디렉터리의 `CLAUDE.md`를 읽어 아래를 확인합니다:
- `source`: 출처 (책 제목)
- `obsidian`: 옵시디언 원본 경로
- `naming`: 파일 네이밍 규칙
- `title_format`: 제목 형식
- `style_ref`: 문체 참고 파일

### 3단계: 옵시디언 원본 읽기

`obsidian` 경로에서 해당 챕터 파일을 찾아 읽습니다.

### 4단계: 기존 마지막 파일 확인

대상 디렉터리에서 가장 마지막 번호의 `.md` 파일을 읽어:
- 다음 번호(weight) 결정
- 문체/구조 참고

### 5단계: 새 파일 작성

원본을 기반으로 아래를 보강하여 작성합니다:
- 코드 예제 추가
- 비교 표/다이어그램 추가 (코드 블록 다이어그램 40자 이내 규칙 준수)
- callout 주의사항 (실무 팁, 흔한 실수)
- 단계별 동작 흐름 시각화

### 6단계: _index.md 업데이트

해당 디렉터리의 `_index.md`에 새 파일 링크를 추가합니다.

### 7단계: 상위 _index.md 확인

상위 디렉터리의 `_index.md`에도 반영이 필요하면 업데이트합니다.

### 8단계: 커밋 & 푸시

루트 CLAUDE.md의 Git 규칙(한글, 생성/수정/삭제 섹션)에 따라 커밋하고 `git push origin main`합니다.
