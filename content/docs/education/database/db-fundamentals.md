## 1. 데이터와 정보

데이터(Data)는 관찰이나 측정으로 수집한 가공되지 않은 사실이나 값이다. 정보(Information)는 데이터를 가공·해석해 의사결정에 활용할 수 있도록 만든 결과물이다.

```text
  데이터           가공           정보
 ┌───────┐      ─────→      ┌───────┐
 │ 사실  │                  │ 판단  │
 │ 값    │                  │ 근거  │
 └───────┘                  └───────┘
```

| 구분 | 데이터 | 정보 |
|:---|:---|:---|
| 정의 | 관찰/측정한 사실과 값 | 가공·해석된 결과물 |
| 상태 | 원시 상태 | 의사결정 활용 가능 |
| 예시 | `25`, `서울`, `2024-01-15` | "서울 1월 평균 기온 -2도" |

## 2. 정보 시스템과 데이터베이스

정보 시스템은 데이터를 수집하고 저장하며 처리해 정보를 제공하는 시스템이다. 이 시스템의 중심에서 데이터를 저장·관리하는 구성요소가 데이터베이스다.

```text
 수집 → 저장 → 처리 → 제공
         │
         ▼
    데이터베이스
```

## 3. 데이터 수명 주기

조직에서 다루는 데이터는 정적인 값이 아니라 생성부터 폐기까지 단계적으로 변화한다.

```text
 생성 → 가공 → 저장 → 분석 → 폐기
```

| 단계 | 설명 | 예시 |
|:---|:---|:---|
| 생성 | 업무 이벤트로 데이터 발생 | 주문 접수, 로그 기록 |
| 가공 | 정제, 변환, 통합 | 형식 변환, 결측치 처리 |
| 저장 | DB·파일·웨어하우스에 적재 | RDB, 데이터 레이크 |
| 분석 | 집계·모델링으로 정보 추출 | BI 리포트, 예측 모델 |
| 폐기 | 보존 기한 경과 데이터 삭제 | 개인정보 파기 |

{{< callout type="info" >}}
법·규정에 따른 보존 기한과 파기 절차는 수명 주기의 필수 단계다. 삭제되지 않은 개인정보는 보안 사고와 컴플라이언스 위반으로 이어진다.
{{< /callout >}}

## 4. 데이터베이스의 정의

데이터베이스(DB)는 여러 사용자가 함께 사용할 수 있도록 조직적으로 통합되어 저장된 운영 데이터의 집합이다.

```text
 DB = 공유 + 통합 + 저장 + 운영 데이터
```

| 특성 | 설명 | 예시 |
|:---|:---|:---|
| 공유 데이터 | 여러 사용자가 함께 사용 | 회사 전체가 고객 DB 공유 |
| 통합 데이터 | 중복 최소화, 통제된 중복만 허용 | 고객 정보 단일 관리 |
| 저장 데이터 | 컴퓨터가 접근 가능한 매체에 저장 | HDD, SSD, 클라우드 |
| 운영 데이터 | 조직 운영에 필수적인 데이터 | 주문, 재고, 회원 정보 |

### 데이터 중복의 문제

중복이 통제되지 않으면 값 사이에 불일치가 생겨 데이터 일관성이 깨진다.

```text
[중복 허용]
테이블 A: "홍길동", "010-1234-5678"
테이블 B: "홍길동", "010-9999-8888"  ← 불일치

[통합 DB]
고객: id=1, "홍길동", "010-1234-5678"
주문: customer_id=1 (참조만)
```

## 5. 데이터베이스의 4가지 특징

### 실시간 접근 (Real-time Accessibility)

사용자 요청에 즉시 응답하는 것이 원칙이다.

```sql
-- 밀리초 단위 응답
SELECT * FROM orders WHERE customer_id = 123;
```

### 계속 변화 (Continuous Evolution)

현실 세계의 변화를 삽입·수정·삭제로 반영한다.

```sql
INSERT INTO orders VALUES (1001, 123, NOW());
UPDATE products SET stock = stock - 1;
DELETE FROM cart WHERE order_completed = true;
```

### 동시 공유 (Concurrent Sharing)

여러 사용자가 같은 데이터 또는 서로 다른 데이터에 동시에 접근할 수 있다.

```text
 사용자 A ─┐
 사용자 B ─┼─→ 데이터베이스
 사용자 C ─┘
```

### 내용으로 참조 (Content Reference)

물리적 주소가 아니라 값(내용)으로 데이터를 찾는다.

```sql
SELECT * FROM employees WHERE name = '홍길동';
SELECT * FROM products WHERE price > 10000;
SELECT * FROM orders WHERE date = '2024-01-15';
```

## 6. 데이터 분류

### 형태에 따른 분류

```text
 ┌───────────────────────────┐
 │    데이터 형태 분류         │
 ├────────┬────────┬─────────┤
 │ 정형   │ 반정형  │ 비정형  │
 ├────────┼────────┼─────────┤
 │ 스키마 │ 내부   │ 구조     │
 │ 사전   │ 구조   │ 없음     │
 │ 정의   │ 포함   │          │
 ├────────┼────────┼─────────┤
 │ RDB    │ JSON   │ 이미지   │
 │ 시트   │ XML    │ 영상     │
 └────────┴────────┴─────────┘
```

#### 정형 데이터 (Structured)

```sql
CREATE TABLE customers (
    id INT PRIMARY KEY,
    name VARCHAR(100),
    email VARCHAR(255),
    created_at DATETIME
);
```

#### 반정형 데이터 (Semi-structured)

```json
{
  "id": 1,
  "name": "홍길동",
  "email": "hong@example.com",
  "orders": [
    {"id": 101, "amount": 50000},
    {"id": 102, "amount": 30000}
  ]
}
```

```xml
<customer>
  <id>1</id>
  <name>홍길동</name>
  <email>hong@example.com</email>
</customer>
```

#### 비정형 데이터 (Unstructured)

```text
- 이미지 (.jpg, .png)
- 동영상 (.mp4, .avi)
- 문서 (.txt, .pdf)
- 소셜 미디어 게시글
- 음성 (.mp3, .wav)
```

### 특성에 따른 분류

```text
 ┌─────────────────────────┐
 │      데이터 특성         │
 ├──────────────┬──────────┤
 │   범주형      │  수치형   │
 ├──────┬───────┼─────┬────┤
 │ 명목 │ 순서  │이산│연속 │
 └──────┴───────┴─────┴────┘
```

#### 범주형 데이터

| 유형 | 설명 | 예시 |
|:---|:---|:---|
| 명목형 | 서열 없는 분류 | 성별, 혈액형, 지역 |
| 순서형 | 서열 있는 분류 | 학점(A>B>C), 만족도 |

```sql
-- 명목형
SELECT * FROM users WHERE gender = 'M';

-- 순서형
SELECT * FROM reviews WHERE rating >= 4;
```

#### 수치형 데이터

| 유형 | 설명 | 예시 |
|:---|:---|:---|
| 이산형 | 셀 수 있는 값 | 주문 수, 클릭 횟수 |
| 연속형 | 측정으로 얻는 값 | 키, 몸무게, 온도 |

```sql
-- 이산형
SELECT COUNT(*) FROM orders WHERE user_id = 1;

-- 연속형
SELECT AVG(height) FROM students;
```

## 7. 스키마와 메타데이터

정형 데이터는 외부에 스키마가 사전 정의되고, 반정형 데이터는 데이터 내부에 구조 설명(메타데이터)이 함께 담긴다.

| 용어 | 설명 | 사용 위치 |
|:---|:---|:---|
| 스키마 (Schema) | 사전에 정의된 데이터 구조 | 정형 데이터 |
| 메타데이터 (Metadata) | 데이터 자체에 대한 설명 데이터 | 반정형·전 영역 |

### 메타데이터의 역할

메타데이터는 데이터에 대한 데이터다. 단순히 형식 설명에 머물지 않고 데이터 관리·검색·보안의 핵심 수단이 된다.

| 분류 | 예시 |
|:---|:---|
| 구조 메타데이터 | 테이블/컬럼 정의, JSON 키 이름 |
| 기술 메타데이터 | 데이터 타입, 크기, 인코딩 |
| 업무 메타데이터 | 데이터 의미, 담당자, 출처 |
| 운영 메타데이터 | 생성 시각, 최종 수정, 접근 권한 |

{{< callout type="info" >}}
DBMS는 스키마·제약조건·권한 같은 메타데이터를 **데이터 사전(시스템 카탈로그)** 에 별도로 저장해 질의 최적화와 보안 제어에 활용한다.
{{< /callout >}}

## 8. 정성적 vs 정량적 데이터

| 구분 | 정성적 (Qualitative) | 정량적 (Quantitative) |
|:---|:---|:---|
| 의미 | 질적 데이터 | 양적 데이터 |
| 대응 | 범주형 | 수치형 |
| 연산 | 산술 연산 불가 | 산술 연산 가능 |

## 9. 실무 예시

쇼핑몰 DB는 정형 데이터를 관계형 테이블로 저장하고, 데이터베이스의 4가지 특징을 모두 활용한다.

```sql
CREATE TABLE products (
    id INT PRIMARY KEY,
    name VARCHAR(200),       -- 명목형
    category VARCHAR(50),    -- 명목형
    price DECIMAL(10,2),     -- 연속형
    stock INT,               -- 이산형
    rating VARCHAR(10)       -- 순서형
);

-- 1. 실시간 접근
SELECT * FROM products WHERE id = 100;

-- 2. 계속 변화
UPDATE products SET stock = stock - 1 WHERE id = 100;

-- 3. 동시 공유 (트랜잭션)
BEGIN TRANSACTION;
UPDATE products SET stock = stock - 1 WHERE id = 100;
INSERT INTO orders (product_id, quantity) VALUES (100, 1);
COMMIT;

-- 4. 내용으로 참조
SELECT * FROM products
WHERE category = '전자제품'
  AND price BETWEEN 10000 AND 50000;
```

## 핵심 정리

| 개념 | 설명 |
|:---|:---|
| 데이터 | 관찰·측정으로 수집한 사실과 값 |
| 정보 | 가공·해석된 의사결정용 결과물 |
| 데이터베이스 | 공유·통합·저장·운영 데이터의 집합 |
| 4가지 특징 | 실시간 접근, 계속 변화, 동시 공유, 내용 참조 |
| 수명 주기 | 생성 → 가공 → 저장 → 분석 → 폐기 |
| 정형/반정형/비정형 | 구조 유무와 위치에 따른 분류 |
| 범주형/수치형 | 특성에 따른 분류 |
| 스키마 | 사전에 정의된 데이터 구조 |
| 메타데이터 | 데이터를 설명하는 데이터 |

{{< callout type="info" >}}
**용어 정리**
- **데이터 (Data)**: 관찰·측정으로 수집한 사실과 값
- **정보 (Information)**: 가공·해석되어 판단에 쓰이는 결과물
- **정보 시스템**: 데이터 수집·저장·처리·제공 전반의 시스템
- **데이터베이스**: 공유·통합·저장·운영 데이터의 집합
- **실시간 접근 / 계속 변화 / 동시 공유 / 내용 참조**: DB의 4가지 특징
- **정형 / 반정형 / 비정형**: 구조 위치와 유무에 따른 데이터 분류
- **범주형 / 수치형**: 특성에 따른 분류, 각각 명목·순서 / 이산·연속으로 세분
- **정성적 / 정량적**: 질적 / 양적 데이터
- **스키마 (Schema)**: 사전에 정의된 데이터 구조
- **메타데이터 (Metadata)**: 데이터에 대한 데이터, 구조·의미·운영 정보
- **데이터 수명 주기**: 생성·가공·저장·분석·폐기의 단계적 흐름
- **데이터 중복성**: 동일 데이터가 여러 곳에 존재하는 상태
{{< /callout >}}
