## 1. 텍스트 파일과 바이너리 파일

리눅스에서 파일은 내용 형식에 따라 두 종류로 나뉜다.

| 종류 | 예시 | 특징 |
|:-----|:-----|:-----|
| 텍스트 파일 | `.txt`, `.conf`, `.log`, 스크립트 | 사람이 읽을 수 있는 문자열 |
| 바이너리 파일 | 이미지, 음성, 실행 파일 | 전용 프로그램 필요 |

**텍스트 파일의 장점**

- 전용 애플리케이션 없이 내용을 확인할 수 있다
- `cat`, `less`, `grep`, `sed` 등 리눅스 명령어를 그대로 활용할 수 있다
- 버전 관리(Git)에서 변경 내역을 한눈에 추적할 수 있다

```bash
# 파일 형식 확인
$ file /etc/passwd
/etc/passwd: ASCII text

$ file /bin/ls
/bin/ls: ELF 64-bit LSB executable
```

> 리눅스 시스템 설정 파일은 대부분 텍스트 파일이다. 에디터 하나를 능숙하게 다루는 것이 곧 시스템 관리 능력으로 이어진다.

## 2. Vim: 기본 에디터

### 에디터의 종류

| 에디터 | 성격 |
|:-----|:-----|
| `vi` / `vim` | 거의 모든 리눅스 시스템에 기본 탑재 |
| `nano` | 단축키가 화면에 표시되어 초보자 친화적 |
| `emacs` | 확장성이 매우 높은 고급 에디터 |

> 서버에 SSH로 접속했을 때 `nano`가 없을 수는 있어도 `vi`는 거의 항상 존재한다. 그래서 한 개만 배워야 한다면 Vim이다.

### 설치 확인

```bash
$ vim --version
VIM - Vi IMproved 8.2 ...
```

### 우분투에서 Vim 설치

```bash
$ sudo apt-get install vim
```

### 기동과 종료

```bash
# 빈 화면으로 Vim 시작
$ vim

# 파일을 지정해서 열기 (없으면 새로 만든다)
$ vim memo.txt

# vi라고 입력해도 Vim이 실행된다 (호환성)
$ vi memo.txt
```

**가장 먼저 외워야 할 명령어**

| 명령어 | 내용 |
|:-----|:-----|
| `:q` | Vim 종료 |
| `:w` | 저장 |
| `:wq` | 저장 후 종료 |
| `:q!` | 저장하지 않고 강제 종료 |
| `ZZ` | 저장 후 종료 (보통 모드) |

{{< callout type="warning" >}}
명령어가 먹히지 않으면 십중팔구 **입력 모드**에 들어가 있는 것이다. 먼저 `Esc`를 두세 번 눌러 보통 모드로 돌아간 뒤 명령을 다시 입력하자.
{{< /callout >}}

## 3. Vim의 모드

Vim의 가장 큰 특징은 **모드(mode)** 의 존재다. 같은 키가 모드에 따라 전혀 다른 동작을 한다.

### 주요 모드

| 모드 | 진입 키 | 역할 |
|:-----|:-----|:-----|
| 보통 모드 (Normal) | `Esc` | 커서 이동, 명령 실행 |
| 입력 모드 (Insert) | `i`, `a`, `o` | 텍스트 타이핑 |
| 명령행 모드 (Command-line) | `:` | 저장, 종료, 치환 |
| 비주얼 모드 (Visual) | `v`, `V`, `Ctrl+v` | 범위 선택 |

### 모드 전환 흐름

```
    ┌───────────────┐
    │  보통 모드     │◀─────┐
    │  (Normal)     │      │
    └───┬─────┬─────┘      │
        │     │            │
     i,a,o    │ :          │ Esc
        ▼     ▼            │
   ┌───────┐ ┌──────────┐  │
   │ 입력  │ │ 명령행   │──┘
   │ Insert│ │ Command  │
   └───┬───┘ └──────────┘
       │ Esc
       └─────────────────▶ 보통 모드
```

{{< callout type="info" >}}
**Vim을 쓰는 핵심 원리**: 입력 모드에 오래 머물지 않는다. 타이핑이 끝나는 즉시 `Esc`로 보통 모드로 돌아가야 커서 이동·편집 명령을 자유롭게 쓸 수 있다.
{{< /callout >}}

## 4. 파일 열기와 저장하기

### 파일 열기

```bash
# 기존 파일 열기
$ vim memo.txt

# 존재하지 않는 파일 → 새로 만들어진다
$ vim newfile.txt
```

Vim은 파일의 내용이 끝난 이후 각 행에 `~`(틸드)를 표시해 **빈 행**임을 나타낸다.

### 파일 저장하기

```vim
:w                  " 현재 파일에 저장
:w newname.txt      " 다른 이름으로 저장
:saveas newname.txt " 다른 이름으로 저장 후 그 파일을 계속 편집
```

### 종료 명령어 전체

| 명령어 | 내용 |
|:-----|:-----|
| `:q` | 저장 후 종료 (변경 없을 때) |
| `:w` | 저장만 |
| `:wq` | 저장 후 종료 |
| `:x` | 변경사항이 있을 때만 저장 후 종료 |
| `:q!` | 변경사항 버리고 종료 |
| `ZZ` | 저장 후 종료 |
| `ZQ` | 저장하지 않고 종료 |

> `:wq`를 습관처럼 쳐도 되지만, `:x`는 변경이 없으면 저장하지 않아 파일의 수정 시간을 보존한다. `make` 같은 빌드 도구와 궁합이 좋다.

## 5. 기본 편집

### 커서 이동 (보통 모드)

```
       ↑
       k
   ← h   l →
       j
       ↓
```

| 키 | 이동 |
|:-----|:-----|
| `h` | 왼쪽 |
| `j` | 아래 |
| `k` | 위 |
| `l` | 오른쪽 |

> 방향키도 동작한다. 그러나 `h j k l`은 홈 포지션에서 손을 떼지 않아도 되므로 속도가 훨씬 빠르다.

### 문자 삭제와 입력

| 명령어 | 내용 |
|:-----|:-----|
| `x` | 커서 위치 문자 삭제 |
| `X` | 커서 왼쪽 문자 삭제 |
| `i` | 커서 왼쪽에 문자 입력 |
| `a` | 커서 오른쪽에 문자 입력 |
| `I` | 행의 맨 앞에서 입력 |
| `A` | 행의 맨 끝에서 입력 |
| `o` | 아래에 새 행을 만들고 입력 |
| `O` | 위에 새 행을 만들고 입력 |
| `r` | 커서 위치 한 글자만 교체 |
| `R` | 덮어쓰기 모드 |

### 실전 흐름 예시

```
파일:  Hello wor|ld    ( | 은 커서 위치 )

a  +  "."  + Esc
─────────────────▶
       Hello wor.ld|

o  + "new line" + Esc
─────────────────▶
       Hello wor.ld
       new line|
```

## 6. 효율적인 커서 이동

### 단어 단위 이동

| 명령어 | 내용 |
|:-----|:-----|
| `w` | 다음 단어의 첫 글자 |
| `b` | 이전 단어의 첫 글자 |
| `e` | 현재 단어의 마지막 글자 |
| `W` `B` `E` | 공백만 단어 구분자로 사용 |

{{< callout type="info" >}}
소문자(`w`, `b`, `e`)는 쉼표·괄호 같은 기호도 단어 구분자로 본다. 대문자(`W`, `B`, `E`)는 오직 공백만 구분자로 본다. 코드처럼 기호가 많은 텍스트에서는 대문자가 편리한 경우가 많다.
{{< /callout >}}

### 행 내 이동

| 명령어 | 내용 |
|:-----|:-----|
| `0` | 행의 맨 앞 (열 0) |
| `^` | 행의 첫 non-blank 문자 |
| `$` | 행의 끝 |
| `f<문자>` | 현재 행에서 오른쪽으로 `<문자>` 찾기 |
| `F<문자>` | 현재 행에서 왼쪽으로 `<문자>` 찾기 |
| `;` / `,` | 직전 `f`, `F` 반복 / 반대 방향 |

### 파일 단위 이동

| 명령어 | 내용 |
|:-----|:-----|
| `gg` | 파일 첫 행 |
| `G` | 파일 마지막 행 |
| `<N>G` | `<N>` 행으로 이동 |
| `:<N>` | `<N>` 행으로 이동 (명령행) |
| `Ctrl+f` | 한 화면 아래 |
| `Ctrl+b` | 한 화면 위 |
| `Ctrl+d` | 반 화면 아래 |
| `Ctrl+u` | 반 화면 위 |
| `H` / `M` / `L` | 화면 상/중/하단으로 이동 |

```bash
# 100번째 행으로 바로 이동
100G

# 또는
:100
```

## 7. 자르기, 복사, 붙여넣기

Vim은 용어가 일반 에디터와 다르다.

| 일반 에디터 | Vim 용어 | 키 |
|:-----|:-----|:-----|
| 자르기 | delete | `d` |
| 복사 | yank | `y` |
| 붙여넣기 | put | `p`, `P` |

### 삭제 (`d`)

`d`는 **범위**를 받는다. `d` 뒤에 커서 이동 명령을 붙이면 그 범위가 삭제된다.

| 명령어 | 내용 |
|:-----|:-----|
| `dd` | 현재 행 삭제 |
| `dw` | 단어 하나 삭제 |
| `d$` | 커서부터 행의 끝까지 삭제 |
| `d0` | 커서부터 행의 시작까지 삭제 |
| `dgg` | 커서부터 파일 시작까지 삭제 |
| `dG` | 커서부터 파일 끝까지 삭제 |
| `3dd` | 현재 행부터 3행 삭제 |
| `d3w` | 단어 3개 삭제 |

### 복사 (`y`)

| 명령어 | 내용 |
|:-----|:-----|
| `yy` | 현재 행 복사 |
| `yw` | 단어 하나 복사 |
| `y$` | 커서부터 행의 끝까지 복사 |
| `5yy` | 현재 행부터 5행 복사 |

### 붙여넣기 (`p`, `P`)

| 명령어 | 내용 |
|:-----|:-----|
| `p` | 커서 뒤(행 복사면 아래)에 붙여넣기 |
| `P` | 커서 앞(행 복사면 위)에 붙여넣기 |

### 반복자 (count)

Vim 명령어 대부분은 앞에 숫자를 붙여 반복시킬 수 있다.

```
3dd  → dd 를 3번 (3행 삭제)
10x  → x 를 10번 (10글자 삭제)
5j   → j 를 5번 (5행 아래로)
2yy  → 현재 포함 2행 복사
```

{{< callout type="info" >}}
**Vim의 문법**: `<횟수><연산자><범위>`. 예를 들어 `d3w`는 "단어 3개 지워라", `y2j`는 "아래 두 행까지 복사해라"로 읽으면 된다. 모든 명령을 외우는 것이 아니라 **문법**을 이해하는 것이 핵심이다.
{{< /callout >}}

## 8. 비주얼 모드 (범위 선택)

마우스 없이 범위를 선택하는 기능이다.

| 키 | 선택 단위 |
|:-----|:-----|
| `v` | 문자 단위 |
| `V` | 행 단위 |
| `Ctrl+v` | 사각형 블록 |

선택 후 `d`(삭제), `y`(복사), `>`(오른쪽 들여쓰기), `<`(왼쪽 들여쓰기), `u`/`U`(소문자/대문자 변환) 등을 적용할 수 있다.

```
Shift+V 로 행 선택 → d
┌──────────────────┐
│ line 1           │
│ line 2  ← 선택   │─┐
│ line 3  ← 선택   │ │ d로 두 행 삭제
│ line 4           │ │
└──────────────────┘ ▼
┌──────────────────┐
│ line 1           │
│ line 4           │
└──────────────────┘
```

## 9. 기타 유용한 조작

### 행 결합

| 명령어 | 내용 |
|:-----|:-----|
| `J` | 현재 행과 다음 행 결합 (공백 1개로 연결) |
| `gJ` | 공백 없이 결합 |

### 취소와 재실행

| 명령어 | 내용 |
|:-----|:-----|
| `u` | 직전 동작 취소 (undo) |
| `Ctrl+r` | 취소 되돌리기 (redo) |
| `U` | 현재 행의 모든 변경 취소 |
| `.` | 직전 편집 명령 반복 |

> `.`(점)은 Vim에서 가장 강력한 명령 중 하나다. "단어 교체 후 다음으로 이동 → `.`"를 반복하면 치환을 거의 수동 매크로처럼 쓸 수 있다.

### 들여쓰기

| 명령어 | 내용 |
|:-----|:-----|
| `>>` | 현재 행 오른쪽으로 들여쓰기 |
| `<<` | 현재 행 왼쪽으로 내어쓰기 |
| `5>>` | 5행 들여쓰기 |
| `==` | 현재 행 자동 들여쓰기 |
| `gg=G` | 파일 전체 자동 들여쓰기 |

## 10. 검색과 치환

### 검색

| 명령어 | 내용 |
|:-----|:-----|
| `/<문자열>` | 아래 방향 검색 |
| `?<문자열>` | 위 방향 검색 |
| `n` | 다음 결과 |
| `N` | 이전 결과 |
| `*` | 커서 아래 단어를 아래 방향 검색 |
| `#` | 커서 아래 단어를 위 방향 검색 |

```vim
/error              " error 검색
/error\c            " 대소문자 무시
/^TODO              " 행 시작이 TODO인 곳
/\<word\>           " word 정확히 일치
```

### 치환 (substitute)

```vim
:s/<검색>/<치환>/<옵션>
```

| 범위 지정 | 의미 |
|:-----|:-----|
| `:s` | 현재 행만 (첫 번째만) |
| `:s/.../.../g` | 현재 행 전체 |
| `:%s/.../.../g` | 파일 전체 |
| `:5,10s/.../.../g` | 5~10행 |
| `:'<,'>s/.../.../g` | 비주얼 선택 범위 |

| 옵션 | 의미 |
|:-----|:-----|
| `g` | 행 내 모든 일치 |
| `c` | 치환 전 확인(confirm) |
| `i` | 대소문자 무시 |

```vim
" 파일 전체에서 foo → bar 로 치환
:%s/foo/bar/g

" 파일 전체에서 확인하면서 치환
:%s/foo/bar/gc

" 대소문자 무시하고 치환
:%s/error/ERROR/gi
```

## 11. 여러 파일 다루기

### 동시에 여러 파일 열기

```bash
$ vim file1.txt file2.txt file3.txt
```

| 명령어 | 내용 |
|:-----|:-----|
| `:next` / `:n` | 다음 파일 |
| `:prev` / `:N` | 이전 파일 |
| `:args` | 열린 파일 목록 |

### 화면 분할 (split)

| 명령어 | 내용 |
|:-----|:-----|
| `:sp <파일>` | 수평 분할 |
| `:vsp <파일>` | 수직 분할 |
| `Ctrl+w w` | 다음 창으로 이동 |
| `Ctrl+w h/j/k/l` | 방향별 창 이동 |
| `Ctrl+w q` | 현재 창 닫기 |

```
┌──────────────────────┐
│ config.yaml          │
│                      │
├──────────────────────┤   ← :sp로 수평 분할
│ app.log              │
│                      │
└──────────────────────┘
```

## 12. .vimrc — Vim 설정 파일

홈 디렉터리의 `~/.vimrc`에 설정을 저장하면 Vim 실행 시 자동으로 읽힌다.

```vim
" 기본 UX
set number              " 행 번호 표시
set relativenumber      " 상대 행 번호
set cursorline          " 커서 있는 행 강조
set showmatch           " 괄호 매칭 표시
set ruler               " 커서 위치 표시

" 들여쓰기
set tabstop=4           " 탭 폭 4
set shiftwidth=4        " 자동 들여쓰기 4
set expandtab           " 탭을 공백으로
set autoindent          " 자동 들여쓰기

" 검색
set hlsearch            " 검색 결과 하이라이트
set incsearch           " 타이핑 중 즉시 검색
set ignorecase          " 대소문자 무시
set smartcase           " 대문자가 섞이면 대소문자 구분

" 문법 강조
syntax on
```

{{< callout type="info" >}}

## 요약

### 가장 먼저 외워야 할 11개

```
i / Esc       입력 모드 / 빠져나오기
h j k l       커서 이동
:w / :q / :wq 저장 / 종료 / 저장 후 종료
dd / yy / p   행 삭제 / 복사 / 붙여넣기
u / Ctrl+r    되돌리기 / 다시 실행
```

### Vim의 문법

```
<횟수><연산자><범위>

 3     d        w     → 단어 3개 삭제
 5     y        y     → 5행 복사
 2     >        >     → 2행 들여쓰기
```

### 검색·치환 요약

```bash
/pattern           # 검색
n / N              # 다음 / 이전
:%s/a/b/g          # 전체 치환
:%s/a/b/gc         # 확인 치환
```

### 모드 전환 핵심

```
보통 모드  ─ i,a,o ─▶  입력 모드
보통 모드  ◀─ Esc ──  입력 모드
보통 모드  ─── : ──▶  명령행 모드
```
