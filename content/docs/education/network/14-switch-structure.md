## 1. 스위치란?

### 정의와 특징

스위치는 데이터 링크 계층(2계층)에서 동작하는 장비로, MAC 주소를 기반으로 프레임을 목적지 포트로만 전달한다. 내부에 MAC 주소 테이블을 유지하며, 각 포트가 독립된 콜리전 도메인을 갖는다.

| 항목 | 설명 |
|:---|:---|
| 동작 계층 | 2계층 (데이터 링크) |
| 식별자 | MAC 주소 |
| 테이블 | MAC 주소 테이블 보유 |
| 통신 방식 | 전이중 (Full Duplex) |
| 콜리전 도메인 | 포트별 분리 |

---

## 2. 허브와의 차이

{{< callout type="info" >}}
스위치와 허브는 외형이 비슷해 보여도 동작 계층과 전송 방식이 전혀 다르다. 허브는 받은 신호를 모든 포트로 플러딩하지만, 스위치는 MAC 주소 테이블을 참조해 목적지 포트로만 프레임을 보낸다.
{{< /callout >}}

| 항목 | 허브 | 스위치 |
|:---|:---|:---|
| 동작 계층 | 1계층 | 2계층 |
| 주소 인식 | 불가 | MAC 기반 |
| 전송 방식 | 플러딩 | 포워딩 |
| 콜리전 도메인 | 하나 | 포트별 |
| 통신 | 반이중 | 전이중 |
| 대역폭 | 공유 | 포트별 독립 |
| 보안 | 취약 | 목적지만 수신 |

## 3. MAC 주소 테이블

### 구조

스위치 내부에 유지되는 표로, 각 MAC 주소가 어느 포트에 연결되어 있는지를 기록한다. 브리지 테이블 또는 CAM 테이블로도 불린다.

| VLAN | MAC 주소 | 포트 | 타입 |
|:---|:---|:---|:---|
| 1 | 00:23:AE:D9:7A:01 | Fa0/1 | DYNAMIC |
| 1 | 00:23:AE:D9:7A:02 | Fa0/2 | DYNAMIC |
| 1 | 00:11:22:33:44:55 | Fa0/4 | STATIC |

동적으로 학습된 항목은 기본 300초 뒤 만료(에이징)되며, 관리자가 수동 등록한 정적 항목은 유지된다.

## 4. 학습(Learning) 과정

### 동작 순서

스위치는 프레임을 수신할 때마다 **출발지 MAC**을 수신 포트와 함께 테이블에 등록한다.

```
① 프레임 수신
② 출발지 MAC을 수신 포트와 매핑하여 등록
③ 목적지 MAC을 테이블에서 조회
④ 있으면 포워딩 / 없으면 플러딩
```

### 예시

PC-A(MAC AA:01, 포트 1)에서 PC-C(MAC CC:03)로 첫 프레임 전송 시:

1. 스위치는 `AA:01 → 포트 1` 학습
2. 테이블에 `CC:03` 없음 → 포트 1을 제외한 모든 포트로 플러딩
3. PC-C가 응답하면 `CC:03 → 포트 3` 학습
4. 이후부터는 PC-C 방향 프레임이 포트 3으로만 포워딩

## 5. 스위치의 세 가지 동작

| 동작 | 발생 조건 | 결과 |
|:---|:---|:---|
| 플러딩 | 목적지 MAC 미등록 / 브로드캐스트 | 수신 포트 제외 전체 전송 |
| 포워딩 | 목적지 MAC이 테이블에 있음 | 해당 포트로만 전송 |
| 필터링 | 출발·목적지가 같은 세그먼트 | 프레임 전송하지 않음 |

```
[PC-A] ══→ [SWITCH]
             │→ PC-B  (플러딩일 때만)
             │→ PC-C  (목적지)
             └→ PC-D  (플러딩일 때만)
```

## 6. 스위칭 방식

### 세 가지 프레임 처리 방식

| 방식 | 확인 범위 | 오류 검사 | 지연 | 안정성 |
|:---|:---|:---|:---|:---|
| Store-and-Forward | 전체 프레임 | FCS 검증 | 길다 | 높음 |
| Cut-Through | 목적지 MAC(6 B) | 없음 | 짧다 | 낮음 |
| Fragment-Free | 64 B까지 | 부분 | 중간 | 중간 |

대부분의 현대 스위치는 **Store-and-Forward** 방식을 기본으로 사용한다.

## 7. 콜리전/브로드캐스트 도메인

| 장비 | 콜리전 도메인 | 브로드캐스트 도메인 |
|:---|:---|:---|
| 허브 | 하나 (전체) | 하나 |
| 스위치 | 포트별 분리 | 하나 |
| 라우터 | 포트별 분리 | 포트별 분리 |

스위치는 콜리전 도메인은 나누지만 브로드캐스트는 차단하지 못한다. 브로드캐스트 영역을 나누려면 VLAN 또는 라우터가 필요하다.

---

## 8. 전이중 통신과 대역폭

스위치는 각 포트에 독립된 송·수신 버퍼를 두기 때문에 PC와 스위치 사이는 전이중 통신이 가능하다. 100 Mbps 포트라면 송신 100 Mbps, 수신 100 Mbps로 실효 200 Mbps의 처리가 가능하다.

```
허브(공유):    100M ÷ N 대 = 대역폭 감소
스위치(독립):  각 포트 100M × 2(전이중)
```

## 9. STP (Spanning Tree Protocol)

스위치를 이중화·삼중화로 연결하면 물리적 루프가 생겨 브로드캐스트가 무한 순환(브로드캐스트 스톰)할 위험이 있다. STP는 백업 경로를 논리적으로 차단해 루프를 방지하며, 장애 시 자동으로 경로를 복구한다.

| 상태 | 설명 |
|:---|:---|
| Blocking | 데이터 전송 차단 (BPDU만 수신) |
| Listening | BPDU 교환만 수행 |
| Learning | MAC 학습 시작 |
| Forwarding | 정상 포워딩 |

## 10. 스위치 종류

| 종류 | 특징 |
|:---|:---|
| L2 스위치 | MAC 기반, 일반적인 스위치 |
| L3 스위치 | IP 라우팅 기능 포함 |
| L4 스위치 | TCP/UDP 포트 기반, 로드밸런싱 |
| L7 스위치 | 애플리케이션 레벨, ADC |

## 핵심 정리

| 개념 | 설명 |
|:---|:---|
| 스위치 | MAC 기반 2계층 장비 |
| MAC 테이블 | 포트와 MAC을 매핑하는 표 |
| 학습 | 출발지 MAC을 테이블에 등록 |
| 플러딩/포워딩/필터링 | 스위치의 3가지 동작 |
| 콜리전 분리 | 포트마다 독립된 콜리전 도메인 |
| STP | 루프 방지 프로토콜 |

{{< callout type="info" >}}
**용어 정리**
- **스위치 (Switch)**: MAC 주소로 프레임을 전달하는 2계층 장비
- **MAC 주소 테이블**: 포트-MAC 매핑 DB (브리지 테이블, CAM)
- **학습 (Learning)**: 출발지 MAC과 포트를 자동 등록
- **플러딩 / 포워딩 / 필터링**: 스위치의 세 가지 전송 동작
- **STP**: Spanning Tree Protocol, 스위치 루프 방지
{{< /callout >}}
