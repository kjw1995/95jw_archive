## 1. 캡슐화 개요

웹 브라우저가 서버로 요청을 보낼 때, 응용 계층의 데이터는 하위 계층을 거치며 헤더가 하나씩 추가된다. 마지막에는 랜카드(NIC)가 이더넷 프레임을 전기 신호로 변환하여 매체에 내보낸다.

```
[응용]    데이터 (HTTP)
   │ + TCP 헤더
   ▼
[전송]    세그먼트
   │ + IP 헤더
   ▼
[네트워크] 패킷
   │ + 이더넷 헤더/FCS
   ▼
[데이터링크] 프레임
   │ NIC
   ▼
[물리]    비트/전기 신호
```

### 계층별 PDU

| 계층 | PDU | 추가되는 정보 |
|:---|:---|:---|
| 응용 | 데이터 | HTTP 메시지 |
| 전송 | 세그먼트 | TCP/UDP 헤더 |
| 네트워크 | 패킷 | IP 헤더 |
| 데이터링크 | 프레임 | 이더넷 헤더 + FCS |
| 물리 | 비트 | 전기 신호 |

---

## 2. 전송 계층: TCP 세그먼트

응용 데이터에 TCP 헤더가 붙어 세그먼트가 된다. TCP 헤더는 출발지/목적지 포트, 시퀀스 번호, ACK 번호, 플래그, 체크섬 등을 담는다.

```
┌───────────────────────────┐
│ TCP 헤더 (20~60 바이트)   │
│  - 출발지 포트 (2B)       │
│  - 목적지 포트 (2B)       │
│  - Seq / Ack (각 4B)      │
│  - 플래그, Window, 체크섬 │
├───────────────────────────┤
│ 데이터 (HTTP)             │
└───────────────────────────┘
```

- 출발지 포트: 동적 포트(1024~65535)에서 무작위 선택
- 목적지 포트: 서비스 기본 포트 (HTTP 80, HTTPS 443 등)

---

## 3. 네트워크 계층: IP 패킷

TCP 세그먼트 앞에 IP 헤더가 붙어 IP 패킷이 된다.

```
┌───────────────────────────┐
│ IPv4 헤더 (20 바이트)     │
│  - 버전, IHL, TOS         │
│  - 전체 길이              │
│  - 식별자/플래그/오프셋   │
│  - TTL, 프로토콜, 체크섬  │
│  - 출발지 IP (4B)         │
│  - 목적지 IP (4B)         │
├───────────────────────────┤
│ TCP 세그먼트              │
└───────────────────────────┘
```

주요 필드:

| 필드 | 의미 |
|:---|:---|
| 버전 | IPv4=4, IPv6=6 |
| TTL | 라우터 통과 시 1 감소, 0이면 폐기 |
| 프로토콜 | 6=TCP, 17=UDP, 1=ICMP |
| 출발지/목적지 IP | 종단 주소 (NAT가 없으면 유지) |

## 4. 데이터링크 계층: 이더넷 프레임

IP 패킷 앞뒤로 이더넷 헤더와 FCS(트레일러)가 붙어 프레임이 완성된다.

```
┌────────┬────────┬────────┐
│ 이더넷 │  IP +  │  FCS   │
│ 헤더   │  TCP + │ (4B)   │
│ (14B)  │  HTTP  │ CRC-32 │
└────────┴────────┴────────┘
```

### 이더넷 헤더 구성

| 필드 | 크기 | 의미 |
|:---|:---|:---|
| 목적지 MAC | 6B | 다음 홉의 MAC (같은 LAN이면 상대, 다른 LAN이면 게이트웨이) |
| 출발지 MAC | 6B | 송신 NIC의 MAC |
| 이더넷 타입 | 2B | 0x0800=IPv4, 0x0806=ARP, 0x86DD=IPv6 |

### FCS와 오류 검출

FCS(Frame Check Sequence)는 CRC-32 체크섬으로, 전송 중 비트 오류를 검출한다. 수신 측에서 재계산한 값이 일치하지 않으면 프레임을 **폐기**한다(TCP가 재전송을 담당하므로 데이터링크 계층은 복구하지 않는다).

{{< callout type="info" >}}
CRC는 오류를 복구하는 것이 아니라 "손상되었다"는 사실만 알려준다. 이더넷은 손상된 프레임을 단순히 버리고, 상위 계층(TCP)이 타임아웃과 재전송으로 복구한다. UDP를 쓰는 경우에는 상위 애플리케이션이 직접 책임져야 한다.
{{< /callout >}}

### ARP: IP → MAC

IP 주소는 알지만 같은 LAN에서 상대 MAC을 모를 때는 ARP(Address Resolution Protocol)로 질의한다.

```
1) 같은 LAN:
   "192.168.0.20의 MAC은?" (브로드캐스트)
   "00:1A:2B:...:5E" (유니캐스트 응답)

2) 다른 LAN:
   목적지 MAC = 게이트웨이의 MAC
   (게이트웨이가 그다음 홉으로 전달)
```

## 5. 물리 계층: 전기 신호

NIC(Network Interface Card, 랜카드)는 프레임 앞에 프리앰블을 붙이고 디지털 비트를 전기 신호로 변환한다. 인코딩 방식은 표준에 따라 다르다.

| 표준 | 속도 | 인코딩 |
|:---|:---|:---|
| 10BASE-T | 10 Mbps | Manchester |
| 100BASE-TX | 100 Mbps | 4B/5B + MLT-3 |
| 1000BASE-T | 1 Gbps | PAM-5 |

### NIC의 역할

송신 시:
1. 상위 계층에서 프레임 수신
2. 프리앰블 추가
3. 비트 → 전기 신호 변환
4. 매체 접근 제어(CSMA/CD 또는 전이중)
5. 매체로 전송

수신 시:
1. 전기 신호 수신, 비트로 변환
2. 프리앰블 제거
3. FCS 검사
4. 목적지 MAC이 자신이면 상위 계층으로 전달

## 6. MTU와 단편화

### MTU

MTU(Maximum Transmission Unit)는 단일 프레임으로 전송 가능한 최대 패킷 크기다. 이더넷은 기본 1500 바이트.

```
이더넷 프레임 (최대):
┌────────┬────────┬────┐
│ 헤더   │ 1500B  │FCS │
│ 14B    │ 페이로드│4B │
└────────┴────────┴────┘
총 1518B (VLAN 없음 기준)
```

### IP 단편화

페이로드가 MTU를 초과하면 IP 계층이 패킷을 쪼갠다. 같은 원본의 조각은 Identification 값이 동일하며, Fragment Offset으로 재조립한다. DF(Don't Fragment) 플래그가 설정되어 있으면 단편화 대신 폐기하고 ICMP "Fragmentation Needed"를 돌려준다. 이를 이용해 Path MTU Discovery가 경로상의 최소 MTU를 찾는다.

```bash
# PMTU 확인
ip route get 93.184.216.34
ping -M do -s 1472 example.com
#  1472 = 1500 - 20(IP) - 8(ICMP)
```

## 7. 역캡슐화 (수신)

수신 측은 반대 순서로 헤더를 제거한다.

```
[물리]    전기 신호 → 비트
    │
    ▼
[데이터링크]
  - FCS 검증 (CRC-32)
  - 목적지 MAC 확인
  - 이더넷 헤더/트레일러 제거
    │
    ▼
[네트워크]
  - 목적지 IP 확인
  - TTL, 체크섬 확인
  - IP 헤더 제거
    │
    ▼
[전송]
  - 목적지 포트 확인
  - TCP 체크섬, Seq/Ack 처리
  - TCP 헤더 제거
    │
    ▼
[응용]    HTTP 메시지 처리
```

## 8. 실무 도구

### 인터페이스·MTU 확인

```bash
# Linux
ip link show
ip -s link show eth0

# Windows
ipconfig /all
netsh interface ipv4 show interfaces
```

### ARP 테이블

```bash
arp -a
ip neigh show
```

### 패킷 캡처

```bash
sudo tcpdump -i eth0 port 80 -vv
sudo tcpdump -i eth0 host 93.184.216.34 -A
sudo tcpdump -i eth0 -w capture.pcap
```

Wireshark는 GUI로 캡처된 패킷을 계층별로 풀어 보여준다. 필터 예: `http`, `tcp.port == 80`, `ip.addr == 93.184.216.34`.

## 핵심 정리

| 항목 | 내용 |
|:---|:---|
| 캡슐화 순서 | HTTP → TCP → IP → Ethernet → 비트 |
| NIC 역할 | 프레임 ↔ 전기 신호 변환, MAC 주소 보유 |
| 오류 검출 | 이더넷 FCS(CRC-32), 손상 시 폐기 |
| MTU | 이더넷 1500 바이트 기본, 초과 시 IP 단편화 |

{{< callout type="info" >}}
**용어 정리**
- **PDU**: 계층별 데이터 단위 (세그먼트/패킷/프레임/비트)
- **NIC**: 네트워크 인터페이스 카드, MAC 주소를 가짐
- **FCS**: 프레임 체크 시퀀스, CRC-32 기반 오류 검출
- **ARP**: IP 주소로 MAC 주소를 알아내는 프로토콜
- **MTU**: 단일 프레임으로 보낼 수 있는 최대 페이로드 크기
{{< /callout >}}
