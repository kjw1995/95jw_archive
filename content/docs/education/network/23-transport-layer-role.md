## 1. 전송 계층이란?

전송 계층(Transport Layer)은 OSI 4계층에 해당하며, 종단 간(End-to-End) 통신을 담당한다. 네트워크 계층이 "어느 호스트로 보낼지"를 정한다면 전송 계층은 "그 호스트의 어떤 애플리케이션으로, 얼마나 믿을 만하게 보낼지"를 결정한다.

주요 역할은 다음과 같다.

- 종단 간 데이터 전달
- 포트 번호로 애플리케이션 식별 (멀티플렉싱)
- 오류 검출·재전송 (TCP)
- 흐름 제어, 혼잡 제어 (TCP)

대표 프로토콜은 TCP와 UDP 두 가지다.

## 2. OSI 7계층에서의 위치

```text
 7  Application   │ HTTP, DNS, SMTP
 6  Presentation  │ 암호화, 인코딩
 5  Session       │ 세션 관리
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
 4  Transport     │ TCP, UDP, 포트
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
 3  Network       │ IP, 라우팅
 2  Data Link     │ MAC, 스위치
 1  Physical      │ 케이블, 허브
```

송신 측은 데이터에 각 계층 헤더를 추가(캡슐화)하고, 수신 측은 반대 순으로 헤더를 제거(디캡슐화)한다. 전송 계층 단위는 세그먼트(TCP) 혹은 데이터그램(UDP)이라고 부른다.

## 3. 멀티플렉싱과 포트 번호

전송 계층의 핵심 기능 중 하나는 **멀티플렉싱**이다. 하나의 호스트에서 여러 애플리케이션이 동시에 네트워크를 사용할 수 있도록, 포트 번호로 데이터 흐름을 구분한다.

```text
  브라우저(52341)  메일(52342)  SSH(52343)
        │              │              │
        └──────────────┴──────────────┘
                       │
                  전송 계층
                       │
                  (포트로 식별)
                       │
                  네트워크 계층
```

포트 번호는 16비트(0~65535)이며 아래와 같이 나뉜다.

| 범위 | 분류 | 용도 |
|:---|:---|:---|
| 0 ~ 1023 | Well-Known | 표준 서비스 (HTTP, HTTPS 등) |
| 1024 ~ 49151 | Registered | 등록된 애플리케이션 |
| 49152 ~ 65535 | Dynamic | 클라이언트 임시 포트 |

### 소켓

`IP 주소 + 포트 번호`를 소켓(Socket)이라 하며, 두 소켓의 조합이 하나의 연결을 식별한다.

```text
클라이언트 192.168.1.10:52341
        ↔
서버     203.0.113.50:80
```

## 4. TCP vs UDP

### 핵심 차이

| 항목 | TCP | UDP |
|:---|:---|:---|
| 연결 | 연결 지향 (3-way handshake) | 비연결형 |
| 신뢰성 | 보장 (ACK, 재전송) | 미보장 |
| 순서 | 보장 | 미보장 |
| 속도 | 느림 | 빠름 |
| 헤더 | 20~60 바이트 | 8 바이트 |
| 흐름/혼잡 제어 | 있음 | 없음 |
| 브로드캐스트 | 불가 | 가능 |

### 비유

- **TCP = 등기우편**: 수취 확인, 분실 시 재발송, 느리지만 확실
- **UDP = 엽서**: 그냥 부치고 끝, 빠르지만 도착 보장 없음

### 사용 사례

| 프로토콜 | 사용처 | 이유 |
|:---|:---|:---|
| TCP | HTTP/HTTPS, 이메일, FTP, SSH | 정확성·신뢰성이 필수 |
| UDP | DNS, DHCP, VoIP, 스트리밍, 게임 | 실시간성·저지연이 중요 |

{{< callout type="info" >}}
재전송한 음성 프레임은 이미 지나간 시점의 소리라 의미가 없다. 이런 실시간 데이터는 TCP보다 UDP가 적합하다.
{{< /callout >}}

## 5. 신뢰성 메커니즘 (TCP)

TCP는 세 가지 장치로 신뢰성을 확보한다.

### 1) 순서 번호 (Sequence Number)

각 바이트에 번호를 부여해 수신 측이 원래 순서로 재조립한다.

```text
송신: [1][2][3][4][5]
수신: [1][3][2][5][4]  ← 순서 뒤바뀜
정렬: [1][2][3][4][5]  ← seq 번호로 복구
```

### 2) 확인 응답 (ACK)

수신 측은 "다음에 기대하는 번호"를 ACK로 알린다. 일정 시간 내 ACK가 없으면 송신 측이 재전송한다.

```text
송신 ─ 데이터 #2 ─→ 수신 (손실)
송신 ─ 데이터 #2 ─→ 수신 (재전송 후 ACK 수신)
```

### 3) 체크섬

헤더와 데이터에 체크섬을 계산·첨부한다. 수신 측이 다시 계산해 값이 다르면 패킷을 폐기하고 재전송을 유도한다.

## 6. 흐름 제어와 혼잡 제어 (TCP)

### 흐름 제어

송신 속도와 수신 처리 속도의 차이를 조율한다. 수신측은 **윈도우 크기**로 "앞으로 몇 바이트 더 받을 수 있는지"를 알린다. 0이 되면 전송이 잠시 중단된다.

```text
수신 버퍼 ──▓▓▓▓▓░░░░── 윈도우=5
수신 버퍼 ──▓▓▓▓▓▓▓▓▓▓── 윈도우=0 → 전송 중단
수신 버퍼 ──▓▓░░░░░░░░── 윈도우=8 → 재개
```

### 혼잡 제어

네트워크 전체가 막히지 않도록 송신 속도를 조절한다. 주요 단계는 다음과 같다.

| 단계 | 동작 |
|:---|:---|
| Slow Start | 윈도우를 1부터 시작, ACK마다 2배 증가 (지수) |
| Congestion Avoidance | 임계값 이후 1씩 선형 증가 |
| Fast Retransmit | 중복 ACK 3개 시 즉시 재전송 |
| Fast Recovery | 혼잡 감지 시 윈도우 절반으로 축소 후 선형 증가 |

## 7. TCP 연결 수립·해제 요약

### 3-way handshake (수립)

```text
클라이언트 ─ SYN ─────────→ 서버
클라이언트 ←─ SYN+ACK ────── 서버
클라이언트 ─ ACK ─────────→ 서버
              ESTABLISHED
```

### 4-way handshake (해제)

```text
클라이언트 ─ FIN ─────────→ 서버
클라이언트 ←─ ACK ────────── 서버
클라이언트 ←─ FIN ────────── 서버
클라이언트 ─ ACK ─────────→ 서버
```

상세 내용은 [24. TCP의 구조](24-tcp-structure)에서 다룬다.

## 8. UDP 통신 요약

UDP는 연결 과정이 없다. 송신측은 데이터그램을 만들어 곧장 전송하고, 수신 여부는 확인하지 않는다. 헤더는 8바이트 고정으로 매우 단순하다.

```text
클라이언트 ─ 데이터그램 #1 ─→ 서버 (수신)
클라이언트 ─ 데이터그램 #2 ─→ 서버 (손실, 재전송 없음)
클라이언트 ─ 데이터그램 #3 ─→ 서버 (수신)
```

DNS 쿼리처럼 한 번의 요청-응답으로 끝나는 짧은 통신이나, 스트리밍처럼 손실이 조금 있어도 괜찮은 실시간 통신에 적합하다.

## 9. 프로토콜 선택 기준

```text
        데이터 손실 허용?
              │
        ┌─────┴─────┐
       예           아니오
        │             │
       UDP           TCP
        │             │
  스트리밍, DNS    HTTP, 이메일, SSH
  게임, VoIP       파일 전송, DB
```

## 핵심 정리

| 개념 | 설명 |
|:---|:---|
| 전송 계층 | OSI 4계층, 종단 간 신뢰성 있는 데이터 전송 |
| 멀티플렉싱 | 포트로 여러 애플리케이션을 구분 |
| 포트 번호 | 16비트 (0~65535), Well-Known/Registered/Dynamic |
| 소켓 | IP + Port 조합 |
| TCP | 연결 지향, 신뢰성·순서 보장, 흐름·혼잡 제어 |
| UDP | 비연결형, 빠른 전송, 실시간 애플리케이션용 |
| 3-way / 4-way | TCP 연결 수립 / 해제 절차 |
| 흐름 제어 | 윈도우 크기로 수신측 처리 속도 맞추기 |
| 혼잡 제어 | Slow Start, Congestion Avoidance 등으로 네트워크 보호 |

{{< callout type="info" >}}
**용어 정리**
- **전송 계층 (Transport Layer)**: OSI 4계층, 애플리케이션 간 통신 담당
- **TCP**: 연결 지향, 신뢰성 보장 프로토콜
- **UDP**: 비연결형, 저지연 프로토콜
- **포트 번호**: 16비트 애플리케이션 식별자
- **세그먼트 / 데이터그램**: TCP / UDP의 데이터 단위
- **멀티플렉싱**: 여러 애플리케이션을 하나의 네트워크 인터페이스에서 구분
- **ACK**: 수신 확인 응답
- **흐름 제어**: 송수신 속도 조절
- **혼잡 제어**: 네트워크 혼잡 회피
- **체크섬**: 오류 검출을 위한 값
{{< /callout >}}
