## 1. 물리 계층 장비 개요

### 물리 계층 장비의 역할

OSI 1계층(물리 계층)에서 동작하는 장비는 전기 신호 자체를 다룬다. 데이터의 내용이나 주소를 해석하지 않고 들어온 신호를 그대로 재생·증폭하여 내보낸다.

| 장비 | 포트 수 | 주요 기능 |
|:---|:---|:---|
| 리피터 | 2 | 신호 증폭·재생 |
| 허브 | 4~24 | 다중 연결 + 증폭 |

---

## 2. 리피터

### 신호 감쇄와 리피터의 필요성

전기 신호는 케이블을 통과하면서 점점 약해진다(감쇄). 이대로 두면 수신측에서 신호를 제대로 해석할 수 없으므로, 중간에 리피터를 두어 약해진 신호를 정형·증폭한다.

```
[PC A] ── 100m ── [리피터]
                     │
                     └── 100m ── [PC B]
```

### 리피터 동작 과정

```
1. 수신부: 약해진 신호 수신
2. 정형  : 일그러진 파형 복원
3. 증폭  : 신호 크기 복원
4. 송신부: 다음 구간으로 전송
```

### 케이블별 최대 전송 거리

| 케이블 | 최대 거리 |
|:---|:---|
| UTP (Cat 5e/6) | 100 m |
| 동축 (10BASE2) | 185 m |
| 광케이블 SMF | 수십 km |

## 3. 허브

### 허브의 구조

허브는 포트를 여러 개 가진 다중 포트 리피터이다. 백플레인(공유 버스)에 모든 포트가 연결되어 있어 수신한 신호를 나머지 모든 포트로 그대로 내보낸다.

```
  ┌──────────────────────────┐
  │    공유 버스(백플레인)    │
  └──┬────┬────┬────┬────┬──┘
     │    │    │    │    │
     P1   P2   P3   P4   P5
```

### 플러딩(Flooding) 동작

PC-A가 PC-C에게 보내더라도 허브는 목적지를 판단하지 못하므로 수신 포트를 제외한 모든 포트로 신호를 전송한다. 이를 플러딩이라고 한다.

```
[PC-A] ──→ [허브]
              │→ [PC-B] (무시)
              │→ [PC-C] (수신)
              └→ [PC-D] (무시)
```

{{< callout type="warning" >}}
허브는 목적지 주소를 확인하지 않는다. 그래서 "더미 허브(Dummy Hub)"라 부르기도 한다. 이 구조는 불필요한 트래픽과 도청 위험을 야기하며, 현재 네트워크에서는 거의 사용하지 않고 스위치로 대체되었다.
{{< /callout >}}

## 4. 콜리전 도메인과 CSMA/CD

### 콜리전(충돌)

같은 공유 매체에서 두 장비가 동시에 신호를 보내면 파형이 겹쳐 깨진다. 이 현상을 콜리전이라 하며, 충돌의 영향이 미치는 범위를 콜리전 도메인이라 한다.

```
허브 환경: 전체가 하나의 콜리전 도메인
  [PC-A] ╗
         ╠→ [HUB] → 충돌!
  [PC-B] ╝
```

{{< callout type="warning" >}}
허브에 연결된 장비들은 하나의 콜리전 도메인을 공유한다. 연결 장비가 늘어날수록 충돌 빈도가 증가하고 실효 대역폭이 급격히 떨어진다.
{{< /callout >}}

### CSMA/CD 규칙

반이중 이더넷에서 충돌을 다루는 접근 제어 방식이다.

| 약어 | 의미 | 동작 |
|:---|:---|:---|
| CS | Carrier Sense | 케이블이 비었는지 확인 |
| MA | Multiple Access | 비었으면 전송 시작 |
| CD | Collision Detection | 충돌 감지 시 재전송 |

충돌이 감지되면 JAM 신호로 알린 뒤 랜덤 시간 대기(백오프) 후 재전송한다.

## 5. 반이중과 전이중

| 항목 | 반이중 (허브) | 전이중 (스위치) |
|:---|:---|:---|
| 동시 통신 | 불가 | 가능 |
| 충돌 | 발생 | 없음 |
| CSMA/CD | 필요 | 불필요 |
| 비유 | 워키토키 | 전화 |

허브 환경에서는 같은 회선으로 송수신을 번갈아 수행하지만, 스위치 환경에서는 송신선과 수신선이 분리되어 동시에 양방향 통신이 가능하다.

---

## 6. 허브에서 스위치로

과거에는 허브가 널리 쓰였지만 다음 이유로 현재는 거의 사용되지 않는다.

| 허브의 한계 | 스위치의 개선 |
|:---|:---|
| 모든 포트 플러딩 | 목적지 포트로만 전달 |
| 하나의 콜리전 도메인 | 포트별 도메인 분리 |
| 반이중·대역폭 공유 | 전이중·대역폭 독립 |
| 도청 위험 | 목적지만 수신 |

{{< callout type="info" >}}
허브와 스위치의 가격 차이가 거의 없어지면서 허브는 시장에서 퇴출되었다. 리피터 역시 대부분 이더넷 스위치·광 변환기로 대체되었다.
{{< /callout >}}

## 핵심 정리

| 개념 | 설명 |
|:---|:---|
| 리피터 | 신호를 정형·증폭하는 2포트 1계층 장비 |
| 허브 | 다중 포트 리피터, 모든 포트로 플러딩 |
| 콜리전 도메인 | 충돌 영향이 미치는 범위 |
| CSMA/CD | 반이중 환경의 충돌 감지·재전송 규칙 |
| 반이중/전이중 | 교대 통신 / 동시 양방향 통신 |

{{< callout type="info" >}}
**용어 정리**
- **리피터 (Repeater)**: 약해진 신호를 정형하고 증폭하는 1계층 장비
- **허브 (Hub)**: 여러 포트를 가진 리피터, 플러딩으로 동작
- **플러딩 (Flooding)**: 수신 포트를 제외한 모든 포트로 전송
- **콜리전 (Collision)**: 두 신호가 동시에 전송되어 겹치는 현상
- **CSMA/CD**: 충돌 감지 기반의 매체 접근 제어 방식
{{< /callout >}}
