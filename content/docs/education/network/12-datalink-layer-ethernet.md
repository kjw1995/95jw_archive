## 1. 데이터 링크 계층 개요

### 위치와 역할

데이터 링크 계층은 OSI 2계층으로, 동일한 LAN 구간에서 장비 간 데이터를 정확히 주고받기 위한 규칙을 정의한다. 물리 계층이 전달한 비트열에 의미를 부여하여 프레임(Frame) 단위로 다룬다.

```
  7  응용 계층
  6  표현 계층
  5  세션 계층
  4  전송 계층
  3  네트워크 계층
  2  데이터 링크 계층  ← 여기
  1  물리 계층
```

### 주요 기능

| 기능 | 설명 |
|:---|:---|
| 프레이밍 | 비트열을 프레임 단위로 구성 |
| 물리 주소 | MAC 주소로 장비 식별 |
| 오류 제어 | FCS(CRC)로 프레임 오류 검출 |
| 흐름 제어 | 송수신 속도 조절 |
| 접근 제어 | 공유 매체 접근 규칙(CSMA/CD 등) |

### 하위 계층 구분

| 하위 계층 | 역할 |
|:---|:---|
| LLC (Logical Link Control) | 상위 프로토콜 식별, 흐름/오류 제어 |
| MAC (Media Access Control) | 물리 주소 지정, 매체 접근 제어 |

## 2. 이더넷

### 이더넷이란?

LAN에서 가장 널리 사용되는 데이터 링크 프로토콜이다. IEEE 802.3으로 표준화되었고, MAC 주소 기반으로 프레임을 주고받는다.

| 특징 | 설명 |
|:---|:---|
| 표준 | IEEE 802.3 |
| 주소 | 48bit MAC |
| 단위 | 프레임(Frame) |
| 접근 제어 | CSMA/CD (반이중), Full-Duplex |

---

## 3. 이더넷 프레임 구조

### 필드 구성

```
┌────┬────┬────┬───┬──────┬────┐
│PRE │DST │SRC │TYP│ DATA │FCS │
└────┴────┴────┴───┴──────┴────┘
 8B   6B   6B   2B  46-1500B 4B
```

| 필드 | 크기 | 설명 |
|:---|:---|:---|
| 프리앰블 + SFD | 8 B | 동기화 + 프레임 시작 |
| 목적지 MAC | 6 B | 수신 장비 주소 |
| 출발지 MAC | 6 B | 송신 장비 주소 |
| 타입/EtherType | 2 B | 상위 프로토콜 식별 |
| 데이터 | 46~1500 B | 페이로드(IP 패킷 등) |
| FCS | 4 B | CRC-32 체크섬 |

### 주요 EtherType

| 값 | 프로토콜 |
|:---|:---|
| 0x0800 | IPv4 |
| 0x0806 | ARP |
| 0x86DD | IPv6 |
| 0x8100 | VLAN (802.1Q) |

{{< callout type="info" >}}
이더넷 프레임의 최소 크기는 64바이트(헤더 14 + 데이터 46 + FCS 4)이다. 이 하한은 CSMA/CD에서 충돌을 안정적으로 감지하기 위한 최소 전송 시간을 보장한다.
{{< /callout >}}

## 4. CSMA/CD

### 원리

반이중 이더넷에서 공유 매체의 충돌을 제어하는 규칙이다.

```
 ┌──────────────┐
 │ 전송할 데이터 │
 └──────┬───────┘
        ▼
   매체가 비었는가?
    │          │
    No         Yes
    │          │
   대기      전송 시작
              │
              ▼
         충돌 감지?
          │       │
         Yes      No
          │       │
       JAM+대기  완료
```

### 동작 단계

1. **CS (Carrier Sense)**: 케이블에 신호가 있는지 감지
2. **MA (Multiple Access)**: 비어 있으면 전송 시작
3. **CD (Collision Detection)**: 전송 중 충돌을 감지하면 JAM 신호 송출
4. **백오프**: 충돌 횟수 n에 대해 0~(2^n - 1) 중 랜덤 슬롯만큼 대기 후 재전송 (최대 16회)

## 5. 듀플렉스 (Duplex)

### 반이중과 전이중

| 항목 | 반이중 (Half) | 전이중 (Full) |
|:---|:---|:---|
| 송·수신 | 교대 | 동시 |
| 회선 | 공유 | 분리 |
| 충돌 | 발생 | 없음 |
| CSMA/CD | 필요 | 불필요 |
| 실효 대역폭 | 50% 이하 | 100% × 2 |
| 대표 환경 | 허브 | 스위치 |

UTP 케이블은 1,2번 쌍을 송신, 3,6번 쌍을 수신으로 분리하기 때문에 전이중 통신이 가능하다. 기가비트(1000BASE-T)부터는 4쌍을 모두 사용하며, 에코 캔슬레이션 기술로 양방향 통신을 구현한다.

{{< callout type="warning" >}}
허브와 스위치를 혼용할 때 한쪽은 Full, 다른 쪽은 Half로 협상되는 "듀플렉스 미스매치"가 발생할 수 있다. 이 경우 간헐적 패킷 손실과 성능 저하를 유발하므로 양단의 설정을 일치시켜야 한다.
{{< /callout >}}

## 6. 프레임 송수신 과정

### 송신측

```
상위 계층 데이터
   ↓  (IP 헤더 추가 → 패킷)
   ↓  (이더넷 헤더 + FCS 추가)
  프레임
   ↓  (비트열 → 전기 신호)
  케이블
```

### 수신측

```
케이블 (전기 신호)
   ↓
 프레임 복원
   ↓  FCS 검증
   ↓  목적지 MAC 확인
   ↓  (자신/브로드캐스트면 수신)
 상위 계층으로 전달
```

FCS가 일치하지 않거나 목적지 MAC이 자신이 아니면 프레임은 폐기된다.

## 핵심 정리

| 개념 | 설명 |
|:---|:---|
| 데이터 링크 계층 | OSI 2계층, 프레임·MAC 주소 관리 |
| 이더넷 | LAN의 표준 데이터 링크 프로토콜 |
| 프레임 | 헤더 + 데이터 + FCS로 구성된 단위 |
| CSMA/CD | 반이중 충돌 감지·재전송 규칙 |
| Full Duplex | 동시 양방향 통신, 충돌 없음 |

{{< callout type="info" >}}
**용어 정리**
- **이더넷 (Ethernet)**: IEEE 802.3으로 정의된 LAN 프로토콜
- **프레임 (Frame)**: 데이터 링크 계층의 데이터 단위
- **FCS (Frame Check Sequence)**: CRC-32 기반 오류 검출 필드
- **CSMA/CD**: Carrier Sense Multiple Access / Collision Detection
- **Full / Half Duplex**: 동시 양방향 / 교대 통신
- **LLC / MAC**: 데이터 링크 계층의 두 하위 계층
{{< /callout >}}
