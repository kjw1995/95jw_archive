## 1. 범위에 따른 분류

네트워크는 연결되는 장치들의 지리적 범위에 따라 분류한다.

```
┌──────────────────────┐
│      인터넷          │
│  ┌────────────────┐  │
│  │      WAN       │  │
│  │  ┌──────────┐  │  │
│  │  │   MAN    │  │  │
│  │  │ ┌──────┐ │  │  │
│  │  │ │ LAN  │ │  │  │
│  │  │ │ ┌──┐ │ │  │  │
│  │  │ │ │P │ │ │  │  │
│  │  │ │ │AN│ │ │  │  │
│  │  │ │ └──┘ │ │  │  │
│  │  │ └──────┘ │  │  │
│  │  └──────────┘  │  │
│  └────────────────┘  │
└──────────────────────┘
```

## 2. LAN (Local Area Network)

건물, 사무실, 학교 등 제한된 지역 안에서 기기를 연결하는 근거리 통신망이다.

### 구성 예시

```
   ┌──┐ ┌──┐ ┌──┐
   │PC│ │PC│ │PC│
   └─┬┘ └─┬┘ └─┬┘
     └────┼────┘
       ┌──┴──┐
       │스위치│
       └──┬──┘
     ┌────┼────┐
   ┌─┴┐ ┌─┴┐ ┌─┴┐
   │서│ │프│ │NA│
   │버│ │린│ │S │
   └──┘ └──┘ └──┘
```

### 특징

| 항목 | 내용 |
|:---|:---|
| 범위 | 건물·캠퍼스 (수 km 이내) |
| 속도 | 100Mbps ~ 10Gbps |
| 소유 | 조직이 직접 구축·관리 |
| 지연 | 매우 낮음 (1ms 이하) |
| 오류율 | 낮음 |

### 토폴로지

| 형태 | 설명 | 용도 |
|:---|:---|:---|
| 스타 | 중앙 장비에 모두 연결 | 가장 일반적 |
| 버스 | 하나의 버스에 여러 장비 연결 | 과거 방식 |
| 링 | 고리 모양 연결 | 토큰링 |
| 메시 | 서로 직접 연결 | 고가용 백본 |

---

## 3. WAN (Wide Area Network)

도시, 국가, 대륙에 걸친 광역 통신망이다. 통신사(ISP)가 회선을 제공한다.

### 구성 개념

```
 서울 지사            부산 본사
 ┌───────┐           ┌───────┐
 │  LAN  │           │  LAN  │
 └───┬───┘           └───┬───┘
     │                   │
     │    ┌────────┐     │
     └────┤  ISP   ├─────┘
          │ 네트워크│
          └────────┘
```

### 특징

| 항목 | 내용 |
|:---|:---|
| 범위 | 도시·국가·전 세계 |
| 속도 | 수 Mbps ~ 수 Gbps |
| 소유 | ISP가 관리 (임대 사용) |
| 지연 | 상대적으로 큼 |
| 오류율 | LAN보다 높음 |

### ISP 계층

| Tier | 설명 | 예시 |
|:---|:---|:---|
| Tier 1 | 국제 백본망 | AT&T, NTT |
| Tier 2 | 국가·지역 사업자 | KT, SKB, LG U+ |
| Tier 3 | 지역·소규모 | 지역 케이블사 |

---

## 4. MAN (Metropolitan Area Network)

도시 규모의 네트워크로, LAN과 WAN 중간 규모다. 대학 캠퍼스 연결, 도시 CCTV망, 지자체 전용망 등이 해당한다.

| 항목 | 내용 |
|:---|:---|
| 범위 | 수십 km |
| 기술 | 광케이블 주류 |
| 예시 | 지하철 Wi-Fi, 케이블TV, 캠퍼스망 |

---

## 5. PAN (Personal Area Network)

개인 주위 수 m 범위의 기기 간 통신망이다. 블루투스 이어폰, 스마트워치, USB 연결 등이 이에 속한다.

---

## 6. LAN vs WAN 비교

| 비교 항목 | LAN | WAN |
|:---|:---|:---|
| 범위 | 건물·캠퍼스 | 도시·국가·전 세계 |
| 속도 | 빠름 (~10Gbps) | 느림 (~수 Gbps) |
| 지연 | 1ms 미만 | 10~300ms |
| 오류율 | 낮음 | 높음 |
| 소유 | 조직 자체 | ISP |
| 구축 비용 | 초기 투자 | 월 이용료 |
| 보안 | 통제 용이 | 복잡 |

{{< callout type="info" >}}
가정의 공유기 한 대는 WAN 포트로 ISP에 연결되고, LAN 포트·Wi-Fi로 내부 기기를 연결해 **LAN과 WAN을 동시에 담당**한다.
{{< /callout >}}

## 7. 인터넷의 기원

| 시기 | 사건 |
|:---|:---|
| 1969 | ARPANET 4개 노드 연결 (UCLA, 스탠포드, UCSB, 유타대) |
| 1970s | TCP/IP 프로토콜 개발 |
| 1983 | ARPANET이 TCP/IP로 전환 |
| 1991 | 팀 버너스 리의 WWW 발명 |
| 1990s | 상용 인터넷 보급 시작 |

ARPANET은 미국 국방부 ARPA가 주관한 프로젝트로, 패킷 교환 방식을 실증한 최초의 네트워크다.

## 핵심 정리

| 용어 | 핵심 |
|:---|:---|
| LAN | 근거리 통신망, 조직 관리 |
| WAN | 광역 통신망, ISP 관리 |
| MAN | 도시권 통신망 |
| PAN | 개인 영역 통신망 |
| ISP | 인터넷 서비스 제공자 |
| ARPANET | 인터넷의 시초 (1969) |

{{< callout type="info" >}}
**용어 정리**
- **LAN**: Local Area Network
- **WAN**: Wide Area Network
- **MAN**: Metropolitan Area Network
- **PAN**: Personal Area Network
- **ISP**: Internet Service Provider
{{< /callout >}}
