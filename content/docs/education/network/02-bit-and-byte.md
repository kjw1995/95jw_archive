## 1. 디지털 데이터

컴퓨터가 처리하는 데이터는 모두 0과 1의 조합이다. 자연계의 연속적인 정보(아날로그)를 불연속적인 신호(디지털)로 표현해 저장·전송한다.

| 구분 | 특징 | 예 |
|:---|:---|:---|
| 아날로그 | 연속적인 파형 | 음성, 자연의 소리 |
| 디지털 | 0과 1의 불연속 신호 | 컴퓨터 데이터 |

---

## 2. 비트 (bit)

정보를 나타내는 최소 단위이며, Binary Digit의 줄임말이다. 전기 신호의 ON/OFF로 표현된다.

```
┌───────────────┐
│    1 bit      │
│  ┌─┐    ┌─┐   │
│  │0│ or │1│   │
│  └─┘    └─┘   │
│  OFF    ON    │
└───────────────┘
```

### 비트로 표현할 수 있는 가짓수

| 비트 수 | 경우의 수 |
|:---|:---|
| 1 bit | 2 (2¹) |
| 2 bit | 4 (2²) |
| 4 bit | 16 (2⁴) |
| 8 bit | 256 (2⁸) |
| 16 bit | 65,536 (2¹⁶) |

n 비트로 2ⁿ 가지 값을 표현할 수 있다.

---

## 3. 바이트 (Byte)

8개의 비트를 묶은 단위를 1 바이트라 한다. 영문 1글자를 표현할 수 있는 크기다.

```
┌───┬───┬───┬───┬───┬───┬───┬───┐
│ 0 │ 1 │ 0 │ 0 │ 0 │ 0 │ 0 │ 1 │
└───┴───┴───┴───┴───┴───┴───┴───┘
 MSB                         LSB
  = 65 (10진)  = 'A' (ASCII)
```

### 데이터 크기 단위

| 단위 | 환산 (2진 기준) |
|:---|:---|
| 1 Byte | 8 bits |
| 1 KB | 1,024 Bytes |
| 1 MB | 1,024 KB |
| 1 GB | 1,024 MB |
| 1 TB | 1,024 GB |

{{< callout type="info" >}}
운영체제는 보통 1024배 단위(KiB, MiB)를 쓰지만 저장장치 제조사는 1000배 단위(KB, MB)를 쓰는 경우가 많다. 표시 용량과 실제 용량이 달라 보이는 이유다.
{{< /callout >}}

## 4. 진법 변환

컴퓨터 내부는 2진수를 쓰지만, 사람이 보기 편하도록 10진수와 16진수로도 표기한다.

### 변환 예시

| 10진 | 2진 | 16진 |
|:---|:---|:---|
| 0 | 0000 | 0x0 |
| 10 | 1010 | 0xA |
| 15 | 1111 | 0xF |
| 16 | 10000 | 0x10 |
| 255 | 11111111 | 0xFF |

### 2진 → 10진

```
비트:   1  0  1  1  0  1
가중치: 32 16  8  4  2  1
값:    32 + 0 + 8 + 4 + 0 + 1 = 45
```

### 16진 → 2진

16진수 한 자리는 2진수 4자리에 대응되어 변환이 간편하다.

| 16진 | 2진 |
|:---|:---|
| 0xA | 1010 |
| 0xB | 1011 |
| 0xF | 1111 |
| 0x2A | 0010 1010 |

## 5. 문자 인코딩

숫자와 문자를 대응시킨 약속을 문자 코드 또는 문자 인코딩이라 한다.

### ASCII 코드

7비트로 128개의 영문자·숫자·기호를 표현한다. 한글·한자 표현은 불가능하다.

| 문자 | 10진 | 2진 |
|:---|:---|:---|
| 'A' | 65 | 01000001 |
| 'a' | 97 | 01100001 |
| '0' | 48 | 00110000 |
| ' ' | 32 | 00100000 |

### 주요 인코딩 비교

| 인코딩 | 길이 | 특징 |
|:---|:---|:---|
| ASCII | 7비트 | 영문 128자 |
| EUC-KR | 2바이트 | 한글 완성형 |
| CP949 | 2바이트 | EUC-KR 확장 |
| UTF-8 | 1~4바이트 가변 | 유니코드, 웹 표준 |
| UTF-16 | 2~4바이트 | JVM·Windows 내부 |

### UTF-8 가변 길이

```
범위                 바이트  예시
─────────────────────────────────────
U+0000 ~ U+007F      1       'A'  → 0x41
U+0080 ~ U+07FF      2       'ñ'  → 0xC3 0xB1
U+0800 ~ U+FFFF      3       '가' → 0xEA 0xB0 0x80
U+10000 ~ U+10FFFF   4       '😀' → 0xF0 0x9F 0x98 0x80
```

{{< callout type="info" >}}
UTF-8은 ASCII와 하위 호환되면서(영문은 1바이트) 전 세계 문자를 표현할 수 있어 현재 웹·리눅스·최신 시스템의 기본 인코딩으로 자리잡았다.
{{< /callout >}}

## 6. 데이터 전송과 전기 신호

네트워크에서 데이터는 비트 → 전기 신호(또는 빛·전파)로 변환되어 전달된다.

```
     송신                    수신
  ┌────────┐             ┌────────┐
  │  'Hi'  │             │  'Hi'  │
  └───┬────┘             └───▲────┘
      ▼                      │
  01001000                01001000
  01101001                01101001
      ▼                      │
   ▁█▁▁█▁▁▁  ═══ 전송 ═══  ▁█▁▁█▁▁▁
```

### 전송 매체별 신호

| 매체 | 신호 | 특징 |
|:---|:---|:---|
| UTP 케이블 | 전압 | 100m 이내 |
| 광케이블 | 빛 | 고속·장거리 |
| Wi-Fi | 전파 | 이동성, 간섭 |

## 7. 비트 연산

### 논리 연산

| 연산 | 기호 | 예 (4비트) |
|:---|:---|:---|
| AND | `&` | 1010 & 1100 = 1000 |
| OR | `\|` | 1010 \| 1100 = 1110 |
| XOR | `^` | 1010 ^ 1100 = 0110 |
| NOT | `~` | ~1010 = 0101 |

### 시프트 연산

```java
// 왼쪽 시프트 (×2ⁿ)
10 << 2 = 40      // 00001010 → 00101000

// 오른쪽 시프트 (÷2ⁿ)
40 >> 2 = 10      // 00101000 → 00001010
```

## 핵심 정리

| 용어 | 핵심 |
|:---|:---|
| 비트 | 최소 단위 (0 또는 1) |
| 바이트 | 8비트, 영문 1글자 |
| ASCII | 7비트 영문 인코딩 |
| UTF-8 | 유니코드, 웹 표준 |
| 2진/16진 | 컴퓨터 내부 표기 |

{{< callout type="info" >}}
**용어 정리**
- **bit**: Binary Digit, 정보의 최소 단위
- **Byte**: 8 비트, 주소 지정 기본 단위
- **MSB / LSB**: 최상위/최하위 비트
- **인코딩**: 문자를 숫자로 변환하는 규칙
- **ASCII**: 미국 표준 문자 코드 (128자)
- **UTF-8**: 가변 길이 유니코드 인코딩
{{< /callout >}}
