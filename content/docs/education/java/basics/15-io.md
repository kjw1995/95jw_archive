## 1. 자바에서의 입출력

### 1.1 입출력이란?

I/O(Input/Output)는 프로그램과 외부(파일·네트워크·콘솔·메모리) 간에 데이터를 주고받는 것을 말한다. 자바는 `java.io` 패키지를 통해 **스트림(stream) 추상화**로 이를 일관되게 다룬다.

### 1.2 스트림

스트림은 **데이터를 운반하는 단방향 연결 통로**다.

```
┌──────────┐  InputStream   ┌──────────┐
│          │ ◀───────────── │          │
│ 프로그램  │                │ I/O 대상  │
│          │ ─────────────▶ │          │
└──────────┘  OutputStream  └──────────┘
```

| 특성 | 설명 |
|:---|:---|
| 단방향 | 입력·출력을 동시에 하려면 스트림 2개 필요 |
| FIFO | 먼저 보낸 데이터가 먼저 읽힘 |
| 연속적 | 중간에 건너뜀 없이 순차 전달 |

### 1.3 스트림의 분류

```
              스트림
      ┌─────────┴─────────┐
   바이트 기반           문자 기반
   (1 byte)             (2 byte)
   ┌───┴───┐           ┌───┴───┐
InputStr OutputStr   Reader  Writer
```

바이트 스트림은 모든 종류의 데이터(이미지, 동영상, 바이너리)를 다루고, 문자 스트림은 인코딩을 고려한 텍스트 처리에 적합하다.

## 2. 바이트 기반 스트림

### 2.1 InputStream / OutputStream

모든 바이트 기반 스트림의 추상 조상이다.

| 입력 | 출력 | 대상 |
|:---|:---|:---|
| `FileInputStream` | `FileOutputStream` | 파일 |
| `ByteArrayInputStream` | `ByteArrayOutputStream` | 메모리 |
| `PipedInputStream` | `PipedOutputStream` | 쓰레드 간 통신 |
| `AudioInputStream` | `AudioOutputStream` | 오디오 장치 |

**InputStream 주요 메서드**

| 메서드 | 설명 |
|:---|:---|
| `int read()` | 1 byte 읽기, EOF면 -1 |
| `int read(byte[] b)` | 배열 크기만큼 읽기 |
| `int read(byte[] b, int off, int len)` | 지정 범위만큼 읽기 |
| `int available()` | 블로킹 없이 읽을 수 있는 바이트 수 |
| `long skip(long n)` | n byte 건너뛰기 |
| `void close()` | 자원 반환 |

**OutputStream 주요 메서드**

| 메서드 | 설명 |
|:---|:---|
| `void write(int b)` | 1 byte 출력 |
| `void write(byte[] b)` | 배열 전체 출력 |
| `void write(byte[] b, int off, int len)` | 지정 범위 출력 |
| `void flush()` | 버퍼 내용 강제 배출 |
| `void close()` | 자원 반환 |

{{< callout type="warning" >}}
**스트림은 반드시 닫아야 한다.** 파일·소켓 핸들 같은 OS 자원이 연결되어 있으므로, 닫지 않으면 자원 누수와 파일 락 문제가 생긴다. **`try-with-resources`** 문을 사용해 자동으로 닫도록 하는 것이 표준이다. (단, `ByteArrayInputStream`·`System.in`·`System.out`처럼 메모리 기반은 예외로 해도 무방하다.)
{{< /callout >}}

```java
// try-with-resources — 블록을 벗어날 때 자동 close
try (FileInputStream fis = new FileInputStream("input.txt")) {
    int data;
    while ((data = fis.read()) != -1) {
        System.out.print((char) data);
    }
} catch (IOException e) {
    e.printStackTrace();
}
```

### 2.2 FileInputStream / FileOutputStream

```java
// 파일 복사
try (FileInputStream fis = new FileInputStream("src.txt");
     FileOutputStream fos = new FileOutputStream("dst.txt")) {
    int b;
    while ((b = fis.read()) != -1) fos.write(b);
}

// 이어쓰기 모드
new FileOutputStream("log.txt", true);
```

### 2.3 ByteArrayInputStream / ByteArrayOutputStream

메모리(byte 배열)에 데이터를 입출력한다. 테스트·임시 버퍼·포맷 변환에 유용하다.

```java
byte[] src = {0, 1, 2, 3, 4};
ByteArrayInputStream  in  = new ByteArrayInputStream(src);
ByteArrayOutputStream out = new ByteArrayOutputStream();

int b;
while ((b = in.read()) != -1) out.write(b);

byte[] copied = out.toByteArray();
```

{{< callout type="info" >}}
**블로킹 I/O의 의미:** `read()`는 데이터가 올 때까지 쓰레드를 멈춘다. 이 점이 클라이언트 수가 많은 서버에서는 한계가 되어 NIO(Non-blocking I/O)가 등장했다. `available()`은 블로킹 없이 읽을 수 있는 바이트 수를 힌트로 알려준다.
{{< /callout >}}

## 3. 바이트 기반 보조 스트림 (데코레이터 패턴)

보조 스트림은 **단독으로 동작하지 않으며**, 기반 스트림을 감싸서 기능을 추가한다. 이는 GoF의 **데코레이터 패턴**을 I/O에 적용한 대표적 사례다.

```
     ┌─────────────────────────────┐
     │   DataInputStream            │
     │ ┌───────────────────────┐   │
     │ │ BufferedInputStream   │   │
     │ │ ┌─────────────────┐   │   │
     │ │ │ FileInputStream │   │   │
     │ │ └─────────────────┘   │   │
     │ └───────────────────────┘   │
     └─────────────────────────────┘
        기능 3층: 형변환 읽기
        기능 2층: 버퍼링
        기능 1층: 파일 접근
```

```java
DataInputStream dis = new DataInputStream(
    new BufferedInputStream(
        new FileInputStream("data.bin")));
```

### 3.1 FilterInputStream / FilterOutputStream

모든 보조 스트림의 추상 조상이다. 자체 기능은 없고 상속해서 사용한다.

### 3.2 BufferedInputStream / BufferedOutputStream

내부에 **8 KB 버퍼**를 두어 OS 호출 횟수를 줄인다. 한 byte씩 읽는 코드라도 실제로는 8 KB씩 한 번에 읽어 메모리에서 꺼내 쓴다.

```
프로그램 ◀→ [Buffer 8 KB] ◀→ 파일
1. read() 호출
2. 버퍼가 비면 파일에서 8 KB 한 번에 읽기
3. 버퍼에서 1 byte 반환
```

```java
try (BufferedInputStream bis = new BufferedInputStream(
        new FileInputStream("large.dat"));
     BufferedOutputStream bos = new BufferedOutputStream(
        new FileOutputStream("copy.dat"))) {
    int b;
    while ((b = bis.read()) != -1) bos.write(b);
}
```

{{< callout type="warning" >}}
**BufferedOutputStream은 버퍼가 차야만 실제로 쓴다.** 프로그램이 비정상 종료되면 마지막 데이터가 디스크에 반영되지 않을 수 있다. `close()`는 내부적으로 `flush()`를 호출하지만, 중요한 시점에는 직접 `flush()`로 강제 배출하는 것이 안전하다.
{{< /callout >}}

### 3.3 DataInputStream / DataOutputStream

`int`, `float`, `boolean`, `UTF-8 String` 등 **기본 자료형 단위**로 읽고 쓴다.

```java
try (DataOutputStream dos = new DataOutputStream(
        new FileOutputStream("data.bin"))) {
    dos.writeInt(100);
    dos.writeFloat(3.14f);
    dos.writeBoolean(true);
    dos.writeUTF("Hello");
}

try (DataInputStream dis = new DataInputStream(
        new FileInputStream("data.bin"))) {
    int   i = dis.readInt();
    float f = dis.readFloat();
    boolean b = dis.readBoolean();
    String s = dis.readUTF();
}
```

{{< callout type="warning" >}}
**쓴 순서·타입대로 읽어야 한다.** DataInputStream은 각 타입의 바이트 크기가 정해져 있어 순서가 어긋나면 잘못된 값을 읽는다. 데이터 끝을 지나 읽으면 `EOFException`이 발생한다.
{{< /callout >}}

### 3.4 PrintStream

`print`, `println`, `printf` 메서드를 제공한다. `System.out`이 바로 `PrintStream`이다.

| 포맷 | 설명 | 예 (`i=65`) |
|:---|:---|:---|
| `%d` | 10진 | `65` |
| `%x` | 16진 | `41` |
| `%c` | 문자 | `A` |
| `%s` | 문자열 | `65` |
| `%5d` | 5자리 공백 정렬 | `   65` |
| `%05d` | 5자리 0 패딩 | `00065` |
| `%8.2f` | 8자리, 소수 2자리 | `   65.00` |

```java
System.out.printf("이름: %s, 점수: %.1f%n", name, score);
```

## 4. 문자 기반 스트림

Java의 `char`는 2 byte이므로 바이트 스트림으로 문자를 처리하면 인코딩 변환이 번거롭다. 문자 스트림은 **인코딩을 자동으로 적용**해 이를 해결한다.

### 4.1 바이트 ↔ 문자 대응

| 바이트 기반 | 문자 기반 |
|:---|:---|
| `FileInputStream` / `FileOutputStream` | `FileReader` / `FileWriter` |
| `ByteArrayInputStream` / `ByteArrayOutputStream` | `CharArrayReader` / `CharArrayWriter` |
| `PipedInputStream` / `PipedOutputStream` | `PipedReader` / `PipedWriter` |
| `BufferedInputStream` / `BufferedOutputStream` | `BufferedReader` / `BufferedWriter` |
| `PrintStream` | `PrintWriter` |

### 4.2 Reader / Writer

문자 기반 추상 조상이다. `byte[]` 대신 `char[]`를 사용한다.

| Reader | Writer |
|:---|:---|
| `int read()` | `void write(int c)` |
| `int read(char[] c)` | `void write(char[] c)` |
| `int read(char[] c, int off, int len)` | `void write(String str)` |
| `void close()` | `void flush()` / `void close()` |

### 4.3 FileReader / FileWriter

```java
try (FileReader fr = new FileReader("input.txt")) {
    int ch;
    while ((ch = fr.read()) != -1) System.out.print((char) ch);
}

try (FileWriter fw = new FileWriter("output.txt")) {
    fw.write("안녕하세요\n");
}
```

{{< callout type="warning" >}}
**`FileReader`·`FileWriter`는 JVM 기본 인코딩을 사용한다.** Windows에서는 보통 MS949, 리눅스·macOS에서는 UTF-8이므로 OS가 다르면 같은 파일을 깨뜨릴 수 있다. **항상 인코딩을 명시**하려면 `InputStreamReader`/`OutputStreamWriter`를 사용하거나 Java 11+의 `new FileReader(file, StandardCharsets.UTF_8)` 오버로드를 사용하라.
{{< /callout >}}

### 4.4 StringReader / StringWriter

입출력 대상이 문자열인 스트림. 로그 포매팅, 템플릿 렌더링 테스트 등에 사용한다.

```java
StringWriter sw = new StringWriter();
sw.write("Hello ");
sw.write("World");
String result = sw.toString();  // "Hello World"
```

## 5. 문자 기반 보조 스트림

### 5.1 BufferedReader / BufferedWriter

`readLine()`으로 **라인 단위 읽기**를 지원한다.

```java
try (BufferedReader br = new BufferedReader(
        new FileReader("input.txt"))) {
    String line;
    while ((line = br.readLine()) != null) {
        System.out.println(line);
    }
}

try (BufferedWriter bw = new BufferedWriter(
        new FileWriter("output.txt"))) {
    bw.write("첫 줄");
    bw.newLine();
    bw.write("둘째 줄");
}
```

### 5.2 InputStreamReader / OutputStreamWriter

**바이트 스트림과 문자 스트림을 연결하는 다리**이며, 인코딩을 명시할 수 있다.

```java
// 콘솔 입력 (바이트 → 문자 → 라인)
BufferedReader br = new BufferedReader(
    new InputStreamReader(System.in, StandardCharsets.UTF_8));
String input = br.readLine();

// 파일 읽을 때 인코딩 지정
InputStreamReader isr = new InputStreamReader(
    new FileInputStream("data.txt"), StandardCharsets.UTF_8);
```

{{< callout type="info" >}}
**Charset을 항상 명시하라.** `String.getBytes()`, `new String(bytes)`, `FileReader`, `FileWriter` 등 많은 API가 기본 인코딩에 의존한다. 플랫폼 독립적인 코드를 원하면 **`java.nio.charset.StandardCharsets`** 의 상수(`UTF_8`, `UTF_16`, `ISO_8859_1` 등)를 명시적으로 전달하라.
{{< /callout >}}

## 6. 표준 입출력과 파일 처리

### 6.1 표준 입출력

JVM 시작 시 자동으로 생성되는 3개 스트림이다.

| 스트림 | 타입 | 용도 |
|:---|:---|:---|
| `System.in` | `InputStream` | 콘솔 입력 |
| `System.out` | `PrintStream` | 콘솔 출력 |
| `System.err` | `PrintStream` | 에러 출력 |

대상을 바꾸려면 `System.setOut(new PrintStream(...))`처럼 재지정한다.

### 6.2 RandomAccessFile

자바에서 **하나의 클래스로 읽기·쓰기를 모두 지원**하는 유일한 스트림이다. 파일 포인터를 이동시켜 임의 위치에 접근할 수 있다.

```java
try (RandomAccessFile raf = new RandomAccessFile("data.dat", "rw")) {
    raf.writeInt(100);
    raf.writeUTF("Hello");
    raf.seek(0);                 // 파일 시작으로 이동
    int    n = raf.readInt();
    String s = raf.readUTF();
}
```

### 6.3 File 클래스

파일·디렉터리의 **메타 정보**를 다룬다. `File` 인스턴스를 만든다고 실제 파일이 생기지는 않는다.

```java
File f = new File("data/test.txt");
f.createNewFile();

File dir = new File("data/newDir");
dir.mkdirs();   // 중간 경로까지 생성

f.exists();        f.isFile();      f.isDirectory();
f.length();        f.lastModified();
```

| 메서드 | 설명 |
|:---|:---|
| `getName()` | 파일 이름 |
| `getPath()` | 경로 (그대로) |
| `getAbsolutePath()` | 절대 경로 (`.`·`..` 포함 가능) |
| `getCanonicalPath()` | 정규 경로 (기호·링크 해소) |
| `list()` / `listFiles()` | 디렉터리 내 목록 |

{{< callout type="info" >}}
**모던한 대안: `java.nio.file.Files`** — Java 7부터 도입된 NIO.2 API는 훨씬 간결하고 강력하다.

```java
Path src = Paths.get("a.txt");
Path dst = Paths.get("b.txt");

Files.copy(src, dst, StandardCopyOption.REPLACE_EXISTING);
Files.move(src, dst);
Files.delete(src);
Files.size(src);
Files.isDirectory(src);

// 전체 읽기/쓰기
String text = Files.readString(src, StandardCharsets.UTF_8);
Files.writeString(dst, text);

// 스트림으로 라인 처리
try (Stream<String> lines = Files.lines(src)) {
    lines.filter(l -> !l.isBlank()).forEach(System.out::println);
}

// 디렉터리 순회
try (Stream<Path> walk = Files.walk(Paths.get("."))) {
    walk.filter(Files::isRegularFile).forEach(System.out::println);
}
```

새 코드를 작성한다면 `File` 대신 `Path` + `Files`를 선택하라.
{{< /callout >}}

## 7. 직렬화 (Serialization)

### 7.1 개념

직렬화는 **객체의 상태(인스턴스 변수 값)를 바이트 스트림으로 변환**하는 것이고, 역직렬화는 그 반대다. 파일 저장이나 네트워크 전송을 위해 사용된다.

```
    직렬화                    역직렬화
객체 ────▶ 바이트 스트림 ────▶ 객체
      (파일·네트워크)
```

메서드와 `static` 변수는 직렬화 대상이 아니다.

### 7.2 ObjectOutputStream / ObjectInputStream

```java
// 직렬화
try (ObjectOutputStream oos = new ObjectOutputStream(
        new FileOutputStream("user.ser"))) {
    oos.writeObject(new UserInfo("홍길동", 25));
}

// 역직렬화
try (ObjectInputStream ois = new ObjectInputStream(
        new FileInputStream("user.ser"))) {
    UserInfo u = (UserInfo) ois.readObject();
}
```

### 7.3 Serializable

직렬화 가능한 클래스는 **마커 인터페이스 `Serializable`** 을 구현해야 한다.

```java
public class UserInfo implements Serializable {
    private static final long serialVersionUID = 1L;

    private String name;
    private int age;
    private transient String password;  // 직렬화 제외
}
```

| 규칙 | 설명 |
|:---|:---|
| Serializable 구현 | 필수 |
| 조상이 미구현 | 조상 인스턴스 변수는 직렬화되지 않음 |
| `transient` | 직렬화에서 제외 (복원 시 기본값) |
| `static` | 클래스 변수 → 직렬화 대상 아님 |

### 7.4 serialVersionUID

직렬화된 데이터와 클래스의 **버전 일치**를 확인하는 식별자다. 선언하지 않으면 컴파일러가 클래스 구조를 해시하여 자동 생성하지만, 이 값은 **필드 하나만 추가해도 바뀌어** 역직렬화 시 `InvalidClassException`이 발생한다.

```java
class MyData implements Serializable {
    private static final long serialVersionUID = 1L;
    int value;
    String name;
}
```

{{< callout type="warning" >}}
**`serialVersionUID`는 반드시 직접 선언하라.** 자동 생성에 의존하면 사소한 변경(필드 순서, 이름)에도 호환성이 깨진다. 또한 직렬화는 **보안 취약점**(임의 객체 생성, RCE)의 온상이 되기 쉬우므로, 신뢰할 수 없는 입력은 역직렬화하지 말고 JSON·Protobuf 같은 **데이터 포맷**을 사용하는 것이 현대적인 기본값이다.
{{< /callout >}}

## 8. 요약

### 스트림 체계

```
                 스트림
        ┌──────────┴──────────┐
     바이트 기반              문자 기반
   InputStream           Reader
   OutputStream          Writer
        │                   │
        └── 보조 스트림 ─────┘
  Buffered  Data  Object  Print
  InputStreamReader (바이트→문자)
```

### 핵심 개념

| 개념 | 설명 |
|:---|:---|
| 스트림 | 단방향 연속 데이터 통로 |
| 바이트 스트림 | 1 byte 단위, 바이너리 |
| 문자 스트림 | 2 byte(char), 인코딩 자동 변환 |
| 보조 스트림 | 데코레이터로 기능 추가 |
| try-with-resources | 자동 close, 자원 누수 방지 |
| `java.nio.file.Files` | 모던 파일 API |
| RandomAccessFile | 임의 위치 읽기/쓰기 |
| 직렬화 | 객체 ↔ 바이트, Serializable |
| `transient` | 직렬화 제외 |
| serialVersionUID | 직접 선언 필수 |

### 선택 가이드

```
바이너리 (이미지·동영상)
  └▶ FileInputStream + Buffered

텍스트 (txt·csv·log)
  └▶ Files.lines / Files.readString

대용량 텍스트 라인 처리
  └▶ BufferedReader + InputStreamReader

콘솔 입력
  └▶ Scanner 또는 BufferedReader

파일 복사/이동/속성
  └▶ java.nio.file.Files

임의 위치 접근
  └▶ RandomAccessFile

객체 저장·전송
  └▶ JSON/Protobuf 권장
     (Serializable은 내부 전용)
```
