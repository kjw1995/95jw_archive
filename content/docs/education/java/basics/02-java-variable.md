## 1. 변수(Variable)란?

**값을 저장할 수 있는 메모리상의 공간**이다. 하나의 변수에는 **하나의 값**만 저장할 수 있으며, 새 값을 넣으면 기존 값은 덮어써진다.

```java
int age;        // 변수 선언
age = 25;       // 값 저장 (초기화)
int score = 90; // 선언과 동시에 초기화
```

### 1.1 변수 초기화

- **초기화**: 변수를 사용하기 전에 처음으로 값을 저장하는 것
- JVM의 힙에 할당된 필드는 기본값으로 자동 초기화되지만, **지역 변수는 반드시 명시적으로 초기화**해야 한다
- 초기화하지 않은 지역 변수를 읽으면 컴파일 오류가 발생한다

{{< callout type="info" >}}
**필드 기본값**: 숫자형은 `0`, `boolean`은 `false`, 참조형은 `null`로 자동 초기화된다. 반면 메서드 안의 지역 변수는 기본값이 없고, 사용 전 초기화가 강제된다.
{{< /callout >}}

## 2. 변수 명명규칙

| 규칙 | 설명 | 예시 |
|:-----|:-----|:-----|
| 대소문자 구분 | 길이 제한 없음 | `True` ≠ `true` |
| 예약어 사용 불가 | 키워드 사용 불가 | `int`, `class` |
| 숫자로 시작 불가 | 첫 글자는 문자 | `top10` OK, `7up` NG |
| 특수문자 제한 | `_`와 `$`만 허용 | `$harp` OK, `S#arp` NG |

### 2.1 권장 명명법

| 대상 | 규칙 | 예시 |
|:-----|:-----|:-----|
| 클래스 | 첫 글자 대문자 (PascalCase) | `StringBuffer` |
| 변수/메서드 | 첫 글자 소문자 (camelCase) | `lastName` |
| 상수 | 모두 대문자, `_`로 구분 | `MAX_VALUE` |
| 패키지 | 모두 소문자 | `java.util` |

## 3. 자료형(Data Type)

### 3.1 기본형 vs 참조형

| 구분 | 설명 | 종류 |
|:-----|:-----|:-----|
| 기본형 (primitive) | 실제 값을 변수에 직접 저장 | 8개 |
| 참조형 (reference) | 객체가 있는 메모리 주소를 저장 | 기본형 제외 모든 타입 |

### 3.2 기본형 8가지

```
기본형 (Primitive Type)
┌────────┬────────┬────────┐
│ 논리형 │ 문자형 │ 수치형 │
├────────┼────────┼────────┤
│boolean │  char  │ byte   │
│  (1B)  │  (2B)  │ short  │
│        │        │ int    │
│        │        │ long   │
│        │        │ float  │
│        │        │ double │
└────────┴────────┴────────┘
```

### 3.3 기본형 크기와 범위

| 타입 | 크기 | 범위 |
|:-----|:-----|:-----|
| `boolean` | 1 byte | `true`, `false` |
| `char` | 2 byte | 0 ~ 65,535 (유니코드) |
| `byte` | 1 byte | -128 ~ 127 |
| `short` | 2 byte | -32,768 ~ 32,767 |
| `int` | 4 byte | 약 ±21억 |
| `long` | 8 byte | 약 ±922경 |
| `float` | 4 byte | 정밀도 약 7자리 |
| `double` | 8 byte | 정밀도 약 15자리 |

> **n비트 부호 있는 정수의 범위**: -2^(n-1) ~ 2^(n-1)-1

## 4. 상수와 리터럴

```java
final int MAX_SPEED = 100;  // 상수: 한 번 초기화 후 변경 불가
int year = 2024;            // 2024가 리터럴
```

| 용어 | 정의 |
|:-----|:-----|
| 변수 | 하나의 값을 저장하기 위한 공간 |
| 상수 | 값을 한 번만 저장할 수 있는 공간 (`final`) |
| 리터럴 | 그 자체로 값을 의미하는 표현 (`123`, `3.14`, `'A'`) |

### 4.1 리터럴 접미사

| 타입 | 접미사 | 예시 |
|:-----|:-------|:-----|
| `long` | `L` | `100L` |
| `float` | `f` | `3.14f` |
| `double` | `d` (생략 가능) | `3.14d` |

{{< callout type="warning" >}}
**정수 리터럴의 기본 타입은 `int`, 실수 리터럴의 기본 타입은 `double`이다.** 따라서 `long l = 10000000000;`는 컴파일 오류이며 `10000000000L`로 써야 한다. 마찬가지로 `float f = 3.14;`도 오류이며 `3.14f`가 필요하다.
{{< /callout >}}

### 4.2 진법 접두사

| 진법 | 접두사 | 예시 |
|:-----|:-------|:-----|
| 2진수 | `0b` | `0b0101` |
| 8진수 | `0` | `077` |
| 16진수 | `0x` | `0xFF` |

Java 7부터는 리터럴 중간에 `_`를 넣어 가독성을 높일 수 있다.

```java
int million = 1_000_000;
long card = 1234_5678_9012_3456L;
```

## 5. 형식화된 출력 - printf()

| 지시자 | 설명 | 예시 |
|:-------|:-----|:-----|
| `%d` | 10진 정수 | `123` |
| `%x`, `%X` | 16진 정수 | `7b`, `7B` |
| `%o` | 8진 정수 | `173` |
| `%f` | 부동소수점 | `3.140000` |
| `%e` | 지수 표현 | `3.14e+00` |
| `%c` | 문자 | `A` |
| `%s` | 문자열 | `Hello` |
| `%b` | boolean | `true` |
| `%n` | 플랫폼 줄바꿈 | — |

```java
System.out.printf("%-10s | %5d%n", "apple", 3);
System.out.printf("%.2f%n", 3.14159);   // 3.14
```

## 6. 진법

### 6.1 진법 변환

```
10진 → 2진: 2로 나누고 나머지를 역순으로
2진 → 10진: 각 자리수 × 2^n 을 합산

2진 → 8진:  3자리씩 묶기 (2^3 = 8)
2진 → 16진: 4자리씩 묶기 (2^4 = 16)
```

### 6.2 음수 표현 - 2의 보수법

컴퓨터는 음수를 저장할 때 **2의 보수(Two's Complement)** 를 사용한다. 덧셈기 하나로 뺄셈까지 처리할 수 있어 하드웨어 설계가 간단해진다.

```
양수 5 (4비트): 0101
   ↓ 1의 보수 (비트 반전)
               1010
   ↓ +1
음수 -5:       1011
```

> **MSB(Most Significant Bit)**: 최상위 비트. 0이면 양수, 1이면 음수이다.

## 7. 문자형 (char)

### 7.1 문자와 유니코드

자바의 `char`는 **2바이트 크기의 유니코드(UTF-16)** 를 저장한다. 따라서 문자도 결국 정수이다.

```java
char ch = 'A';         // 실제로는 65 저장
char ch2 = 65;         // 위와 동일
int code = (int) ch;   // 유니코드 값 확인: 65
```

### 7.2 특수 문자 (Escape)

| 이스케이프 | 의미 |
|:-----------|:-----|
| `\t` | 탭 |
| `\n` | 줄바꿈 |
| `\\` | 역슬래시 |
| `\'` | 작은따옴표 |
| `\"` | 큰따옴표 |
| `A` | 유니코드 직접 지정 (`'A'`) |

### 7.3 인코딩 방식

| 방식 | 특징 |
|:-----|:-----|
| ASCII | 7bit, 128개 문자 |
| UTF-16 | 2byte 고정, Java `char` 내부 표현 |
| UTF-8 | 1~4byte 가변, 웹 표준 |

{{< callout type="info" >}}
**자바 `String`은 내부적으로 UTF-16을 사용한다.** 다만 Java 9부터 Compact Strings 최적화로 ASCII 범위만 쓰는 문자열은 byte 배열로 저장되어 메모리를 절약한다.
{{< /callout >}}

## 8. 정수형 오버플로우

타입이 표현할 수 있는 범위를 넘으면 **최상위 비트가 순환**하면서 반대쪽 극단으로 돌아간다.

```
최대값 + 1 → 최소값
최소값 - 1 → 최대값
```

```java
byte b = 127;   // byte 최대값
b++;            // 오버플로우 발생
// b == -128    // 최소값으로 순환
```

{{< callout type="warning" >}}
**자바는 오버플로우를 예외로 던지지 않는다.** 단순히 값이 잘려 나가기 때문에 버그가 늦게 발견될 수 있다. 큰 값을 다룰 때는 `long` 또는 `BigInteger`를 쓰거나 `Math.addExact()`처럼 오버플로우를 감지하는 API를 사용한다.
{{< /callout >}}

> **정수형 선택 가이드**: 기본적으로 `int`를 사용한다(CPU가 가장 효율적으로 처리). 약 ±21억을 넘어가면 `long`을 사용한다.

## 9. 실수형

### 9.1 float vs double

| 항목 | float | double |
|:-----|:------|:-------|
| 크기 | 4 byte | 8 byte |
| 정밀도 | 약 7자리 | 약 15자리 |
| 기본 실수형 | — | O |

### 9.2 실수형 저장 구조 (IEEE 754)

```
float (32bit): S(1) E(8)  M(23)
double(64bit): S(1) E(11) M(52)

S: 부호 (0=양수, 1=음수)
E: 지수 (Exponent)
M: 가수 (Mantissa, 정밀도 결정)
```

값은 `(-1)^S × 1.M × 2^(E-bias)` 형태로 저장된다. 가수부(M)의 비트 수가 정밀도를 결정하기 때문에, `double`이 `float`보다 훨씬 많은 유효 숫자를 표현한다.

### 9.3 부동소수점의 한계

```java
double a = 0.1 + 0.2;
System.out.println(a);        // 0.30000000000000004
System.out.println(a == 0.3); // false
```

2진수로 정확히 표현할 수 없는 10진 소수가 존재하기 때문에 **반올림 오차**가 필연적으로 발생한다. 실수끼리의 `==` 비교는 피해야 한다.

{{< callout type="warning" >}}
**금액·세율처럼 정확한 10진 연산이 필요하면 `BigDecimal`을 사용하라.** `double`로 누적된 오차는 금융 계산에서 치명적이다. `BigDecimal` 생성 시에는 `new BigDecimal(0.1)`이 아니라 `new BigDecimal("0.1")` 또는 `BigDecimal.valueOf(0.1)`을 써야 오차가 개입하지 않는다.
{{< /callout >}}

### 9.4 오버플로우 vs 언더플로우

| 현상 | 결과 |
|:-----|:-----|
| 오버플로우 | `Infinity` (또는 `-Infinity`) |
| 언더플로우 | `0.0` (양의 최소값보다 작은 값) |
| `0.0 / 0.0` | `NaN` (Not a Number) |

## 10. 요약

| 항목 | 핵심 |
|:-----|:-----|
| 변수 | 값을 저장하는 메모리 공간, 지역 변수는 초기화 필수 |
| 기본형 | 8가지 (`boolean`, `char`, `byte`, `short`, `int`, `long`, `float`, `double`) |
| 참조형 | 객체 주소 저장, `null` 가능 |
| 상수 | `final` 키워드, 변경 불가 |
| 리터럴 | 값 자체, 접미사로 타입 구분 (`L`, `f`, `d`) |
| 정수형 | `int` 기본, 오버플로우 시 순환 |
| 실수형 | `double` 기본, `==` 비교 금지, 정확한 계산은 `BigDecimal` |
