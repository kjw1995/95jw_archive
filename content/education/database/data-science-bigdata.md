## 1. 데이터 과학

### 필요성

사물 인터넷(IoT)으로 세상의 모든 것이 연결되면서 다양한 형태의 데이터가 끊임없이 생산·수집된다. 인공지능 기술도 방대한 데이터를 통한 학습이 필수이며, 이러한 데이터에서 의미 있는 지식을 추출하는 기술이 데이터 과학이다.

### 개념

데이터 과학은 데이터를 수집·분석하여 숨겨진 지식을 발견하고 문제 해결에 활용하는 모든 과정과 기술이다.

### DIKW 계층 구조

```text
┌──────────────┐
│   지혜(W)    │ 통찰·창의
├──────────────┤
│   지식(K)    │ 규칙·패턴
├──────────────┤
│   정보(I)    │ 가공 데이터
├──────────────┤
│   데이터(D)  │ 수집 값
└──────────────┘
```

| 계층 | 의미 | 예시 |
|:---|:---|:---|
| 데이터 | 수집된 원시 값 | 매장별 일일 매출 숫자 |
| 정보 | 가공된 데이터 | "A매장이 B매장보다 매출 20% 높음" |
| 지식 | 패턴·규칙 발견 | "주말 오후 2~4시 구매율 최고" |
| 지혜 | 의사결정 활용 | "주말 오후에 프로모션 집중" |

### 데이터 과학자의 역량

드류 콘웨이(Drew Conway)의 벤다이어그램에 따르면 세 가지 역량이 겹치는 영역이 데이터 과학이다.

| 역량 | 설명 | 예시 |
|:---|:---|:---|
| 컴퓨터 과학 | 프로그래밍·알고리즘 | Python, R, SQL |
| 통계·수학 | 확률·회귀분석 | 가설 검정, 분포 분석 |
| 도메인 지식 | 분석 대상 이해 | 금융, 의료, 마케팅 |

최근에는 시각화 기술도 중요한 자질로 평가된다.

{{< callout type="info" >}}
데이터 과학의 대상이 반드시 빅데이터일 필요는 없다. 일반 관계 데이터베이스 데이터도 분석 대상이 된다.
{{< /callout >}}

## 2. 빅데이터

### 개념

빅데이터는 기존 데이터베이스의 저장·관리 범위를 넘어서는 대규모의 다양한 데이터와 이를 처리하는 기술을 가리킨다. 좁게는 데이터 자체를 뜻하고, 넓게는 저장·관리·분석 기술까지 포함한다.

### 특징 (3V → 7V)

기본 3V는 Volume·Velocity·Variety이며, 여기에 Value·Veracity·Visualization·Variability가 더해져 7V로 확장된다.

| 특징 | 의미 |
|:---|:---|
| Volume | TB 단위 이상의 대량 데이터 |
| Velocity | 실시간 수집·분석 처리 |
| Variety | 정형 + 반정형 + 비정형 |
| Value | 의사결정에 활용할 가치 |
| Veracity | 정확하고 신뢰할 수 있는 데이터 |
| Visualization | 이해하기 쉬운 시각적 표현 |
| Variability | 맥락에 따른 의미 변화 |

### 데이터 형태 분류

| 형태 | 설명 | 예시 |
|:---|:---|:---|
| 정형 | 고정된 스키마 | RDB 테이블, CSV |
| 반정형 | 내부적 구조 포함 | XML, HTML, JSON |
| 비정형 | 정해진 구조 없음 | 텍스트, 이미지, 영상 |

## 3. 배치 vs 스트리밍 처리

데이터 처리 방식은 크게 두 가지로 나뉜다.

| 구분 | 배치 처리 | 스트리밍 처리 |
|:---|:---|:---|
| 처리 단위 | 대용량 묶음 | 레코드 단위·윈도우 |
| 지연 시간 | 분~시간 | 초·밀리초 |
| 데이터 경계 | 유한 | 무한 연속 |
| 대표 기술 | Hadoop MapReduce, Spark | Kafka Streams, Flink |
| 적합 사례 | 일·월 단위 집계, ETL | 실시간 알림, 모니터링 |

```text
[배치]
데이터 ── 수집 ──► 저장 ──► 일괄 처리
                         (새벽 ETL)

[스트리밍]
이벤트 ─► 큐 ─► 실시간 처리 ─► 결과
        (연속)    (윈도우)
```

{{< callout type="info" >}}
람다(Lambda) 아키텍처는 배치와 스트리밍을 병행해 정확성과 지연 시간을 동시에 확보한다. 카파(Kappa) 아키텍처는 스트리밍만으로 통일한 단순한 구조다.
{{< /callout >}}

## 4. Hadoop과 MapReduce

### Hadoop 구성

Hadoop은 자바 기반 오픈소스 분산 처리 프레임워크다.

| 구성 요소 | 역할 |
|:---|:---|
| HDFS | 여러 서버에 데이터를 분산 저장 |
| MapReduce | 대규모 데이터를 병렬로 처리 |
| YARN | 자원 스케줄링 |

```text
┌──────────────────┐
│    MapReduce     │
│  (분산 처리)      │
├──────────────────┤
│      HDFS        │
│  (분산 파일)      │
├──┬──┬──┬──┬──┬──┤
│N1│N2│N3│N4│N5│N6│
└──┴──┴──┴──┴──┴──┘
 분산된 데이터 노드
```

### MapReduce 동작 원리

대규모 데이터를 Map·Shuffle·Reduce 세 단계로 병렬 처리한다.

```text
[입력]
"hello world hello"
       │
       ▼
┌─────────────┐
│    Map      │ 분할·키값 생성
│ (h,1)(w,1)  │
│ (h,1)       │
└──────┬──────┘
       ▼
┌─────────────┐
│  Shuffle    │ 같은 키끼리
│ h → [1,1]   │ 그룹화
│ w → [1]     │
└──────┬──────┘
       ▼
┌─────────────┐
│   Reduce    │ 합산
│ (h,2)(w,1)  │
└─────────────┘
```

| 단계 | 입력 | 출력 |
|:---|:---|:---|
| Map | 원시 레코드 | (key, value) 쌍 |
| Shuffle | Map 출력 | key로 그룹화된 리스트 |
| Reduce | (key, list) | 집계된 결과 |

### 분석·표현 기술

| 기술 | 설명 |
|:---|:---|
| 텍스트 마이닝 | 비정형 텍스트에서 자연어 처리로 정보 추출 |
| 오피니언 마이닝 | SNS·블로그에서 긍정/부정/중립 분석 |
| 소셜 네트워크 분석 | 연결 구조로 영향력·행동 패턴 추출 |
| 군집 분석 | 유사 특성 데이터를 그룹으로 묶음 |
| R | 통계 분석·시각화용 오픈소스 언어 |

### 빅데이터 이전 vs 빅데이터

| 구분 | 빅데이터 이전 | 빅데이터 |
|:---|:---|:---|
| 데이터 유형 | 정형 중심 | 정형 + 반정형 + 비정형 |
| 기술 | RDB, 통계 패키지 | Hadoop, NoSQL, R |
| 저장 장치 | 고가 DB/DW | 저렴한 클라우드 |

## 5. Data Lake vs Data Warehouse

Data Lake와 Data Warehouse는 대용량 데이터를 다루는 두 가지 대표 아키텍처다.

| 구분 | Data Warehouse (DW) | Data Lake |
|:---|:---|:---|
| 데이터 | 정형 중심 | 정형 + 반정형 + 비정형 |
| 스키마 | Schema-on-Write | Schema-on-Read |
| 처리 시점 | 적재 시 가공 (ETL) | 사용 시 가공 (ELT) |
| 사용자 | 경영진·분석가 | 데이터 과학자·엔지니어 |
| 저장 비용 | 높음 | 낮음 (오브젝트 스토리지) |
| 대표 기술 | Redshift, Snowflake, BigQuery | S3, HDFS, Azure Data Lake |

```text
[DW]                  [Data Lake]
RDB ── ETL ──► DW     원본 ──► Lake
               │                 │
               ▼                 ▼
            정형 분석         탐색·ML
```

{{< callout type="info" >}}
최근에는 두 장점을 결합한 Lakehouse(Delta Lake, Apache Iceberg 등)가 부상하고 있다. 원본 보존과 정형 분석을 한 플랫폼에서 지원한다.
{{< /callout >}}

## 6. NoSQL

### 등장 배경

NoSQL은 Not Only SQL의 약자로, 관계 데이터베이스만 고집하지 말고 필요에 따라 다른 특성을 제공하는 데이터베이스를 사용하자는 의미다. 1998년에 처음 언급되었으며, 현재의 의미는 2009년 요한 오스칼손(Johan Oskarsson) 이후로 굳어졌다.

### 특징

- ACID 트랜잭션 대신 분산 처리에 초점
- 스키마 없이 동작해 구조 변경이 자유로움
- 비정형 데이터 저장에 적합
- 대부분 오픈 소스로 제공

### 관계 DB vs NoSQL

| 구분 | 관계 DB | NoSQL |
|:---|:---|:---|
| 데이터 유형 | 정형 | 정형 + 비정형 |
| 대용량 처리 | 성능 저하 | 대용량 지원 |
| 스키마 | 미리 정의 | 없거나 자유 |
| 트랜잭션 | ACID 보장 | 미지원 (일부 예외) |
| 검색 | 조인 등 복잡 | 단순 검색 |
| 확장성 | 클러스터 부적합 | 클러스터 적합 |
| 비용 | 고가 라이선스 | 오픈 소스 |

### CAP 정리

분산 시스템에서 세 가지 속성을 동시에 만족할 수 없다는 이론이다.

| 속성 | 의미 |
|:---|:---|
| Consistency | 모든 노드가 같은 데이터를 봄 |
| Availability | 항상 응답을 받을 수 있음 |
| Partition Tolerance | 네트워크 장애에도 동작 |

```text
   Consistency
       │
   ┌───┴───┐
   │  CA   │
   ├───┬───┤
   │CP │AP │
   └───┴───┘
   P         A
```

| 조합 | 특성 | 대표 DB |
|:---|:---|:---|
| CP | 일관성 + 분할 허용 | MongoDB, HBase |
| AP | 가용성 + 분할 허용 | Cassandra, DynamoDB |
| CA | 일관성 + 가용성 | 전통 관계 DB |

{{< callout type="warning" >}}
분산 환경에서는 P(분할 허용)를 포기할 수 없으므로 실제로는 CP 또는 AP 중에서 선택하게 된다.
{{< /callout >}}

### NoSQL 종류

**키-값 (Key-Value)**

가장 단순한 형태로, 키와 값의 쌍으로 데이터를 저장한다.

```text
┌─────────┬──────────┐
│  Key    │  Value   │
├─────────┼──────────┤
│user:101 │{name:..} │
│user:102 │{name:..} │
│session  │abc123..  │
│img:001  │<binary>  │
└─────────┴──────────┘
```

| 특징 | 설명 |
|:---|:---|
| 장점 | 어떤 형태의 값이든 저장, 빠른 질의 |
| 단점 | 값의 일부 검색·내용 기반 질의 불가 |
| 대표 제품 | DynamoDB, Redis |

**문서 기반 (Document)**

키-값 모델의 확장으로, JSON·XML 같은 반정형 문서로 데이터를 저장한다.

```json
{
  "_id": "user_101",
  "name": "홍길동",
  "age": 30,
  "orders": [
    {"item": "책", "qty": 2},
    {"item": "노트북", "qty": 1}
  ]
}
```

| 특징 | 설명 |
|:---|:---|
| 장점 | 문서 내 일부 검색, 계층 구조 지원 |
| 단점 | 문서 간 조인이 어려움 |
| 대표 제품 | MongoDB, CouchDB |

**컬럼 기반 (Column-Family)**

컬럼 패밀리와 키의 쌍으로 저장한다. 관계 모델의 테이블과 비슷하나 행마다 컬럼 구성이 다를 수 있다.

```text
Row Key: user_101
┌──────────────────┐
│ 기본정보          │
│  name: 홍길동    │
│  age: 30         │
├──────────────────┤
│ 주문정보          │
│  order1: 책      │
│  order2: 노트북  │
└──────────────────┘
```

| 특징 | 설명 |
|:---|:---|
| 장점 | 대규모 분산 저장, 유연한 컬럼 |
| 단점 | 복잡한 질의 처리가 어려움 |
| 대표 제품 | BigTable, HBase, Cassandra |

**그래프 기반 (Graph)**

노드에 데이터를 저장하고 간선으로 관계를 표현한다. 다른 NoSQL과 달리 ACID를 지원한다.

```text
(홍길동)──친구──►(김철수)
    │               │
  구매             구매
    ▼               ▼
 (노트북)        (태블릿)
```

| 특징 | 설명 |
|:---|:---|
| 장점 | 관계 표현에 적합, ACID 지원 |
| 단점 | 클러스터 환경에 부적합 |
| 대표 제품 | Neo4J, OrientDB, AgensGraph |

### NoSQL 종류 비교

| 유형 | 데이터 구조 | 적합 분야 |
|:---|:---|:---|
| 키-값 | Key → Value | 세션, 캐시, 장바구니 |
| 문서 기반 | Key → Document | 콘텐츠 관리, 카탈로그 |
| 컬럼 기반 | Key → Column Family | 로그 분석, 시계열 |
| 그래프 | Node + Edge | 소셜 네트워크, 추천 |

## 7. 데이터 마이닝

### 개념

데이터 분석은 쌓인 데이터에서 유용한 지식을 찾기 위해 가공하는 역할이다. SQL로 원하는 데이터를 추출해 분석하는 것도 포함된다.

### 기계 학습 vs 데이터 마이닝

두 기술은 중복이 많지만 초점이 다르다.

| 구분 | 데이터 마이닝 | 기계 학습 |
|:---|:---|:---|
| 초점 | 발견(Discovery) | 예측(Prediction) |
| 목적 | 숨겨진 규칙·패턴 발견 | 새 데이터 결과 예측 |
| 관점 | 통계학 | 컴퓨터 과학 |

### 기계 학습의 분류

| 유형 | 방식 | 예시 |
|:---|:---|:---|
| 지도 학습 | 정답 있는 데이터로 학습 | 스팸 분류, 가격 예측 |
| 비지도 학습 | 정답 없이 패턴 발견 | 고객 군집화, 이상 탐지 |
| 강화 학습 | 보상을 통해 학습 | 게임 AI, 자율 주행 |

### 분석 기법

**분류 분석 (Classification)**

미리 정의된 기준에 따라 새 데이터가 어떤 그룹에 속하는지 판단한다.

```text
[학습]
성적 90+ → A
성적 80+ → B
성적 70+ → C

[예측]
성적 85 → B
```

군집 분석과 달리 미리 정의된 그룹이 존재한다. 대표 알고리즘으로 의사결정 트리, k-NN, SVM, 나이브 베이즈가 있다.

**군집 분석 (Clustering)**

미리 정해진 기준 없이 유사한 특성의 데이터를 독립적인 군집으로 나눈다.

| 유형 | 방법 | 세부 기법 |
|:---|:---|:---|
| 계층적 | 순서대로 묶음 | 최단/최장/평균/와드 |
| 비계층적 | 한번에 군집 형성 | K-Means |

K-Means 동작.

```text
1) 초기 중심점 지정
   ★A        ★B

2) 가까운 중심에 할당
   ★A ● ●    ★B ○ ○
      ●         ○

3) 중심점 재계산
   ★A'        ★B'

4) 수렴까지 반복
```

**연관 분석 (Association)**

데이터 간 발생 빈도를 분석해 연관 규칙을 찾는다. 장바구니 분석이라고도 한다.

| 지표 | 의미 | 공식 |
|:---|:---|:---|
| 지지도 | X와 Y가 함께 발생할 확률 | P(X∩Y) |
| 신뢰도 | X 발생 시 Y도 발생할 확률 | P(X∩Y)/P(X) |
| 향상도 | 연관 관계의 강도 | 신뢰도/P(Y) |

```text
[거래]
T1: 맥주, 기저귀
T2: 맥주, 빵, 기저귀
T3: 빵, 우유
T4: 맥주, 기저귀, 우유

[규칙] 맥주 → 기저귀
 지지도: 3/4 = 75%
 신뢰도: 3/3 = 100%
```

Apriori 알고리즘은 최소 지지도를 충족하는 빈발 항목 집합을 단계적으로 찾아 연관 규칙을 생성한다.

## 8. 데이터 시각화

### 개념

데이터 분석 결과를 이해하기 쉽게 그림이나 그래프로 표현하는 기술이다. 단순한 결과 표현을 넘어, 이전에 보이지 않던 규칙이나 패턴을 발견하는 분석 과정의 일부이기도 하다.

| 구분 | 인포그래픽 | 데이터 시각화 |
|:---|:---|:---|
| 초점 | 정보의 직관적 전달 | 분석 활용 |
| 목적 | 명확한 커뮤니케이션 | 패턴 발견 지원 |

### 시각화 방법

| 방법 | 목적 | 활용 차트 |
|:---|:---|:---|
| 시간 시각화 | 시간 흐름 변화 | 막대, 꺾은선 |
| 분포 시각화 | 분포 상태 | 트리맵, 원 |
| 관계 시각화 | 상관관계 | 버블, 산점도 |
| 비교 시각화 | 유사성·차이 | 방사형, 히트맵 |
| 공간 시각화 | 지역 분포 | 지도 매핑 |

### 데이터 성격별 추천

| 데이터 성격 | 권장 시각화 | 예시 |
|:---|:---|:---|
| 추세·변화 | 시간 시각화 | 월별 매출 추이 |
| 비율·구성 | 분포 시각화 | 시장 점유율 |
| 상관 관계 | 관계 시각화 | 광고비와 매출 |
| 순위·크기 | 비교 시각화 | 부서별 성과 |
| 위치·지역 | 공간 시각화 | 지역별 인구 밀도 |

### 시각화 도구

| 도구 | 특징 |
|:---|:---|
| R | 통계 분석 + 시각화, 오픈소스 |
| Tableau | 드래그&드롭 기반 빠른 시각화 |
| Python(Matplotlib) | 프로그래밍 기반 유연한 시각화 |
| D3.js | 웹 기반 인터랙티브 시각화 |
| Gephi | 네트워크·그래프 시각화 |

R은 1993년 뉴질랜드 오클랜드 대학교에서 개발된 프로그래밍 언어이자 데이터 분석 소프트웨어다. 대화형 방식으로 운영되며 R 스튜디오를 통해 GUI 환경에서 작업할 수 있다.

{{< callout type="info" >}}
**용어 정리**
- **데이터 과학**: 데이터에서 지식을 추출하는 학제 융합 분야
- **DIKW**: Data, Information, Knowledge, Wisdom 계층
- **빅데이터**: 기존 DB가 다루기 어려운 규모·속도·다양성의 데이터
- **3V**: Volume, Velocity, Variety
- **Hadoop**: HDFS·MapReduce 기반 분산 처리 프레임워크
- **MapReduce**: Map → Shuffle → Reduce 단계의 병렬 처리 모델
- **배치 처리**: 유한 데이터를 한 번에 처리
- **스트리밍 처리**: 무한 이벤트를 실시간 처리
- **Data Lake**: 원본을 저렴하게 저장하는 Schema-on-Read 저장소
- **Data Warehouse**: ETL로 정제된 정형 분석용 저장소
- **NoSQL**: Not Only SQL, 비관계형 DB 계열
- **CAP**: Consistency, Availability, Partition Tolerance
- **데이터 마이닝**: 숨겨진 규칙·패턴을 찾는 기술
- **기계 학습**: 데이터로 모델을 학습해 예측하는 기술
- **분류 / 군집 / 연관**: 지도·비지도·규칙 기반의 대표 분석 기법
- **Apriori**: 최소 지지도 기반 빈발 항목 집합 탐색 알고리즘
