## 1. 관계 데이터 연산 개요

관계 데이터 연산은 원하는 데이터를 얻기 위해 릴레이션에 필요한 처리 요구를 수행하는 방식이다. 크게 관계 대수와 관계 해석으로 나뉜다.

| 구분 | 관계 대수 | 관계 해석 |
|:---|:---|:---|
| 언어 유형 | 절차적 (Procedural) | 비절차적 (Non-procedural) |
| 기술 방식 | 처리 과정을 순서대로 기술 | 원하는 데이터만 기술 |
| 관점 | HOW (어떻게) | WHAT (무엇을) |
| 표현력 | 동등 | 동등 |

{{< callout type="info" >}}
**관계적으로 완전 (Relationally Complete)**: 관계 대수나 관계 해석으로 기술 가능한 모든 질의를 표현할 수 있는 언어. SQL은 이 기준을 만족한다.
{{< /callout >}}

### 폐쇄 특성 (Closure Property)

관계 대수에서 피연산자도 릴레이션이고 연산 결과도 릴레이션이다. 따라서 결과에 연산을 연쇄 적용할 수 있다.

```text
릴레이션 ─→ [연산자] ─→ 릴레이션 ─→ [연산자] ─→ 릴레이션
             ▲                        ▲
          연산 가능               연쇄 적용 가능
```

## 2. 연산자 분류

관계 대수 연산자는 **일반 집합 연산자**와 **순수 관계 연산자**로 구분된다.

```text
 관계 대수
 ├─ 일반 집합 연산자 (수학 집합에서 유래)
 │   ├─ 합집합 (∪)
 │   ├─ 교집합 (∩)
 │   ├─ 차집합 (−)
 │   └─ 카티션 프로덕트 (×)
 │
 └─ 순수 관계 연산자 (관계 모델 전용)
     ├─ 셀렉트 (σ)
     ├─ 프로젝트 (π)
     ├─ 조인 (⋈)
     └─ 디비전 (÷)
```

| 구분 | 연산자 | 기호 | 특징 |
|:---|:---|:---|:---|
| 일반 집합 | 합집합 | ∪ | 합병 가능 필요 |
| 일반 집합 | 교집합 | ∩ | 합병 가능 필요 |
| 일반 집합 | 차집합 | − | 합병 가능 필요, 교환 불가 |
| 일반 집합 | 카티션 프로덕트 | × | 모든 조합 |
| 순수 관계 | 셀렉트 | σ | 행 선택 (수평) |
| 순수 관계 | 프로젝트 | π | 열 선택 (수직) |
| 순수 관계 | 조인 | ⋈ | 공통 속성으로 연결 |
| 순수 관계 | 디비전 | ÷ | 모든 값과 매칭되는 투플 |

## 3. 예제 릴레이션

이후 예제에서 사용할 릴레이션이다.

```text
학생 (R)                    수강 (S)
┌─────┬──────┬────┐        ┌─────┬─────┐
│학번 │이름  │학과│        │학번 │과목 │
├─────┼──────┼────┤        ├─────┼─────┤
│001  │홍길동│CS  │        │001  │DB   │
│002  │김영희│BA  │        │001  │OS   │
│003  │이철수│CS  │        │002  │DB   │
│004  │박민수│EE  │        │003  │NET  │
└─────┴──────┴────┘        └─────┴─────┘
```

## 4. 일반 집합 연산자

### 합병 가능 조건

합집합·교집합·차집합은 다음 조건을 만족해야 한다.

1. 두 릴레이션의 차수(속성 개수)가 같아야 한다.
2. 대응되는 속성의 도메인이 같아야 한다(이름은 달라도 됨).

### 4.1 합집합 (Union) — R ∪ S

두 릴레이션에 속하는 모든 투플(중복 제거).

```text
CS학과             BA학과
┌─────┬──────┐     ┌─────┬──────┐
│001  │홍길동│     │002  │김영희│
│003  │이철수│     └─────┴──────┘
└─────┴──────┘

CS ∪ BA
┌─────┬──────┐
│001  │홍길동│
│002  │김영희│
│003  │이철수│
└─────┴──────┘
```

```sql
SELECT 학번, 이름 FROM CS학과
UNION
SELECT 학번, 이름 FROM BA학과;
```

### 4.2 교집합 (Intersection) — R ∩ S

두 릴레이션에 공통으로 속하는 투플.

```text
DB수강              OS수강
┌─────┬──────┐     ┌─────┬──────┐
│001  │홍길동│     │001  │홍길동│
│002  │김영희│     │003  │이철수│
└─────┴──────┘     └─────┴──────┘

DB ∩ OS
┌─────┬──────┐
│001  │홍길동│
└─────┴──────┘
```

```sql
SELECT 학번, 이름 FROM DB수강
INTERSECT
SELECT 학번, 이름 FROM OS수강;
```

### 4.3 차집합 (Difference) — R − S

R에는 있지만 S에는 없는 투플.

```text
전체학생            장학생
┌─────┬──────┐     ┌─────┬──────┐
│001  │홍길동│     │001  │홍길동│
│002  │김영희│     │003  │이철수│
│003  │이철수│     └─────┴──────┘
│004  │박민수│
└─────┴──────┘

전체 − 장학
┌─────┬──────┐
│002  │김영희│
│004  │박민수│
└─────┴──────┘
```

```sql
SELECT 학번, 이름 FROM 전체학생
EXCEPT  -- Oracle: MINUS
SELECT 학번, 이름 FROM 장학생;
```

{{< callout type="warning" >}}
차집합은 **교환적 특성이 없다**: R − S ≠ S − R. 순서에 따라 결과가 달라지니 주의한다.
{{< /callout >}}

### 4.4 카티션 프로덕트 (Cartesian Product) — R × S

R의 각 투플과 S의 각 투플을 모두 연결한 모든 조합.

```text
R                     S
┌─────┬──────┐        ┌──────┐
│001  │홍길동│        │DB    │
│002  │김영희│        │OS    │
└─────┴──────┘        └──────┘

R × S
┌─────┬──────┬──────┐
│001  │홍길동│DB    │
│001  │홍길동│OS    │
│002  │김영희│DB    │
│002  │김영희│OS    │
└─────┴──────┴──────┘

 차수 = R차수 + S차수
 카디널리티 = |R| × |S|
```

```sql
SELECT * FROM 학생, 과목;
-- 또는
SELECT * FROM 학생 CROSS JOIN 과목;
```

### 일반 집합 연산자 요약

| 연산자 | 기호 | 합병 가능 | 교환 | 결합 | 결과 차수 | 결과 카디널리티 |
|:---|:---|:---|:---|:---|:---|:---|
| 합집합 | ∪ | 필요 | O | O | 동일 | ≤ R + S |
| 교집합 | ∩ | 필요 | O | O | 동일 | ≤ min(R,S) |
| 차집합 | − | 필요 | X | X | 동일 | ≤ R |
| 카티션 프로덕트 | × | 불필요 | O | O | R + S | R × S |

## 5. 순수 관계 연산자

### 5.1 셀렉트 (Select) — σ

조건을 만족하는 투플(행)을 선택한다. 수평적 부분집합을 얻는다.

```text
 σ(조건식)(릴레이션)
```

```text
학생
┌─────┬──────┬────┬────┐
│001  │홍길동│CS  │ 3  │
│002  │김영희│BA  │ 2  │
│003  │이철수│CS  │ 1  │
│004  │박민수│EE  │ 3  │
└─────┴──────┴────┴────┘

σ(학과='CS')(학생)
┌─────┬──────┬────┬────┐
│001  │홍길동│CS  │ 3  │
│003  │이철수│CS  │ 1  │
└─────┴──────┴────┴────┘

σ(학과='CS' ∧ 학년≥2)(학생)
┌─────┬──────┬────┬────┐
│001  │홍길동│CS  │ 3  │
└─────┴──────┴────┴────┘
```

조건식 연산자: 비교(`>`, `≥`, `<`, `≤`, `=`, `≠`), 논리(`∧`, `∨`, `¬`).

```sql
SELECT * FROM 학생 WHERE 학과 = 'CS';
SELECT * FROM 학생 WHERE 학과 = 'CS' AND 학년 >= 2;
```

### 5.2 프로젝트 (Project) — π

선택한 속성(열)만 추출한다. 수직적 부분집합을 얻는다.

```text
 π(속성리스트)(릴레이션)
```

```text
π(이름, 학과)(학생)
┌──────┬────┐
│홍길동│CS  │
│김영희│BA  │
│이철수│CS  │
│박민수│EE  │
└──────┴────┘

π(학과)(학생)
┌────┐
│CS  │   ← 중복 자동 제거
│BA  │
│EE  │
└────┘
```

```sql
SELECT 이름, 학과 FROM 학생;
SELECT DISTINCT 학과 FROM 학생;
```

### 셀렉트 + 프로젝트 조합

```text
π(이름)( σ(학과='CS')(학생) )

1단계: σ(학과='CS')(학생) → CS 학생만
2단계: π(이름)(결과)      → 이름만

결과
┌──────┐
│홍길동│
│이철수│
└──────┘
```

```sql
SELECT 이름 FROM 학생 WHERE 학과 = 'CS';
```

### 5.3 조인 (Join) — ⋈

공통 속성을 기준으로 두 릴레이션의 투플을 연결한다.

#### 세타 조인 (Theta Join)

비교 연산자(θ)로 연결.

```text
 R ⋈(A θ B) S    θ ∈ { =, ≠, <, ≤, >, ≥ }
```

#### 동등 조인 (Equi Join)

θ가 `=`인 경우(가장 일반적).

```text
학생 ⋈(학생.학번 = 수강.학번) 수강

┌─────┬──────┬────┬─────┬─────┐
│학번 │이름  │학과│학번 │과목 │
├─────┼──────┼────┼─────┼─────┤
│001  │홍길동│CS  │001  │DB   │
│001  │홍길동│CS  │001  │OS   │
│002  │김영희│BA  │002  │DB   │
│003  │이철수│CS  │003  │NET  │
└─────┴──────┴────┴─────┴─────┘
      학번 컬럼이 중복됨 ↑   ↑
```

#### 자연 조인 (Natural Join) — ⋈ₙ

동등 조인 + 중복 속성 제거.

```text
학생 ⋈ₙ 수강

┌─────┬──────┬────┬─────┐
│학번 │이름  │학과│과목 │
├─────┼──────┼────┼─────┤
│001  │홍길동│CS  │DB   │
│001  │홍길동│CS  │OS   │
│002  │김영희│BA  │DB   │
│003  │이철수│CS  │NET  │
└─────┴──────┴────┴─────┘

 주의: 004 박민수는 수강 기록이 없어 제외됨
```

```sql
-- 동등 조인
SELECT * FROM 학생, 수강 WHERE 학생.학번 = 수강.학번;
SELECT * FROM 학생 INNER JOIN 수강 ON 학생.학번 = 수강.학번;

-- 자연 조인
SELECT * FROM 학생 NATURAL JOIN 수강;
```

#### 조인 분류

```text
 세타 조인 (θ-Join)
   └─ 동등 조인 (θ = '=')
        └─ 자연 조인 (중복 속성 제거)

 외부 조인 (Outer Join)
   ├─ 왼쪽 외부 (⟕)
   ├─ 오른쪽 외부 (⟖)
   └─ 완전 외부 (⟗)

 세미 조인 (Semi Join, ⋉)
```

### 5.4 디비전 (Division) — ÷

S의 모든 투플과 관련 있는 R의 투플을 반환한다.

```text
수강 (R)              필수과목 (S)
┌─────┬─────┐        ┌─────┐
│001  │DB   │        │DB   │
│001  │OS   │        │OS   │
│001  │NET  │        └─────┘
│002  │DB   │
│002  │OS   │
│003  │DB   │
└─────┴─────┘

수강 ÷ 필수과목
"DB와 OS를 모두 수강한 학생"

┌─────┐
│001  │ ← DB, OS 모두 수강
│002  │ ← DB, OS 모두 수강
└─────┘
      003은 DB만 → 제외
```

SQL에는 DIVISION이 직접 없으므로 다음과 같이 구현한다.

```sql
SELECT 학번 FROM 수강
WHERE 과목 IN (SELECT 과목 FROM 필수과목)
GROUP BY 학번
HAVING COUNT(DISTINCT 과목) = (SELECT COUNT(*) FROM 필수과목);
```

## 6. 확장된 관계 대수 연산자

### 6.1 세미 조인 (Semi Join) — ⋉

조인 속성으로 프로젝트한 후 조인한다.

```text
 R ⋉ S = R ⋈ ( π(조인속성)(S) )
```

```text
학생 ⋉ 수강 = 수강 기록이 있는 학생

┌─────┬──────┬────┐
│001  │홍길동│CS  │
│002  │김영희│BA  │
└─────┴──────┴────┘

 주의: R ⋉ S ≠ S ⋉ R
```

```sql
SELECT * FROM 학생
 WHERE 학번 IN (SELECT 학번 FROM 수강);

-- 또는
SELECT * FROM 학생 s
 WHERE EXISTS (SELECT 1 FROM 수강 WHERE 학번 = s.학번);
```

### 6.2 외부 조인 (Outer Join)

조인에서 제외된 투플을 NULL로 채워서 포함한다.

```text
학생                수강
┌─────┬──────┐     ┌─────┬─────┐
│001  │홍길동│     │001  │DB   │
│002  │김영희│     │002  │OS   │
│003  │이철수│     │005  │NET  │ ← 학생에 없음
└─────┴──────┘     └─────┴─────┘
```

#### 왼쪽 외부 조인 (⟕)

```text
학생 ⟕ 수강
┌─────┬──────┬─────┬─────┐
│001  │홍길동│001  │DB   │
│002  │김영희│002  │OS   │
│003  │이철수│NULL │NULL │ ← 왼쪽 전체 포함
└─────┴──────┴─────┴─────┘
```

#### 오른쪽 외부 조인 (⟖)

```text
학생 ⟖ 수강
┌─────┬──────┬─────┬─────┐
│001  │홍길동│001  │DB   │
│002  │김영희│002  │OS   │
│NULL │NULL  │005  │NET  │ ← 오른쪽 전체 포함
└─────┴──────┴─────┴─────┘
```

#### 완전 외부 조인 (⟗)

```text
학생 ⟗ 수강
┌─────┬──────┬─────┬─────┐
│001  │홍길동│001  │DB   │
│002  │김영희│002  │OS   │
│003  │이철수│NULL │NULL │
│NULL │NULL  │005  │NET  │
└─────┴──────┴─────┴─────┘
```

```sql
SELECT * FROM 학생 LEFT OUTER JOIN 수강
  ON 학생.학번 = 수강.학번;

SELECT * FROM 학생 RIGHT OUTER JOIN 수강
  ON 학생.학번 = 수강.학번;

SELECT * FROM 학생 FULL OUTER JOIN 수강
  ON 학생.학번 = 수강.학번;
```

## 7. 관계 대수 ↔ SQL 대응표

| 관계 대수 | 기호 | SQL |
|:---|:---|:---|
| 셀렉트 | σ | `WHERE` |
| 프로젝트 | π | `SELECT` (컬럼 지정) |
| 합집합 | ∪ | `UNION` |
| 교집합 | ∩ | `INTERSECT` |
| 차집합 | − | `EXCEPT` / `MINUS` |
| 카티션 프로덕트 | × | `CROSS JOIN` 또는 `,` |
| 자연 조인 | ⋈ₙ | `NATURAL JOIN` |
| 동등 조인 | ⋈ | `INNER JOIN ... ON` |
| 세타 조인 | ⋈(θ) | `INNER JOIN ... ON A θ B` |
| 세미 조인 | ⋉ | `WHERE ... IN (SELECT ...)` |
| 왼쪽 외부 조인 | ⟕ | `LEFT OUTER JOIN` |
| 오른쪽 외부 조인 | ⟖ | `RIGHT OUTER JOIN` |
| 완전 외부 조인 | ⟗ | `FULL OUTER JOIN` |
| 디비전 | ÷ | `GROUP BY + HAVING COUNT` |

## 8. 집합 연산 vs 순수 관계 연산 구분

| 구분 | 일반 집합 연산 | 순수 관계 연산 |
|:---|:---|:---|
| 기원 | 수학의 집합론 | 관계 모델 전용 |
| 대상 | 투플 단위 집합 연산 | 속성·투플 선택, 연결 |
| 예시 | ∪, ∩, −, × | σ, π, ⋈, ÷ |
| SQL | UNION, INTERSECT, EXCEPT, CROSS JOIN | WHERE, SELECT, JOIN, (복합 구문) |
| 합병 가능성 요구 | ∪, ∩, − 필요 | 불필요 |

## 핵심 정리

| 개념 | 설명 |
|:---|:---|
| 관계 대수 | 절차적 연산 언어 |
| 관계 해석 | 비절차적 연산 언어 |
| 폐쇄 특성 | 연산 결과도 릴레이션 |
| 합집합 ∪ | R 또는 S에 속하는 투플 |
| 교집합 ∩ | R과 S 공통 투플 |
| 차집합 − | R에만 있는 투플 (비교환) |
| 카티션 × | 모든 투플 조합 |
| 셀렉트 σ | 조건을 만족하는 투플 (수평) |
| 프로젝트 π | 선택 속성으로 구성 (수직, 중복 제거) |
| 조인 ⋈ | 공통 속성으로 연결 |
| 자연 조인 ⋈ₙ | 동등 조인 + 중복 제거 |
| 디비전 ÷ | S의 모든 값과 매칭되는 R 투플 |
| 외부 조인 ⟕⟖⟗ | 제외 투플을 NULL로 포함 |

{{< callout type="info" >}}
**용어 정리**
- **관계 대수 (Relational Algebra)**: 절차적 연산 언어, 처리 순서를 기술
- **관계 해석 (Relational Calculus)**: 비절차적 언어, 원하는 결과만 기술
- **폐쇄 특성 (Closure)**: 연산 결과도 릴레이션이라 연쇄 연산 가능
- **합병 가능 (Union Compatible)**: 차수와 도메인이 일치하는 두 릴레이션
- **합집합 / 교집합 / 차집합 / 카티션 프로덕트**: 일반 집합 연산자 (∪, ∩, −, ×)
- **셀렉트 (Select, σ)**: 조건을 만족하는 투플 선택 (수평)
- **프로젝트 (Project, π)**: 지정 속성 추출 (수직, 중복 제거)
- **조인 (Join, ⋈)**: 공통 속성 기준 연결
- **세타 / 동등 / 자연 조인**: 비교 연산 / `=` 사용 / 중복 속성 제거
- **외부 조인 (Outer Join)**: 제외 투플을 NULL로 포함 (⟕, ⟖, ⟗)
- **세미 조인 (Semi Join, ⋉)**: 조인 속성으로 프로젝트 후 조인
- **디비전 (Division, ÷)**: S의 모든 값과 매칭되는 R의 투플
- **관계적으로 완전 (Relationally Complete)**: 관계 대수·해석의 모든 질의 표현 가능
