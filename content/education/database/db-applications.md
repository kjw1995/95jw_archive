## 1. 객체지향 데이터베이스

### 등장 배경

관계 데이터베이스는 SQL 표현력이 제한적이고, 복잡한 데이터 구조를 표현하기 어렵다. CAD/CAM/CASE처럼 특수 분야에서는 한계가 뚜렷하다. 이를 극복하고자 객체지향 개념을 데이터 모델에 도입한 것이 객체지향 데이터베이스다.

### 핵심 개념

| 개념 | 설명 |
|:---|:---|
| 객체 | 현실 세계의 개체 표현 |
| OID | 객체 식별자 (유일) |
| 속성 | 객체의 상태 |
| 메서드 | 객체의 동작 |
| 클래스 | 유사 객체의 그룹 |
| 상속 | 속성·메서드를 물려받음 |

### 객체와 OID

```text
┌──────────────┐
│    객체      │
├──────────────┤
│ OID          │
├──────────────┤
│ 속성         │
│  - 이름      │
│  - 나이      │
├──────────────┤
│ 메서드       │
│  - getName() │
│  - setAge()  │
└──────────────┘
```

OID로 객체 간 참조가 이루어지고, 메시지로 객체에 접근한다.

### 클래스 계층과 상속

```text
      [Person]
          │
      IS-A 관계
          │
    ┌─────┴─────┐
    ▼           ▼
[Student] [Employee]
    │
    ▼
[Graduate]
```

| 유형 | 설명 |
|:---|:---|
| 단일 상속 | 하나의 상위 클래스 |
| 다중 상속 | 여러 상위 클래스 |

### 복합 객체

다른 객체를 속성 값으로 참조하는 객체다. Is-Part-Of 관계를 표현하며 속성 값으로 다른 객체의 OID를 저장한다.

```text
[자동차] ──┬──► [엔진]
           ├──► [바퀴]
           └──► [핸들]
         Is-Part-Of
```

### 객체지향 질의

| 구분 | 관계 DB | 객체지향 DB |
|:---|:---|:---|
| 대상 | 릴레이션 | 클래스 |
| 결과 | 릴레이션 | 객체 집합 |

표준화된 질의어가 부족한 점이 한계다.

## 2. 객체-관계 데이터베이스

관계 DB에 객체지향 개념을 결합한 형태다. 릴레이션을 유지하면서도 객체·메서드·클래스·상속·캡슐화·사용자 정의 타입을 지원한다.

| 구분 | 객체지향 DB | 객체-관계 DB |
|:---|:---|:---|
| 기반 | OOP 개념 | 관계 DB |
| 목적 | DB 기능 추가 | 타입 확장 |
| 표준 | 미흡 | SQL3 |

SQL3(1999~)는 객체지향 개념을 공식 표준에 반영해 사용자 정의 타입, 객체 식별자, 메서드 정의를 지원한다.

## 3. 관계형 → 객체-관계형 → NoSQL 진화

데이터 모델은 다루는 데이터의 형태와 규모에 맞춰 변화해 왔다.

```text
┌──────────┐  ┌──────────┐  ┌──────────┐
│  관계형  │─►│ 객체-관계 │─►│  NoSQL   │
│  (RDB)   │  │  (ORDB)  │  │          │
└──────────┘  └──────────┘  └──────────┘
정형 중심      복잡 타입     비정형·분산
SQL 중심       SQL3          CAP 기반
```

| 세대 | 특징 | 한계 |
|:---|:---|:---|
| 관계형 | 정규화·SQL 표준 | 복잡 타입·대용량 |
| 객체-관계형 | 사용자 정의 타입·상속 | 분산 확장 어려움 |
| NoSQL | 분산·비정형 지원 | ACID·표준 질의 미흡 |

### 선택 기준 체크리스트

| 질문 | Yes → 선택 |
|:---|:---|
| 트랜잭션 정합성이 핵심인가 | 관계형 / 객체-관계형 |
| 스키마가 자주 바뀌는가 | 문서형 NoSQL |
| 초당 수만 건의 대용량 쓰기가 있는가 | 컬럼형 NoSQL |
| 데이터 간 관계 탐색이 핵심인가 | 그래프 DB |
| 캐시·세션 같은 단순 키-값인가 | 키-값 NoSQL |
| 복잡 타입(BLOB, 사용자 정의)이 많은가 | 객체-관계형 |

{{< callout type="info" >}}
한 시스템에 여러 저장소를 조합하는 **폴리글랏 퍼시스턴스(Polyglot Persistence)** 도 일반적이다. 주문은 RDB, 추천 그래프는 Neo4j, 세션은 Redis처럼 용도에 맞는 저장소를 선택한다.
{{< /callout >}}

## 4. 분산 데이터베이스

### 개념

물리적으로 분산된 여러 데이터베이스를 논리적으로 하나처럼 사용하게 해주는 시스템이다.

```text
┌──────────────────────────┐
│   논리적: 하나의 DB       │
├──────────────────────────┤
│  ┌────┐     ┌────┐       │
│  │지역A│◄──►│지역B│       │
│  └──┬─┘     └──┬─┘       │
│     │  네트워크 │          │
│     └────┬─────┘          │
│          ▼                │
│       ┌────┐              │
│       │지역C│              │
│       └────┘              │
│   물리적: 분산된 DB        │
└──────────────────────────┘
```

| 구성 | 설명 |
|:---|:---|
| 분산 처리기 | 지역 컴퓨터 + DBMS |
| 분산 DB | 지역별 데이터베이스 |
| 네트워크 | 통신망 연결 |

### 분산 투명성

분산되어 있음을 사용자가 인식하지 않아도 된다.

| 투명성 | 의미 |
|:---|:---|
| 위치 | 저장 위치 몰라도 접근 |
| 중복 | 중복 저장 인식 불필요 |
| 단편화 | 분할 저장 인식 불필요 |
| 병행 | 동시 수행해도 일관성 유지 |
| 장애 | 일부 장애에도 계속 동작 |

### 데이터 중복

| 방식 | 설명 |
|:---|:---|
| 완전 중복 | 전체 데이터 복제 |
| 부분 중복 | 일부만 복제 |
| 분할 | 중복 없이 분할 |

| 장점 | 단점 |
|:---|:---|
| 신뢰성·가용성 향상 | 저장 공간 증가 |
| 병렬 처리 가능 | 변경 비용 증가 |
| 부하 분산 | 불일치 위험 |

### 단편화

릴레이션을 작은 조각으로 분할하는 기법이다. 완전성(모든 데이터 포함)·회복성(원본 복구 가능)·분리성(조각 간 중복 없음)을 만족해야 한다.

| 유형 | 분할 단위 |
|:---|:---|
| 수평적 | 투플(행) 단위 |
| 수직적 | 속성(열) 단위 |
| 혼합 | 둘 다 사용 |

### 분산 DB 스키마

```text
전역 개념 스키마
      │
      ▼
단편화 스키마 (논리 분할)
      │
      ▼
할당 스키마 (물리 배치)
      │
      ▼
지역 스키마
```

| 장점 | 단점 |
|:---|:---|
| 신뢰성·가용성 증대 | 설계·구축 비용 증가 |
| 지역 자치성 | 관리 복잡 |
| 확장 용이 | 통신 비용 발생 |

## 5. 멀티미디어 데이터베이스

### 멀티미디어 데이터

| 유형 | 예시 |
|:---|:---|
| 텍스트 | 문서, 문자 |
| 그래픽 | 벡터 이미지 |
| 이미지 | 사진, 비트맵 |
| 비디오 | 동영상 |
| 오디오 | 음성, 음악 |

### 특성

대용량이어서 압축 저장이 필요하고, 검색 방식도 정형 데이터와 다르다.

| 검색 방식 | 설명 |
|:---|:---|
| 설명 기반 | 키워드·설명으로 검색 |
| 내용 기반 | 실제 내용으로 검색 |

멀티미디어 데이터는 원시 데이터, 등록 데이터(처리용 메타), 서술 데이터(검색용 키워드)로 계층화된다.

```text
┌─────────────────┐
│ 멀티미디어 데이터 │
├─────────────────┤
│ 원시 데이터      │
├─────────────────┤
│ 등록 데이터      │
├─────────────────┤
│ 서술 데이터      │
└─────────────────┘
```

### 구현 방식

| 방식 | 특징 |
|:---|:---|
| 파일 시스템 | 직접 관리, 개발 부담 |
| 관계 DB + 파일 | SQL + 파일 병행 |
| 확장 관계 DB | BLOB 타입 지원 |
| 객체지향 DB | 추상화·상속 |

BLOB(Binary Large Object)은 이미지·비디오 같은 대용량 이진 데이터를 한 컬럼에 저장하는 타입이다.

### 질의 처리 기법

| 기법 | 설명 |
|:---|:---|
| 매칭 | 유사도 계산 |
| 랭킹 | 관련성 순 정렬 |
| 필터링 | 단계적 범위 축소 |
| 인덱스 | 인덱스 활용 검색 |

## 6. 웹 데이터베이스

웹 서비스와 DB를 통합한 구조다. 브라우저 → 웹 서버 → 미들웨어 → DBMS의 계층으로 동작한다.

```text
┌─────────┐
│ 브라우저 │
└────┬────┘
     │ HTTP
     ▼
┌─────────┐
│ 웹 서버  │
└────┬────┘
     ▼
┌─────────────┐
│  미들웨어   │
│ (DB Gateway)│
└────┬────────┘
     │ SQL
     ▼
┌─────────┐
│  DBMS   │
└─────────┘
```

| 미들웨어 방식 | 위치 |
|:---|:---|
| 서버 확장 | 웹 서버 측 (주류) |
| 클라이언트 확장 | 브라우저 측 |

## 7. 데이터 웨어하우스

### 개념

여러 운영 DB에서 데이터를 추출·변환·통합해 의사 결정 지원용으로 구축한 읽기 전용 DB다.

```text
┌───────┐ ┌───────┐
│운영DB1│ │운영DB2│
└───┬───┘ └───┬───┘
    │  추출   │
    └────┬────┘
         ▼
    ┌────────┐
    │변환·통합│
    └────┬───┘
         ▼
   ┌──────────┐
   │ 웨어하우스│
   │(읽기 전용)│
   └────┬─────┘
        ▼
  [분석] [리포트]
```

### 일반 DB vs 웨어하우스

| 구분 | 일반 DB | 웨어하우스 |
|:---|:---|:---|
| 목적 | 업무 처리 | 의사 결정 |
| 작업 | CRUD | 읽기 전용 |
| 데이터 | 최신 | 과거 + 현재 |
| 사용자 | 업무 담당자 | 경영진·분석가 |

### 4가지 특징

| 특징 | 설명 |
|:---|:---|
| 주제 지향적 | 의사 결정 주제 중심 |
| 통합성 | 여러 DB에서 추출·통합 |
| 시간 변이성 | 과거 + 현재 스냅샷 |
| 비소멸성 | 읽기 전용, 변경 없음 |

## 핵심 정리

| 개념 | 설명 |
|:---|:---|
| 객체지향 DB | 객체·OID·클래스·상속 |
| 객체-관계 DB | 관계 DB + 객체 기능, SQL3 |
| NoSQL | 비정형·분산 지향 |
| 분산 DB | 물리 분산, 논리 통합 |
| 분산 투명성 | 위치·중복·단편화·병행·장애 |
| 단편화 | 수평·수직·혼합 |
| 멀티미디어 DB | 대용량 미디어, BLOB |
| 웹 DB | 웹 서버 + 미들웨어 + DBMS |
| 데이터 웨어하우스 | 의사결정 지원, 읽기 전용 |
| 폴리글랏 | 용도별 저장소 조합 |

{{< callout type="info" >}}
**용어 정리**
- **객체지향 DB**: OID·상속·메서드를 데이터 모델에 도입한 DB
- **OID**: 객체의 유일한 식별자
- **객체-관계 DB (ORDB)**: 관계 DB에 객체 기능을 더한 모델
- **SQL3**: 객체지향 개념을 반영한 SQL 표준
- **NoSQL**: 비관계형·분산 지향 DB의 총칭
- **폴리글랏 퍼시스턴스**: 용도별로 서로 다른 DB를 조합
- **분산 DB**: 물리 분산·논리 통합 DB 시스템
- **분산 투명성**: 분산 사실을 숨기는 다섯 가지 속성
- **단편화**: 릴레이션을 조각으로 나누는 기법
- **BLOB**: 대용량 이진 데이터 타입
- **미들웨어**: 웹 서버와 DBMS를 연결하는 중간 계층
- **데이터 웨어하우스**: 여러 운영 DB를 통합한 분석용 DB
{{< /callout >}}
