## 1. 엔티티 매니저 팩토리와 엔티티 매니저

### 구조

```
   EntityManagerFactory
   (앱당 1개, 스레드 안전)
    ┌──────────────┐
    │ Connection   │
    │    Pool      │
    └──────────────┘
          │
   createEntityManager()
    ┌─────┼─────┐
    ▼     ▼     ▼
   EM    EM    EM
  (req1)(req2)(req3)
   스레드 공유 금지
    │     │     │
    ▼     ▼     ▼
  ┌──────────────┐
  │  영속성       │
  │  컨텍스트     │
  └──────────────┘
```

### 핵심 특징

| 구분 | EntityManagerFactory | EntityManager |
|:-----|:---------------------|:--------------|
| 생성 비용 | 매우 높음 | 낮음 |
| 생성 횟수 | 앱당 1개 | 요청마다 |
| 스레드 안전 | O (공유 가능) | X (공유 금지) |
| 커넥션 | 풀 관리 | 필요 시점에 획득 |

```java
// 팩토리 생성 (1번만, 비용 큼)
EntityManagerFactory emf =
    Persistence.createEntityManagerFactory("jpabook");

// 엔티티 매니저 생성 (요청마다)
EntityManager em = emf.createEntityManager();
```

{{< callout type="info" >}}
엔티티 매니저는 **DB 커넥션을 필요한 시점까지 얻지 않는다** (지연 획득). 보통 트랜잭션이 시작될 때 커넥션을 빌려오고, 커밋/롤백 시 반환한다.
{{< /callout >}}

## 2. 영속성 컨텍스트란?

> **"엔티티를 영구 저장하는 환경"**

`persist()` 라는 이름은 "DB 에 저장" 이 아니라 **"영속성 컨텍스트에 등록"** 한다는 뜻이다. 실제 SQL 은 플러시 시점에 실행된다.

```java
em.persist(entity);   // 영속성 컨텍스트에 저장
```

### 개념

- **논리적 개념**이라 눈에 보이지 않는다
- 엔티티 매니저를 통해서만 접근한다
- 엔티티 매니저 1개 = 영속성 컨텍스트 1개 (기본값)

```
  ┌─────────────────────┐
  │   EntityManager     │
  │          │          │
  │          ▼          │
  │  ┌───────────────┐  │
  │  │ 영속성 컨텍스트 │  │
  │  │ ┌───────────┐ │  │
  │  │ │ 1차 캐시   │ │  │
  │  │ │ @Id│Entity│ │  │
  │  │ │────┼──────│ │  │
  │  │ │"1" │member│ │  │
  │  │ └───────────┘ │  │
  │  │ ┌───────────┐ │  │
  │  │ │쓰기 지연   │ │  │
  │  │ │ SQL 저장소 │ │  │
  │  │ └───────────┘ │  │
  │  └───────────────┘  │
  └─────────────────────┘
```

### 영속성 컨텍스트의 5가지 이점

| 이점 | 설명 |
|:-----|:-----|
| 1차 캐시 | 동일 트랜잭션 내 캐싱 |
| 동일성 보장 | 같은 엔티티는 같은 인스턴스 |
| 쓰기 지연 | SQL 을 모아 한 번에 전송 |
| 변경 감지 | 엔티티 변경 시 자동 UPDATE |
| 지연 로딩 | 실제 사용 시점에 조회 |

각 이점은 다음 챕터(04·05)에서 상세히 다룬다.

## 요약

| 항목 | 핵심 |
|:-----|:-----|
| EntityManagerFactory | 앱당 1개, 스레드 안전 |
| EntityManager | 요청당 1개, 공유 금지 |
| 영속성 컨텍스트 | 엔티티를 영구 저장하는 환경 |
| `persist()` | 영속성 컨텍스트에 저장 |
| 커넥션 획득 | 실제 필요 시점까지 지연 |

### 핵심 코드

```java
// 팩토리 생성 (앱 시작 시 1번)
EntityManagerFactory emf =
    Persistence.createEntityManagerFactory("jpabook");

// 엔티티 매니저 생성 (요청마다)
EntityManager em = emf.createEntityManager();

// 영속성 컨텍스트에 저장
em.persist(entity);

// 종료
em.close();
emf.close();
```
