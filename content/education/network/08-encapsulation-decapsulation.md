## 1. 캡슐화 (Encapsulation)

송신측에서 데이터를 전송할 때 **각 계층에서 필요한 제어 정보(헤더)를 덧붙이는 과정**이다. 상위 계층에서 하위 계층으로 내려가며 헤더가 순차적으로 붙는다.

### 왜 필요한가

헤더 없이 데이터만 보내면 수신측은 출발지·목적지·순서·오류 여부를 알 수 없다. 헤더는 각 계층이 자신의 역할을 수행하기 위한 메타데이터다.

| 구성 | 위치 | 역할 |
|:---|:---|:---|
| 헤더 (Header) | 앞 | 주소·제어 정보 |
| 페이로드 | 중간 | 실제 데이터 |
| 트레일러 (Trailer) | 뒤 | 오류 검출 (FCS) |

---

## 2. 캡슐화 흐름

```
응용     [             데이터             ]
             ▼  TCP 헤더 추가
전송     [TCP][           데이터          ]  세그먼트
             ▼  IP 헤더 추가
네트워크 [IP][TCP][       데이터          ]  패킷
             ▼  이더넷 헤더·FCS 추가
데이터   [ETH][IP][TCP][ 데이터 ][FCS]     프레임
링크
             ▼  신호 변환
물리     10101010101010...                 비트
```

---

## 3. 계층별 PDU

| 계층 | PDU | 구성 |
|:---|:---|:---|
| 응용 | 데이터 (Data) | 사용자 데이터 |
| 전송 | 세그먼트 / 데이터그램 | TCP/UDP 헤더 + 데이터 |
| 네트워크 | 패킷 (Packet) | IP 헤더 + 세그먼트 |
| 데이터링크 | 프레임 (Frame) | MAC 헤더 + 패킷 + FCS |
| 물리 | 비트 (Bit) | 0/1 신호 |

{{< callout type="info" >}}
전송 계층에서 TCP는 **세그먼트(Segment)**, UDP는 **데이터그램(Datagram)**이라 부른다. 네트워크 계층부터는 TCP·UDP 구분 없이 **패킷**이다.
{{< /callout >}}

## 4. 각 계층 헤더의 핵심 필드

### TCP 헤더 (전송 계층)

| 필드 | 크기 | 설명 |
|:---|:---|:---|
| 출발지 포트 | 16bit | 송신 애플리케이션 |
| 목적지 포트 | 16bit | 수신 애플리케이션 |
| 순서 번호 | 32bit | 바이트 순서 |
| 확인 응답 번호 | 32bit | 수신 확인 |
| 플래그 | 6bit | SYN, ACK, FIN 등 |
| 윈도우 크기 | 16bit | 흐름 제어 |

### IP 헤더 (네트워크 계층)

| 필드 | 크기 | 설명 |
|:---|:---|:---|
| 버전 | 4bit | IPv4=4, IPv6=6 |
| TTL | 8bit | 패킷 수명 |
| 프로토콜 | 8bit | TCP=6, UDP=17, ICMP=1 |
| 출발지 IP | 32bit | 송신자 IP |
| 목적지 IP | 32bit | 수신자 IP |

### 이더넷 헤더 (데이터링크 계층)

| 필드 | 크기 | 설명 |
|:---|:---|:---|
| 목적지 MAC | 6B | 수신 NIC 주소 |
| 출발지 MAC | 6B | 송신 NIC 주소 |
| 타입 | 2B | 상위 프로토콜 (IPv4=0x0800) |
| FCS | 4B | 오류 검출 |

## 5. 역캡슐화 (Decapsulation)

수신측에서 **각 계층의 헤더를 역순으로 제거**하며 원본 데이터를 추출하는 과정이다.

```
물리     101010...     ▼ 비트 → 프레임
링크    [ETH][IP][TCP][데이터][FCS]
           ├ FCS 검사
           ├ MAC 확인
           ▼ 헤더 제거
네트워크 [IP][TCP][데이터]
           ├ IP 확인
           ▼ 헤더 제거
전송     [TCP][데이터]
           ├ 포트 확인
           ▼ 헤더 제거
응용     [데이터]
```

### 계층별 검증

| 계층 | 검증 | 불일치 시 |
|:---|:---|:---|
| 데이터링크 | MAC·FCS | 프레임 폐기 |
| 네트워크 | IP·체크섬 | 폐기 또는 라우팅 |
| 전송 | 포트·체크섬 | 세그먼트 폐기 |

## 6. 웹 요청 전체 흐름

```
[클라이언트]                       [서버]
192.168.1.10                      10.0.0.5

응용     HTTP GET /index.html
           ▼ 캡슐화
전송     [TCP: 50000→80]
           ▼
네트워크 [IP: 192.168.1.10→10.0.0.5]
           ▼
링크    [ETH: AA→BB][...][FCS]
           ▼
물리     101011001...

         ═══ 네트워크 전송 ═══

물리     신호 수신
           ▼ 역캡슐화
링크    MAC 확인 → 헤더 제거
           ▼
네트워크 IP 확인 → 헤더 제거
           ▼
전송     포트 80 → 웹 서버로 전달
           ▼
응용     HTTP 요청 처리
```

## 7. MTU와 단편화

| 용어 | 설명 |
|:---|:---|
| MTU | Maximum Transmission Unit, 한 프레임 최대 크기 |
| 이더넷 MTU | 1500 bytes |
| MSS | Maximum Segment Size, TCP 세그먼트 데이터 최대 크기 |

### 프레임 크기 예시

```
응용     [  1460 B 데이터  ]
         ▼
전송    [20B ][  1460 B    ]  = 1480 B
         TCP
         ▼
네트워크[20B][20B][ 1460 B ]  = 1500 B  ← MTU
        IP  TCP
         ▼
링크   [14B][20B][20B][1460][4B] = 1518 B
       ETH  IP  TCP Data FCS
```

MTU를 초과하면 네트워크 계층에서 **단편화(Fragmentation)**가 발생한다.

## 8. VPN과 추가 캡슐화

VPN(Virtual Private Network)은 공용 인터넷 위에 **암호화된 터널**을 만들어 원격지 사이를 전용선처럼 연결하는 기술이다.

### VPN 캡슐화

```
원본 패킷
[IP Header][TCP Header][Data]

VPN 캡슐화 후
[새 IP][VPN][암호화된 원본 패킷]
```

### VPN 유형

| 유형 | 프로토콜 | 용도 |
|:---|:---|:---|
| 거점 간 | IPsec | 본사-지사 |
| 원격 접속 | SSL/TLS | 재택·출장 |
| MPLS (IP-VPN) | 통신사 전용망 | 대기업 폐쇄망 |

### IPsec 구성

| 구성 | 역할 |
|:---|:---|
| AH | 인증·무결성 |
| ESP | 인증·암호화 |

| 모드 | 동작 |
|:---|:---|
| 전송 모드 | 페이로드만 암호화 |
| 터널 모드 | 전체 패킷 암호화 + 새 IP 헤더 |

## 핵심 정리

| 개념 | 설명 |
|:---|:---|
| 캡슐화 | 송신 시 헤더 추가 |
| 역캡슐화 | 수신 시 헤더 제거 |
| 헤더 | 제어 정보 |
| 트레일러 | 오류 검출 (FCS) |
| PDU | 계층별 데이터 단위 |
| MTU | 최대 전송 단위 (1500B) |
| VPN | 암호화된 가상 터널 |

### 캡슐화 순서

```
송신: 응용 → 전송 → 네트워크 → 링크 → 물리 (헤더 추가)
수신: 물리 → 링크 → 네트워크 → 전송 → 응용 (헤더 제거)
```

{{< callout type="info" >}}
**용어 정리**
- **캡슐화 / 역캡슐화**: 헤더 추가·제거 과정
- **PDU**: Protocol Data Unit
- **MTU**: Maximum Transmission Unit
- **VPN**: Virtual Private Network
- **IPsec**: IP 계층 보안 프로토콜 (AH, ESP)
- **MPLS**: Multi-Protocol Label Switching
{{< /callout >}}
