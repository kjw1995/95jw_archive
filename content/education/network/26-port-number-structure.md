## 1. 포트 번호란?

포트 번호(Port Number)는 16비트 정수(0~65535)로, 한 호스트에서 실행되는 **여러 애플리케이션을 구분**하기 위한 식별자다.

- IP 주소: 어느 컴퓨터인가?
- 포트 번호: 그 컴퓨터 안의 어떤 프로그램인가?

```text
192.168.1.10 : 80
└─── IP ───┘  └ Port
```

비유하자면 IP가 "아파트 동번호", 포트가 "호수"에 해당한다. 운영체제는 도착한 세그먼트/데이터그램의 포트 번호를 보고 해당 애플리케이션에 전달한다.

{{< callout type="info" >}}
포트 없이 IP만 있다면 같은 서버에서 웹, 메일, SSH가 동시에 돌 수 없다. 전송 계층의 멀티플렉싱은 포트 번호에 기대고 있다.
{{< /callout >}}

## 2. 포트 번호의 범위

전체 범위는 0~65535이며 IANA가 다음과 같이 구분한다.

| 범위 | 분류 | 설명 |
|:---|:---|:---|
| 0 ~ 1023 | Well-Known Ports | 표준 서비스, 시스템 포트 |
| 1024 ~ 49151 | Registered Ports | IANA 등록 애플리케이션 |
| 49152 ~ 65535 | Dynamic / Private | 임시 포트 (에페메럴) |

운영체제마다 실제 동적 포트 범위는 조금씩 다르다.

| OS | 기본 범위 |
|:---|:---|
| Linux | 32768 ~ 60999 |
| Windows | 49152 ~ 65535 |

## 3. Well-Known Ports (0~1023)

| 포트 | 서비스 | 설명 |
|:---|:---|:---|
| 20 / 21 | FTP | 데이터 / 제어 |
| 22 | SSH | 암호화 원격 셸 |
| 23 | Telnet | 평문 원격 셸 |
| 25 | SMTP | 메일 전송 |
| 53 | DNS | 이름 해석 |
| 67 / 68 | DHCP | 서버 / 클라이언트 |
| 80 | HTTP | 웹 |
| 110 | POP3 | 메일 수신 |
| 143 | IMAP | 메일 수신 (고급) |
| 443 | HTTPS | TLS 웹 |
| 465 | SMTPS | TLS 메일 전송 |
| 993 | IMAPS | TLS IMAP |
| 995 | POP3S | TLS POP3 |

{{< callout type="warning" >}}
Unix/Linux에서 1024 미만 포트를 바인딩하려면 root 권한이 필요하다. 컨테이너나 일반 사용자 환경에서는 8080처럼 1024 이상 포트를 쓰거나 capabilities로 우회한다.
{{< /callout >}}

## 5. 소켓: IP + Port

두 호스트가 통신할 때 실제로 연결되는 것은 소켓 쌍이다.

```text
클라이언트 192.168.1.10:52341
                 ↕
서버       203.0.113.50:80
```

TCP 연결은 다음 네 값의 조합으로 고유하게 식별된다.

- 출발지 IP
- 출발지 포트
- 목적지 IP
- 목적지 포트

같은 서버(IP:Port)에 여러 탭이 접속해도 클라이언트 포트가 다르므로 별개의 연결이 된다.

```text
(192.168.1.10:52341, 142.250.185.78:443)   # 구글 탭 1
(192.168.1.10:52342, 142.250.185.78:443)   # 구글 탭 2
(192.168.1.10:52343, 140.82.114.4:443)     # 깃허브 탭
```

## 6. 동적 포트 할당

클라이언트는 서버에 연결할 때 자신이 사용할 포트를 직접 고르지 않는다. 운영체제가 사용 가능한 포트 풀(Dynamic Range)에서 하나를 골라 자동으로 할당한다.

과정은 다음과 같다.

1. 애플리케이션이 `connect()` 호출
2. OS가 동적 범위에서 비어 있는 포트 선택
3. 선택된 포트로 SYN 전송
4. 연결 종료 시 포트 반환 (이후 재사용 가능)

### 할당 방식

- **순차 할당**: 49152 → 49153 → ... (예측 가능, 보안 취약)
- **무작위 할당**: 최신 OS 기본, 포트 스캔 방어

## 7. TCP/UDP 헤더에서의 포트

TCP와 UDP 모두 헤더 시작부에 **출발지 포트(16비트)** 와 **목적지 포트(16비트)** 를 둔다.

### TCP 헤더 (앞부분)

```text
 0        16        31
 ├─────────┼─────────┤
 │ src port│ dst port│
 ├─────────┴─────────┤
 │  sequence number  │
 └───────────────────┘
```

### UDP 헤더 (전체 8바이트)

```text
 0        16        31
 ├─────────┼─────────┤
 │ src port│ dst port│
 ├─────────┼─────────┤
 │  length │checksum │
 └─────────┴─────────┘
```

UDP는 응답이 필요 없으면 출발지 포트를 0으로 둘 수 있다.

## 8. 포트 확인과 관리

### 상태 확인

```bash
# Windows
netstat -an
netstat -ano                     # PID 포함
netstat -ano | findstr ":80"

# Linux
ss -tuln                         # 현재 리스닝 포트
ss -tulnp                        # 프로세스 포함
lsof -i :80                      # 특정 포트 사용 프로세스
```

### 포트 충돌 해결

"Address already in use" 오류는 같은 포트를 두 프로세스가 바인딩하려 할 때 발생한다.

1. 점유 프로세스 확인 후 종료
   ```bash
   # Linux
   lsof -i :80
   kill -9 <PID>

   # Windows
   netstat -ano | findstr :80
   taskkill /PID <PID> /F
   ```
2. 다른 포트 사용 (예: 8080)
3. TIME_WAIT 상태라면 `SO_REUSEADDR` 옵션 사용

### 방화벽 설정

```bash
# Windows
netsh advfirewall firewall add rule name="HTTP 80" dir=in action=allow protocol=TCP localport=80

# Linux (firewalld)
firewall-cmd --permanent --add-port=80/tcp
firewall-cmd --reload

# Linux (ufw)
ufw allow 80/tcp
```

## 9. 포트 보안 권장 사항

- **불필요한 포트 닫기**: 사용하지 않는 서비스 중지, 주기적으로 `nmap` 등으로 감사
- **관리 포트 변경**: SSH 22 → 2222 등으로 자동 스캔 공격 회피 (단독으로는 보안이 되지 않으며 키 인증 등과 병행)
- **접근 제어**: 관리 포트는 특정 IP만 허용, DB는 127.0.0.1만 바인딩
- **포트 노킹**: 특정 순서로 포트를 두드려야 개방되는 기법

## 핵심 정리

| 개념 | 설명 |
|:---|:---|
| 포트 번호 | 16비트 (0~65535) 애플리케이션 식별자 |
| Well-Known | 0~1023, 표준 서비스 |
| Registered | 1024~49151, 등록된 애플리케이션 |
| Dynamic | 49152~65535, 클라이언트 임시 포트 |
| 소켓 | IP:Port 조합 |
| 연결 식별 | (출발지 IP:Port, 목적지 IP:Port) 4쌍 |
| 동적 포트 할당 | OS가 클라이언트 포트를 자동 배정 |
| 포트 충돌 | 같은 포트를 두 프로세스가 바인딩 시도 |

{{< callout type="info" >}}
**용어 정리**
- **포트 번호 (Port Number)**: 16비트 애플리케이션 식별자
- **Well-Known Ports**: 0~1023, 표준 서비스용
- **Registered Ports**: 1024~49151, 등록 애플리케이션
- **Dynamic / Ephemeral Ports**: 49152~65535, 클라이언트 임시
- **소켓 (Socket)**: IP + Port 조합, 통신 종단점
- **동적 포트 할당**: OS가 자동으로 클라이언트 포트를 배정하는 과정
- **포트 충돌**: 동일 포트를 복수 프로세스가 점유하려 할 때의 오류
- **포트 노킹**: 지정된 포트 순서를 두드려야 열리는 보안 기법
{{< /callout >}}
