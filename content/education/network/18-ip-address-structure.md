## 1. IP 주소란?

IP 주소는 네트워크 계층에서 장비를 식별하기 위한 **논리적 주소**이다. MAC 주소가 하드웨어에 고정된 물리 주소라면, IP 주소는 네트워크 구성에 따라 할당·변경된다.

| 항목 | IPv4 | IPv6 |
|:---|:---|:---|
| 비트 수 | 32 | 128 |
| 주소 개수 | 약 43억 | 약 3.4×10^38 |
| 표기 | 10진수 점표기 | 16진수 콜론 표기 |
| 예시 | 192.168.1.10 | 2001:db8::1 |

---

## 2. IPv4의 표현

### 32비트 = 4옥텟

IPv4 주소는 32비트를 8비트씩 나눈 4개의 **옥텟(octet)** 으로 이루어진다. 각 옥텟은 0~255 사이의 10진수로 표기한다.

```
2진수:
11000000 10101000 00000001 00001010

10진수 점표기:
   192  .  168  .   1   .  10
```

### 2진수 ↔ 10진수

| 2진수 | 10진수 |
|:---|:---|
| 10000000 | 128 |
| 11000000 | 192 |
| 11100000 | 224 |
| 11111111 | 255 |

각 비트 가중치(128, 64, 32, 16, 8, 4, 2, 1)의 합으로 계산한다.

---

## 3. 네트워크 부분과 호스트 부분

### 두 영역의 역할

IP 주소는 **네트워크 ID**와 **호스트 ID** 두 부분으로 나뉜다.

| 부분 | 역할 | 비유 |
|:---|:---|:---|
| 네트워크 ID | 어떤 네트워크인지 식별 | 도시·동네 |
| 호스트 ID | 네트워크 내 특정 장비 식별 | 번지수 |

```
192.168.1.10 / 24
└────┬─────┘ └┘
 네트워크 ID  호스트 ID
 (24bit)     (8bit)
```

### 같은 네트워크 vs 다른 네트워크

네트워크 ID가 같으면 같은 LAN에 속해 스위치만으로 통신할 수 있다. 다르면 라우터를 거쳐야 한다.

## 4. 서브넷 마스크와 CIDR

### 서브넷 마스크의 원리

IP 주소와 서브넷 마스크를 AND 연산하면 네트워크 주소가 나온다.

```
IP       : 11000000 10101000 00000001 00001010
Mask     : 11111111 11111111 11111111 00000000
─────────────────────────────────────────────── AND
Network  : 11000000 10101000 00000001 00000000
         = 192.168.1.0
```

마스크에서 1인 구간이 네트워크 ID, 0인 구간이 호스트 ID이다.

### CIDR 표기

| 서브넷 마스크 | CIDR | 네트워크/호스트 |
|:---|:---|:---|
| 255.0.0.0 | /8 | 8 / 24 |
| 255.255.0.0 | /16 | 16 / 16 |
| 255.255.255.0 | /24 | 24 / 8 |
| 255.255.255.128 | /25 | 25 / 7 |
| 255.255.255.192 | /26 | 26 / 6 |
| 255.255.255.224 | /27 | 27 / 5 |
| 255.255.255.240 | /28 | 28 / 4 |
| 255.255.255.252 | /30 | 30 / 2 |

### 클래스 기반 분류 (참고)

| 클래스 | 범위 | 기본 마스크 | 용도 |
|:---|:---|:---|:---|
| A | 1~126 | /8 | 대규모 조직 |
| B | 128~191 | /16 | 중규모 조직 |
| C | 192~223 | /24 | 소규모·가정 |

현재는 클래스 구분 대신 CIDR로 유연하게 할당한다.

## 5. 공인 IP와 사설 IP

### 공인 IP

인터넷에서 유일하게 식별되는 주소로 IANA → RIR → ISP를 거쳐 할당된다.

### 사설 IP (RFC 1918)

내부 네트워크에서만 사용하며 인터넷에 직접 노출되지 않는다.

| 클래스 | 범위 |
|:---|:---|
| A | 10.0.0.0/8 |
| B | 172.16.0.0/12 |
| C | 192.168.0.0/16 |

### NAT의 역할

```
 내부(사설 IP)         외부(공인 IP)
[PC1 192.168.1.10]
[PC2 192.168.1.11] ─[NAT]─→ 203.0.113.50
[PC3 192.168.1.12]
```

NAT(특히 PAT/NAPT)는 포트 번호를 이용해 여러 사설 IP가 하나의 공인 IP를 공유하도록 만든다. IPv4 주소 고갈을 늦춘 핵심 기술이다.

## 6. 특수 IP 주소

| 주소 | 의미 |
|:---|:---|
| 0.0.0.0 | 모든 주소 / 기본 경로 |
| 127.0.0.0/8 | 루프백 (자기 자신) |
| 169.254.0.0/16 | APIPA (자동 링크 로컬) |
| 네트워크 주소 (호스트부 모두 0) | 네트워크 자체 |
| 브로드캐스트 (호스트부 모두 1) | 네트워크 전체 |

예: `192.168.1.0/24`에서 `.0`은 네트워크 주소, `.255`는 브로드캐스트 주소, 할당 가능한 호스트는 `.1 ~ .254`이다.

## 7. DHCP

네트워크에 접속한 호스트에게 IP·서브넷 마스크·기본 게이트웨이·DNS를 자동 할당하는 프로토콜이다. 동작은 **DORA** 4단계로 이루어진다.

| 단계 | 방향 | 메시지 | 설명 |
|:---|:---|:---|:---|
| D | Client→Server | Discover | IP 요청(브로드캐스트) |
| O | Server→Client | Offer | 주소 제안 |
| R | Client→Server | Request | 제안 수락 |
| A | Server→Client | Ack | 승인 + 임대 기간 |

```bash
# Windows
ipconfig /all

# Linux
ip addr
ip route
```

## 8. IPv6

### 표기와 축약

128비트를 16비트씩 8그룹으로 나눠 16진수·콜론 표기한다.

```
원본: 2001:0db8:0000:0000:0000:0000:0000:0001
축약: 2001:db8::1
```

| 규칙 | 설명 |
|:---|:---|
| 앞 0 생략 | `0db8` → `db8` |
| 연속 0 그룹 `::` | 한 번만 사용 가능 |

### IPv4와의 차이

| 항목 | IPv4 | IPv6 |
|:---|:---|:---|
| 비트 | 32 | 128 |
| 헤더 | 20~60 B (가변) | 40 B (고정) |
| 체크섬 | 헤더에 포함 | 제거(상위 계층 담당) |
| 브로드캐스트 | 지원 | 없음 (멀티캐스트로 대체) |
| 자동 설정 | DHCP | SLAAC + DHCPv6 |
| 보안 | IPsec 옵션 | IPsec 기본 고려 |

{{< callout type="info" >}}
IPv6는 주소 공간이 사실상 무한하여 NAT가 필요 없다. 다만 IPv4 기반 인터넷과 오랫동안 공존해야 하기 때문에 듀얼 스택(IPv4+IPv6 동시 운영), 터널링, 변환(NAT64/DNS64) 같은 전환 기술이 함께 쓰인다.
{{< /callout >}}

## 핵심 정리

| 개념 | 설명 |
|:---|:---|
| IPv4 | 32비트 점 10진 주소 |
| 옥텟 | 8비트 단위, 0~255 |
| 네트워크/호스트 | 서브넷 마스크로 구분 |
| CIDR | /숫자 형식 접두어 표기 |
| 공인 / 사설 IP | 인터넷 / 내부 네트워크용 |
| NAT | 사설↔공인 IP 변환 |
| DHCP | IP 자동 할당 프로토콜 |
| IPv6 | 128비트 주소, NAT 불필요 |

{{< callout type="info" >}}
**용어 정리**
- **IP 주소**: 네트워크 상 장비의 논리적 주소
- **옥텟 (Octet)**: 8비트 단위
- **서브넷 마스크 / CIDR**: 네트워크·호스트 경계 지정
- **공인 IP / 사설 IP**: 전 세계 고유 / 내부용
- **NAT (PAT)**: 사설↔공인 주소 변환
- **DHCP (DORA)**: IP 자동 할당 프로토콜
- **IPv6 / SLAAC**: 128비트 주소 / 상태 없는 자동 설정
{{< /callout >}}
