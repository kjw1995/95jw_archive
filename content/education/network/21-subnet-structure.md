## 1. 서브넷과 서브넷팅

서브넷(Subnetwork)은 하나의 큰 네트워크를 여러 개의 작은 네트워크로 분할한 논리적 단위다. 서브넷팅은 그 과정을 말한다.

호스트 ID의 일부 비트를 서브넷 식별용으로 "빌려"오면, 하나의 네트워크 주소 공간을 여러 개의 작은 네트워크로 쪼갤 수 있다.

```text
분할 전 (C 클래스)
 NET(24) | HOST(8)

분할 후 (/26)
 NET(24) | SUB(2) | HOST(6)
```

### 서브넷팅이 필요한 이유

- **브로드캐스트 도메인 축소**: 트래픽 감소, 충돌 최소화
- **보안 격리**: 부서·서버 그룹별 경계 설정
- **주소 효율**: 필요한 규모만큼만 할당해 낭비 방지
- **관리 편의성**: 장애 영향 범위 제한, 문서화 용이

## 2. 서브넷 마스크

서브넷 마스크는 IP 주소에서 네트워크 부분과 호스트 부분을 구분하는 32비트 값이다. 네트워크 비트는 1, 호스트 비트는 0으로 표현한다.

| 표기 | 예시 |
|:---|:---|
| 10진수 | 255.255.255.0 |
| CIDR 프리픽스 | /24 |
| 2진수 | 11111111.11111111.11111111.00000000 |

### 동작 원리: AND 연산

IP와 서브넷 마스크를 AND 연산하면 네트워크 주소가 나온다.

```text
IP:   192.168.1.130/26
IP:   11000000.10101000.00000001.10000010
MASK: 11111111.11111111.11111111.11000000
AND:  11000000.10101000.00000001.10000000
      → 192.168.1.128
```

두 호스트의 네트워크 주소가 같으면 직접 통신이 가능하고, 다르면 라우터가 필요하다.

## 3. 서브넷 마스크 변환표

| CIDR | 마스크(마지막 옥텟) | 블록 크기 | 호스트 수 |
|:---|:---|:---|:---|
| /24 | 0 (00000000) | 256 | 254 |
| /25 | 128 (10000000) | 128 | 126 |
| /26 | 192 (11000000) | 64 | 62 |
| /27 | 224 (11100000) | 32 | 30 |
| /28 | 240 (11110000) | 16 | 14 |
| /29 | 248 (11111000) | 8 | 6 |
| /30 | 252 (11111100) | 4 | 2 |

빠른 계산 공식은 다음과 같다.

```text
서브넷 수     = 2^(빌린 비트)
호스트 수     = 2^(호스트 비트) - 2
블록 크기     = 256 - 마스크 값
브로드캐스트  = 네트워크 주소 + 블록 크기 - 1
```

## 4. CIDR 표기법

CIDR(Classless Inter-Domain Routing)은 클래스 개념을 없애고 `IP/프리픽스` 형태로 네트워크 크기를 자유롭게 지정한다.

| CIDR | 호스트 수 | 마스크 | 대표 용도 |
|:---|:---|:---|:---|
| /30 | 2 | 255.255.255.252 | 라우터 간 P2P 링크 |
| /29 | 6 | 255.255.255.248 | 매우 작은 세그먼트 |
| /28 | 14 | 255.255.255.240 | 소규모 서버 그룹 |
| /27 | 30 | 255.255.255.224 | 소규모 부서 |
| /26 | 62 | 255.255.255.192 | 중소 부서 |
| /24 | 254 | 255.255.255.0 | 일반 LAN |
| /23 | 510 | 255.255.254.0 | 대규모 LAN |
| /22 | 1,022 | 255.255.252.0 | 큰 부서/지사 |

{{< callout type="info" >}}
클래스풀 방식이 "500대 → B 클래스(65,534개, 낭비 99%)"를 강요했다면, CIDR는 /23을 할당해 낭비를 약 2%로 줄인다.
{{< /callout >}}

## 5. 서브넷팅 단계별 예제

### 예제 1: 부서별 설계

주어진 네트워크: 192.168.10.0/24

| 부서 | 필요 호스트 |
|:---|:---|
| 영업부 | 50 |
| 개발부 | 30 |
| 관리부 | 20 |
| 서버팜 | 10 |

**Step 1.** 각 부서에 맞는 최소 서브넷을 고른다.

| 부서 | 필요 (+2) | 서브넷 | 제공 |
|:---|:---|:---|:---|
| 영업부 | 52 | /26 | 62 |
| 개발부 | 32 | /27 | 30 → /26 필요? 32이므로 /27의 30 부족 → /27의 경우 30개, 부족. 실전에서는 /27 사용 후 주의하거나 /26 사용 |
| 관리부 | 22 | /27 | 30 |
| 서버팜 | 12 | /28 | 14 |

{{< callout type="warning" >}}
"필요 + 2"가 제공 호스트 수를 초과하면 한 단계 큰 서브넷을 사용해야 한다. 예컨대 개발부 30대는 /27(30개)에 딱 맞지만 여유가 없으므로 확장을 고려한다면 /26을 선택한다.
{{< /callout >}}

**Step 2.** 큰 서브넷부터 주소를 할당한다(VLSM 원칙).

| 부서 | 네트워크 | 범위 |
|:---|:---|:---|
| 영업부 | 192.168.10.0/26 | .1 ~ .62 |
| 개발부 | 192.168.10.64/27 | .65 ~ .94 |
| 관리부 | 192.168.10.96/27 | .97 ~ .126 |
| 서버팜 | 192.168.10.128/28 | .129 ~ .142 |
| 여유 | 192.168.10.144 ~ .255 | 확장용 |

### 예제 2: 주어진 IP의 서브넷 정보

IP: 172.16.45.220/20

```text
마스크      = 255.255.240.0
블록 크기   = 256 - 240 = 16 (세 번째 옥텟 기준)
45 위치     = 32 ≤ 45 < 48 → 32 블록
```

- 네트워크 주소: 172.16.32.0
- 브로드캐스트: 172.16.47.255
- 호스트 범위: 172.16.32.1 ~ 172.16.47.254
- 호스트 수: 2^12 - 2 = 4,094

## 6. VLSM (가변 길이 서브넷 마스크)

고정 길이 서브넷 마스크는 모든 서브넷을 같은 크기로 나누기 때문에 주소 낭비가 크다. VLSM은 서브넷마다 서로 다른 마스크 길이를 사용해 필요한 만큼만 할당하는 기법이다.

### 고정 vs VLSM 비교 (192.168.1.0/24)

고정 /26으로 4등분:

| 서브넷 | 필요 | 제공 | 낭비 |
|:---|:---|:---|:---|
| A | 50 | 62 | 12 |
| B | 10 | 62 | 52 |
| C | 5 | 62 | 57 |
| D | 2 | 62 | 60 |

VLSM으로 필요에 맞게 분할:

| 서브넷 | 마스크 | 제공 | 낭비 |
|:---|:---|:---|:---|
| A | /26 | 62 | 12 |
| B | /28 | 14 | 4 |
| C | /29 | 6 | 1 |
| D | /30 | 2 | 0 |

낭비는 181개에서 17개로 줄어든다.

### VLSM 설계 원칙

1. **필요 호스트 수 파악** (각 서브넷별)
2. **큰 서브넷부터 정렬**
3. **큰 것부터 순서대로 할당** (작은 것부터 하면 빈 공간이 파편화됨)
4. **블록 경계 맞추기**
5. **향후 확장을 고려해 여유 확보**

## 7. 빠른 계산법

### 블록 크기 공식

```text
블록 크기 = 256 - 마스크 값
```

| 마스크 | 블록 | 첫 서브넷 경계 |
|:---|:---|:---|
| 255.255.255.192 (/26) | 64 | 0, 64, 128, 192 |
| 255.255.255.224 (/27) | 32 | 0, 32, 64, 96, ... |
| 255.255.255.240 (/28) | 16 | 0, 16, 32, 48, ... |
| 255.255.255.248 (/29) | 8 | 0, 8, 16, 24, ... |
| 255.255.255.252 (/30) | 4 | 0, 4, 8, 12, ... |

### 5단계 체크리스트

IP 192.168.5.130/27을 예로 든다.

1. 마스크 확인: /27 = 255.255.255.224
2. 블록 크기: 256 - 224 = 32
3. 속한 블록: 128 ≤ 130 < 160 → **128 블록**
4. 네트워크/브로드캐스트: 192.168.5.128 / 192.168.5.159
5. 호스트 수: 2^5 - 2 = 30

## 핵심 정리

| 개념 | 설명 |
|:---|:---|
| 서브넷 | 하나의 네트워크를 여러 개의 작은 네트워크로 분할한 단위 |
| 서브넷팅 | 호스트 비트의 일부를 빌려 네트워크를 나누는 과정 |
| 서브넷 마스크 | 네트워크/호스트 부분을 구분하는 32비트 값 |
| CIDR | 클래스 경계 없이 프리픽스로 크기 지정 |
| VLSM | 서브넷마다 다른 마스크 길이를 사용하는 기법 |
| 블록 크기 | 256 - 마스크 값, 서브넷 경계 간격 |

{{< callout type="info" >}}
**용어 정리**
- **서브넷 (Subnet)**: 상위 네트워크를 분할한 작은 네트워크
- **서브넷팅 (Subnetting)**: 네트워크를 분할하는 과정
- **서브넷 마스크 (Subnet Mask)**: 네트워크/호스트 경계를 표시하는 32비트 값
- **CIDR**: 클래스 개념을 없앤 프리픽스 표기법
- **VLSM**: 가변 길이 서브넷 마스크, 서브넷별로 크기를 달리하는 기법
- **블록 크기 (Block Size)**: 각 서브넷이 차지하는 주소 범위
- **프리픽스 길이**: 네트워크 비트 수 (예: /24)
{{< /callout >}}
