## 1. 물리 계층 개요

OSI 7계층 중 최하위 계층으로, 비트를 물리적인 신호로 바꾸어 전송 매체를 통해 전달한다.

### 주요 역할

1. 비트(0/1)를 전기·광·전파 신호로 변환
2. 전송 매체를 통한 신호 전달
3. 물리적 연결 규격 정의 (케이블·커넥터)
4. 전송 속도와 신호 타이밍 정의
5. 신호 동기화

### 데이터 흐름

```
송신
  10110100 ──▶ 신호 변환 ──▶ 매체로 전송

수신
  신호 수신 ──▶ 비트 복원 ──▶ 10110100
```

## 2. 아날로그 vs 디지털 신호

컴퓨터 내부는 디지털이지만, 실제 매체에서는 아날로그적 성질(전압·전파)을 이용해 신호를 싣는다.

### 비교

| 항목 | 아날로그 | 디지털 |
|:---|:---|:---|
| 파형 | 연속적 | 불연속 펄스 |
| 값 | 무한 | 0/1 |
| 노이즈 | 민감 | 강함 |
| 복원 | 어려움 | 재생 가능 |
| 장거리 | 감쇄 큼 | 중계 쉬움 |
| 예 | 전화, 라디오 | 이더넷, USB |

### 디지털 신호 파형

```
전압
 ▲
 │ ┌──┐  ┌──┐   ┌──┐
H├─┘  │  │  │   │  │
 │    │  │  │   │  │
L┼────┴──┴──┴───┴──┴──▶ 시간
비트: 1  0  1  0  0  0  1  0
```

## 3. 아날로그 ↔ 디지털 변환

### 디지털 → 디지털 (라인 코딩)

| 방식 | 특징 | 사용 예 |
|:---|:---|:---|
| NRZ | 1=HIGH, 0=LOW | 단순, 동기화 취약 |
| 맨체스터 | 매 비트 전환 | 10BASE-T |
| 4B/5B | 4비트 → 5비트 | 100BASE-TX |

### 디지털 → 아날로그 (변조, Modulation)

| 방식 | 변화 요소 | 사용 예 |
|:---|:---|:---|
| ASK | 진폭 | 광통신 |
| FSK | 주파수 | 모뎀 |
| PSK | 위상 | Wi-Fi, 위성 |
| QAM | 진폭 + 위상 | ADSL, 케이블 |

### 아날로그 → 디지털 (샘플링)

아날로그 신호를 일정 간격으로 **샘플링(Sampling) → 양자화(Quantization) → 부호화(Encoding)** 과정을 거쳐 디지털로 변환한다. 전화망의 PCM, 오디오의 WAV가 대표 예다.

{{< callout type="info" >}}
샘플링 주파수는 원본 신호 최고 주파수의 2배 이상이어야 한다(나이키스트 정리). CD 오디오는 44.1kHz, 전화는 8kHz다.
{{< /callout >}}

## 4. 랜 카드 (NIC)

NIC(Network Interface Card)는 컴퓨터와 네트워크를 연결하는 하드웨어다. OSI 1~2계층 기능을 담당한다.

### 주요 역할

1. 데이터 ↔ 전기 신호 상호 변환
2. MAC 주소로 기기 식별
3. 프레임 송수신
4. CRC(FCS)로 오류 검출
5. 송수신 데이터 버퍼링

### 구성 요소

```
┌──────────────────────────┐
│         NIC              │
│ ┌────┐ ┌────┐ ┌────┐     │
│ │PHY │ │버퍼│ │ROM │     │
│ │    │ │RAM │ │MAC │     │
│ └─┬──┘ └────┘ └────┘     │
│   │                       │
│ ┌─┴──┐                    │
│ │MAU │ 신호 변환           │
│ └─┬──┘                    │
│   │                       │
│ ┌─┴──┐                    │
│ │RJ45│ 케이블 포트         │
│ └────┘                    │
│ ══════════════════        │
│   PCI/PCIe 버스            │
└──────────────────────────┘
```

| 구성 | 역할 |
|:---|:---|
| 컨트롤러 (PHY) | 신호 처리·프로토콜 제어 |
| 트랜시버 (MAU) | 전기↔디지털 변환 |
| 버퍼 (RAM) | 송수신 임시 저장 |
| ROM | MAC 주소 저장 |
| RJ-45 포트 | UTP 케이블 연결 |
| LED | 링크·활동 상태 |

### NIC 종류

| 형태 | 설명 |
|:---|:---|
| 온보드 | 메인보드 통합 |
| PCIe 카드 | 확장 슬롯 장착, 고성능 |
| USB | 외장형, 노트북 확장 |
| 무선 | Wi-Fi 지원 |

### 이더넷 속도 규격

| 규격 | 속도 | 케이블 |
|:---|:---|:---|
| 10BASE-T | 10 Mbps | Cat3 |
| 100BASE-TX | 100 Mbps | Cat5 |
| 1000BASE-T | 1 Gbps | Cat5e/Cat6 |
| 2.5GBASE-T | 2.5 Gbps | Cat5e |
| 5GBASE-T | 5 Gbps | Cat6 |
| 10GBASE-T | 10 Gbps | Cat6a/Cat7 |

## 5. MAC 주소

NIC 고유의 48비트 물리 주소. 데이터 링크 계층에서 사용한다.

### 구조

```
AA : BB : CC : DD : EE : FF
└─ OUI (24b) ─┘└ 일련번호 (24b) ┘
   제조사 식별    제조사가 부여
```

| 부분 | 크기 | 설명 |
|:---|:---|:---|
| OUI | 24bit | 제조사 코드 (IEEE 할당) |
| 일련번호 | 24bit | 제조사 고유 번호 |

### 주요 OUI 예

| OUI | 제조사 |
|:---|:---|
| 00:00:0C | Cisco |
| 00:50:56 | VMware |
| 08:00:27 | VirtualBox |
| DC:A6:32 | Raspberry Pi |

### 특수 MAC 주소

| 주소 | 용도 |
|:---|:---|
| FF:FF:FF:FF:FF:FF | 브로드캐스트 |
| 01:00:5E:xx:xx:xx | IPv4 멀티캐스트 |
| 33:33:xx:xx:xx:xx | IPv6 멀티캐스트 |

### 확인 명령

```bash
# Windows
ipconfig /all

# Linux / macOS
ip link show
ifconfig
```

## 6. 송수신 과정

### 송신

```
응용 프로그램
    │ 데이터
    ▼
OS 네트워크 스택 (캡슐화)
    │ 프레임
    ▼
NIC
  1. 프레임 버퍼링
  2. CSMA/CD 등 매체 접근 확인
  3. 비트 → 전기 신호 변환
  4. 케이블로 송출
    │
    ▼
   매체 (케이블·전파)
```

### 수신

```
   매체
    │
    ▼
NIC
  1. 신호 수신 → 비트 복원
  2. 목적지 MAC 확인
     - 자신의 MAC → 수신
     - 브로드캐스트 → 수신
     - 타인의 MAC → 폐기
  3. FCS로 오류 검사
  4. 버퍼에 저장 후 인터럽트
    │
    ▼
OS 네트워크 스택 (역캡슐화)
    │
    ▼
응용 프로그램
```

## 7. 물리 계층 장비

| 장비 | 역할 | 특징 |
|:---|:---|:---|
| 리피터 | 신호 증폭 | 거리 연장, 현재 거의 미사용 |
| 허브 | 다중 포트 리피터 | 모든 포트로 플러딩, 반이중 |
| 스위치 | 프레임 전달 (L2) | 콜리전 도메인 분리 |

{{< callout type="warning" >}}
허브는 한 포트로 들어온 신호를 모든 포트로 보내는 탓에 네트워크 부하와 보안 취약점을 유발한다. 현재는 거의 스위치로 대체되었다.
{{< /callout >}}

## 8. 전송 매체

### 유선

| 매체 | 특징 | 거리 |
|:---|:---|:---|
| UTP 케이블 | 구리선 꼬임쌍, 차폐 없음 | 100m |
| STP 케이블 | 차폐 추가 | 100m |
| 광케이블 (SMF) | 싱글모드, 9μm | 수십 km |
| 광케이블 (MMF) | 멀티모드, 50/62.5μm | 수백 m |

### 무선

| 기술 | 주파수 | 범위 | 속도 |
|:---|:---|:---|:---|
| Wi-Fi 6 | 2.4/5 GHz | ~100m | ~9.6 Gbps |
| Bluetooth | 2.4 GHz | ~10m | ~3 Mbps |
| 5G | 600MHz~39GHz | 수 km | ~20 Gbps |

## 핵심 정리

| 개념 | 설명 |
|:---|:---|
| 물리 계층 | 비트 ↔ 신호 변환 |
| 아날로그 / 디지털 | 연속 파형 / 불연속 펄스 |
| NIC | 네트워크 인터페이스 카드 |
| MAC 주소 | 48비트 NIC 고유 주소 |
| 인코딩·변조 | 비트를 신호로 표현 |
| 허브·리피터 | L1 장비 |

{{< callout type="info" >}}
**용어 정리**
- **NIC**: Network Interface Card (랜 카드)
- **MAC**: Media Access Control
- **OUI**: Organizationally Unique Identifier
- **PHY**: 물리 계층 신호 처리 칩
- **MAU**: Medium Attachment Unit, 트랜시버
- **FCS**: Frame Check Sequence, 오류 검출용 체크섬
{{< /callout >}}
