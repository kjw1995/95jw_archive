## 1. 두 주소가 예약된 이유

IP 네트워크의 각 서브넷에는 호스트에 할당할 수 없는 두 개의 주소가 존재한다.

| 주소 | 호스트 ID 패턴 | 역할 |
|:---|:---|:---|
| 네트워크 주소 | 모두 0 | 네트워크 자체를 식별 |
| 브로드캐스트 주소 | 모두 1 | 네트워크 내 모든 호스트에게 전송 |

이 두 주소는 라우팅 테이블, ARP, DHCP 등에서 사용하기 때문에 특정 호스트에 할당하면 통신에 혼선이 발생한다.

{{< callout type="info" >}}
IP 계산에서 "2를 빼는" 이유가 바로 이것이다. 예를 들어 /24에서 호스트 수는 256 - 2 = 254개다.
{{< /callout >}}

## 2. 사용 가능한 호스트 수 공식

```text
사용 가능 호스트 수 = 2^(호스트 비트 수) - 2
```

| 프리픽스 | 호스트 비트 | 전체 주소 | 사용 가능 호스트 |
|:---|:---|:---|:---|
| /24 | 8 | 256 | 254 |
| /25 | 7 | 128 | 126 |
| /26 | 6 | 64 | 62 |
| /27 | 5 | 32 | 30 |
| /28 | 4 | 16 | 14 |
| /30 | 2 | 4 | 2 |

## 3. 네트워크 주소

네트워크 주소는 IP와 서브넷 마스크를 AND 연산하여 구한다. 호스트 ID 비트를 모두 0으로 만든 값이다.

예시: 192.168.1.100/24

```text
IP:      11000000.10101000.00000001.01100100
MASK:    11111111.11111111.11111111.00000000
AND:     11000000.10101000.00000001.00000000
결과:    192.168.1.0
```

### 네트워크 주소의 쓰임

- 라우팅 테이블의 목적지 식별자
- 네트워크 설계 문서 (예: "개발 VLAN 192.168.10.0/24")
- 방화벽/ACL 규칙에서 대상 대역 표기

## 4. 브로드캐스트 주소

브로드캐스트 주소는 네트워크 주소의 호스트 ID 비트를 모두 1로 바꾼 값이다. 이 주소로 전송된 프레임은 해당 네트워크의 모든 호스트가 수신·처리한다.

예시: 192.168.1.0/24의 브로드캐스트는 192.168.1.255

### 다이렉트 vs 리미티드 브로드캐스트

| 구분 | 주소 | 범위 | 라우터 통과 |
|:---|:---|:---|:---|
| 다이렉트 | 192.168.1.255 등 | 특정 네트워크 | 설정에 따라 가능 |
| 리미티드 | 255.255.255.255 | 현재 네트워크 | 불가 |

### 브로드캐스트를 사용하는 프로토콜

- **ARP**: IP → MAC 주소 조회
- **DHCP Discover**: IP가 없는 상태에서 DHCP 서버 탐색
- **NetBIOS**: Windows 네트워크 이름 조회
- **Wake-on-LAN**: 원격 부팅 매직 패킷

## 5. 서브넷팅 시 주소 계산

서브넷을 분할하면 각 서브넷마다 네트워크·브로드캐스트 주소가 새로 정의된다.

### 예제: 192.168.1.0/26

```text
블록 크기 = 256 - 192 = 64
```

| 서브넷 | 네트워크 | 브로드캐스트 | 사용 가능 범위 |
|:---|:---|:---|:---|
| 1 | 192.168.1.0 | 192.168.1.63 | .1 ~ .62 |
| 2 | 192.168.1.64 | 192.168.1.127 | .65 ~ .126 |
| 3 | 192.168.1.128 | 192.168.1.191 | .129 ~ .190 |
| 4 | 192.168.1.192 | 192.168.1.255 | .193 ~ .254 |

### 예제: 192.168.10.65/27의 네트워크·브로드캐스트

```text
블록 크기 = 256 - 224 = 32
블록 경계: 0, 32, 64, 96, ...
65 는 64 ≤ 65 < 96 → 64 블록 소속
```

- 네트워크 주소: 192.168.10.64
- 브로드캐스트 주소: 192.168.10.95
- 사용 가능 호스트: 192.168.10.65 ~ 192.168.10.94 (30개)

## 6. 브로드캐스트 도메인

브로드캐스트 도메인은 하나의 브로드캐스트 프레임이 도달할 수 있는 범위다.

- 스위치는 브로드캐스트를 모든 포트로 복사해 전달한다.
- 라우터는 브로드캐스트를 차단한다.
- 따라서 라우터(또는 VLAN 경계)가 브로드캐스트 도메인을 분리하는 경계선이 된다.

```text
   도메인 1                 도메인 2
  PC1 PC2 PC3             PC4 PC5 PC6
    └── SW1 ──┐         ┌── SW2 ──┘
              │         │
              └── Router ┘
                   ╳
           broadcast 차단
```

{{< callout type="warning" >}}
브로드캐스트 도메인이 지나치게 크면 ARP/DHCP 트래픽이 네트워크 성능을 잠식한다. 중·대규모 환경에서는 VLAN으로 도메인을 잘게 쪼개 관리한다.
{{< /callout >}}

### 브로드캐스트 스톰

스위치 간 루프가 존재하면 브로드캐스트 프레임이 무한히 순환하며 대역폭을 소진한다. 이를 브로드캐스트 스톰이라 하며, STP(Spanning Tree Protocol)로 루프를 감지·차단하여 예방한다.

## 7. 실무 확인 방법

```bash
# Windows
ipconfig

# Linux
ip addr
# inet 192.168.1.10/24 brd 192.168.1.255
```

출력의 `brd` 값이 해당 인터페이스의 브로드캐스트 주소다. 두 호스트가 통신 가능하려면 각자의 네트워크 주소가 일치해야 하며, 다르다면 라우터가 필요하다.

## 핵심 정리

| 개념 | 설명 |
|:---|:---|
| 네트워크 주소 | 호스트 ID 모두 0, 네트워크 자체를 식별 |
| 브로드캐스트 주소 | 호스트 ID 모두 1, 네트워크 내 전체 전송 |
| 사용 가능 호스트 | 2^(호스트 비트) - 2 |
| 다이렉트 브로드캐스트 | 특정 서브넷 대상 (예: 192.168.1.255) |
| 리미티드 브로드캐스트 | 255.255.255.255, 현재 네트워크만 |
| 브로드캐스트 도메인 | 브로드캐스트 프레임이 도달하는 범위 |

{{< callout type="info" >}}
**용어 정리**
- **네트워크 주소**: 네트워크를 식별하는 예약 주소 (호스트 ID = 0)
- **브로드캐스트 주소**: 네트워크 내 모든 호스트에게 전송하는 주소 (호스트 ID = 1)
- **다이렉트 브로드캐스트**: 특정 네트워크를 대상으로 하는 브로드캐스트
- **리미티드 브로드캐스트**: 255.255.255.255, 현재 네트워크에서만 유효
- **브로드캐스트 도메인**: 브로드캐스트가 도달하는 범위
- **브로드캐스트 스톰**: 루프로 브로드캐스트가 과잉 증폭되는 현상
- **STP**: 네트워크 루프를 차단하는 L2 프로토콜
{{< /callout >}}
