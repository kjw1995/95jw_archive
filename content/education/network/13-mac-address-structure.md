## 1. MAC 주소란?

### 정의와 성질

MAC(Media Access Control) 주소는 네트워크 인터페이스(NIC)에 제조 단계에서 새겨지는 물리적 주소이다. 데이터 링크 계층에서 같은 LAN 내 장비를 식별하기 위해 사용된다.

| 항목 | 설명 |
|:---|:---|
| 길이 | 48비트 (6바이트) |
| 표기 | 12자리 16진수 |
| 범위 | 전 세계적으로 유일 |
| 할당 | 제조 시 ROM에 기록 |
| 계층 | 데이터 링크 (2계층) |

---

## 2. 48비트 구조

### OUI와 NIC

MAC 주소는 앞쪽 24비트(OUI)와 뒤쪽 24비트(NIC 고유번호)로 구성된다.

```
 00:23:AE  :  D9:7A:9A
 └─OUI──┘   └─NIC──┘
  24bit       24bit
 제조사ID    일련번호
```

| 부분 | 비트 | 의미 |
|:---|:---|:---|
| OUI (Organizationally Unique Identifier) | 24 | 제조사 식별 코드 (IEEE 할당) |
| NIC 고유번호 | 24 | 제조사가 부여한 개별 번호 |

각 부분은 약 1,677만 개(2^24)의 조합을 제공하며, OUI 한 개로 최대 1,677만 개의 장비를 생산할 수 있다.

### 주요 OUI 예시

| OUI | 제조사 |
|:---|:---|
| 00:00:0C | Cisco |
| 00:50:56 | VMware |
| 08:00:27 | VirtualBox |
| B8:27:EB | Raspberry Pi |
| 3C:5A:B4 | Google |

---

## 3. 표기 방식

같은 MAC 주소도 OS·벤더에 따라 표기법이 다르다.

| 스타일 | 예시 |
|:---|:---|
| 콜론 구분 (Linux) | 00:23:AE:D9:7A:9A |
| 하이픈 구분 (Windows) | 00-23-AE-D9-7A-9A |
| 점 구분 (Cisco) | 0023.AED9.7A9A |
| 구분자 없음 | 0023AED97A9A |

대소문자는 구분하지 않는다.

## 4. 유니캐스트·멀티캐스트·브로드캐스트

### 첫 바이트의 제어 비트

첫 번째 바이트의 **I/G 비트**(Individual/Group)와 **U/L 비트**(Universal/Local)로 주소 유형이 결정된다.

| 비트 | 0일 때 | 1일 때 |
|:---|:---|:---|
| I/G | 유니캐스트 | 멀티캐스트 |
| U/L | 전역(OUI 할당) | 로컬 관리 |

### 주소 유형 분류

| 유형 | 주소 예 | 대상 |
|:---|:---|:---|
| 유니캐스트 | 00:23:AE:D9:7A:9A | 단일 장비 |
| 멀티캐스트 | 01:00:5E:xx:xx:xx | 특정 그룹 (IPv4) |
| 멀티캐스트 | 33:33:xx:xx:xx:xx | 특정 그룹 (IPv6) |
| 브로드캐스트 | FF:FF:FF:FF:FF:FF | 같은 네트워크 전체 |

```
유니캐스트      [A] ──→ [B]

멀티캐스트      [A] ─┬→ [G1]
                    └→ [G2]

브로드캐스트    [A] ─┬→ [B]
                    ├→ [C]
                    └→ [D]
```

{{< callout type="info" >}}
브로드캐스트 프레임은 같은 네트워크(브로드캐스트 도메인)의 모든 장비가 수신하여 처리한다. ARP 요청, DHCP Discover 등이 대표 예이다. 불필요한 브로드캐스트가 많아지면 "브로드캐스트 스톰"으로 성능 저하가 발생할 수 있다.
{{< /callout >}}

## 5. 이더넷 헤더와 FCS

### 헤더 구조

이더넷 프레임에서 MAC 주소는 헤더의 앞부분에 위치한다.

```
┌─────┬─────┬──┬──────┬───┐
│ DST │ SRC │T │ DATA │FCS│
└─────┴─────┴──┴──────┴───┘
 6B    6B   2B 46-1500 4B
```

| 필드 | 크기 | 설명 |
|:---|:---|:---|
| 목적지 MAC | 6 B | 수신자 MAC |
| 출발지 MAC | 6 B | 송신자 MAC |
| 타입 | 2 B | 상위 프로토콜 (0x0800 등) |
| FCS | 4 B | CRC-32 체크섬(트레일러) |

### FCS 검증

송신측은 프레임 내용으로 CRC-32를 계산해 FCS에 담고, 수신측은 동일한 계산을 수행해 값이 일치하는지 확인한다. 불일치 시 프레임을 폐기한다.

ifconfig
```

```

## 7. ARP: IP ↔ MAC 연결

동일 네트워크에서 프레임을 보내려면 목적지 MAC이 필요하다. IP만 알고 있을 때 MAC을 질의하는 프로토콜이 **ARP**이다.

```
1) ARP 요청 (Broadcast)
   "192.168.1.3의 MAC?"

2) ARP 응답 (Unicast)
   "00:23:AE:D9:7A:9C"

3) ARP 캐시에 저장 후 재사용
```

```bash
# ARP 캐시 확인
arp -a
```

## 핵심 정리

| 개념 | 설명 |
|:---|:---|
| MAC 주소 | 48비트 물리 주소, 전 세계 유일 |
| OUI | 앞 24비트, IEEE가 제조사에 할당 |
| NIC 고유번호 | 뒤 24비트, 제조사 개별 번호 |
| I/G 비트 | 유니/멀티캐스트 구분 비트 |
| 브로드캐스트 | FF:FF:FF:FF:FF:FF |
| FCS | CRC-32 기반 프레임 체크섬 |

{{< callout type="info" >}}
**용어 정리**
- **MAC 주소**: Media Access Control Address, 48비트 물리 주소
- **OUI**: IEEE가 제조사에 할당하는 식별 코드
- **유니/멀티/브로드캐스트**: 1:1, 1:그룹, 1:전체 전송 방식
- **FCS**: Frame Check Sequence, CRC-32 오류 검출
- **ARP**: Address Resolution Protocol, IP → MAC 변환
{{< /callout >}}
