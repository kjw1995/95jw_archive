## 1. 컴퓨터 네트워크란?

두 대 이상의 컴퓨터가 연결되어 데이터를 주고받을 수 있는 상태를 네트워크라 한다. 단순한 파일 공유부터 전 세계를 연결하는 인터넷까지 규모는 다양하다.

### 네트워크의 범위별 분류

```
┌────────────────────────────┐
│       인터넷 (Internet)     │
│  ┌──────────────────────┐  │
│  │   WAN (광역 통신망)   │  │
│  │  ┌────────────────┐  │  │
│  │  │ LAN (근거리)    │  │  │
│  │  │ ┌────────────┐ │  │  │
│  │  │ │ PAN (개인)  │ │  │  │
│  │  │ └────────────┘ │  │  │
│  │  └────────────────┘  │  │
│  └──────────────────────┘  │
└────────────────────────────┘
```

| 네트워크 | 범위 | 예시 |
|:---|:---|:---|
| PAN | 개인 영역 (수 m) | 블루투스, 스마트워치 |
| LAN | 건물/사무실 (수백 m) | 회사 내부망 |
| MAN | 도시 (수십 km) | 도시 통신망 |
| WAN | 국가/대륙 (수천 km) | 통신사 망 |
| 인터넷 | 전 세계 | WWW |

---

## 2. 인터넷과 인트라넷

네트워크의 용도와 접근 범위에 따라 두 가지로 구분할 수 있다.

| 구분 | 설명 | 접근 |
|:---|:---|:---|
| 인터넷 (Internet) | 전 세계 네트워크를 연결한 개방망 | 누구나 접근 가능 |
| 인트라넷 (Intranet) | 조직 내부에서만 사용하는 폐쇄망 | 내부 사용자만 |
| 엑스트라넷 (Extranet) | 협력사 등 지정된 외부까지 확장 | 허가된 파트너 |

{{< callout type="info" >}}
**인트라넷**은 사내 포털이나 그룹웨어처럼 외부에서는 접근할 수 없는 내부 네트워크를 가리킨다. TCP/IP 기술을 그대로 쓰되, 방화벽과 VPN으로 외부와 분리한다.
{{< /callout >}}

## 3. 패킷 (Packet)

네트워크를 통해 전송되는 데이터의 작은 조각을 패킷이라 부른다. 큰 데이터를 통째로 보내지 않고 잘게 쪼개어 보낸다.

### 왜 패킷으로 나누는가?

큰 파일을 통째로 보내면 한 사용자가 회선을 독점해 다른 통신을 막는다. 패킷으로 나누면 여러 사용자의 데이터가 번갈아 전송되어 회선을 공평하게 사용할 수 있다.

### 패킷의 구조

```
┌──────┬──────────┬────────┐
│ 헤더 │  데이터  │ 트레일러│
└──────┴──────────┴────────┘
  주소     본문     오류검출
  순번
```

| 영역 | 포함 정보 |
|:---|:---|
| 헤더 | 출발지·목적지 주소, 순서 번호, 프로토콜 |
| 페이로드 | 실제 전송하는 데이터 |
| 트레일러 | CRC 등 오류 검출 코드 |

### 패킷 전송과 재조립

```
송신측            수신측
 ┌─┐┌─┐┌─┐       ┌─┐┌─┐┌─┐
 │1││2││3│  ──▶  │2││3││1│
 └─┘└─┘└─┘       └─┘└─┘└─┘
                    │
                    ▼
               순서 번호로
                 재조립
```

네트워크 상황에 따라 패킷 순서가 뒤바뀌어 도착할 수 있어, 수신측은 헤더의 순서 번호를 보고 원래 순서로 재조립한다.

## 4. 회선 교환 vs 패킷 교환

초기 통신망은 전용 회선을 연결한 뒤 통신이 끝날 때까지 독점하는 **회선 교환 방식**을 사용했다. 인터넷은 **패킷 교환 방식**을 채택해 회선을 효율적으로 공유한다.

| 구분 | 회선 교환 | 패킷 교환 |
|:---|:---|:---|
| 연결 방식 | 통신 전 전용 회선 확보 | 패킷 단위로 경로 선택 |
| 회선 사용 | 통화 동안 독점 | 여러 통신이 공유 |
| 대표 예 | 전통 전화망 (PSTN) | 인터넷, TCP/IP |
| 장점 | 일정한 품질 | 효율·확장성 |
| 단점 | 자원 낭비 | 지연·손실 가능 |

{{< callout type="info" >}}
음성 통화도 현재는 대부분 VoIP(Voice over IP)로 바뀌어 패킷 교환 기반으로 동작한다.
{{< /callout >}}

## 5. 대역폭과 처리량

| 용어 | 의미 | 비유 |
|:---|:---|:---|
| 대역폭 (Bandwidth) | 이론상 최대 전송량 | 도로의 차선 수 |
| 처리량 (Throughput) | 실제 전송량 | 실제 통과 차량 수 |
| 지연 (Latency) | 패킷 왕복 시간 | 출발~도착 소요 시간 |

### 단위 환산

| 단위 | 환산 |
|:---|:---|
| bps | bits per second (기본) |
| Kbps | 1,000 bps |
| Mbps | 1,000,000 bps |
| Gbps | 1,000,000,000 bps |

## 6. 네트워크 구성 요소

### 주요 장비

| 장비 | 역할 | 계층 |
|:---|:---|:---|
| 라우터 | 네트워크 간 경로 설정 | L3 (네트워크) |
| 스위치 | MAC 기반 프레임 전달 | L2 (데이터링크) |
| 허브 | 신호 증폭·분배 | L1 (물리) |
| NIC | 컴퓨터-네트워크 연결 | L1~L2 |

### 토폴로지

| 토폴로지 | 장점 | 단점 |
|:---|:---|:---|
| 버스형 | 구성 단순, 저비용 | 한 곳 장애 시 전체 영향 |
| 스타형 | 장애 격리 용이 | 중앙 장비 의존 |
| 링형 | 균등 접근 | 한 곳 장애 시 영향 큼 |
| 메시형 | 고가용성 | 비용·복잡도 높음 |

## 7. 프로토콜 (Protocol)

프로토콜은 네트워크에서 데이터를 주고받기 위해 합의한 규칙이다. 서로 다른 제조사·운영체제라도 같은 프로토콜을 따르면 통신할 수 있다.

| 프로토콜 | 계층 | 역할 |
|:---|:---|:---|
| HTTP/HTTPS | 응용 | 웹 페이지 전송 |
| DNS | 응용 | 도메인 → IP 변환 |
| TCP | 전송 | 신뢰성 있는 전송 |
| UDP | 전송 | 빠른 비연결 전송 |
| IP | 네트워크 | 주소 지정·라우팅 |
| Ethernet | 데이터링크 | LAN 통신 |

## 핵심 정리

| 용어 | 핵심 |
|:---|:---|
| 네트워크 | 둘 이상 컴퓨터의 연결 |
| 패킷 | 데이터의 작은 조각 |
| 회선 교환 | 전용 회선 확보 방식 |
| 패킷 교환 | 패킷 단위 경로 선택 |
| 대역폭 | 최대 전송 용량 (bps) |
| 프로토콜 | 통신 규칙 |

{{< callout type="info" >}}
**용어 정리**
- **패킷 (Packet)**: 네트워크 계층에서 전송되는 데이터 단위
- **대역폭**: 단위 시간당 전송 가능한 최대 데이터 양
- **지연 시간**: 데이터가 출발지에서 목적지까지 도달하는 데 걸리는 시간
- **인트라넷**: 조직 내부 전용 네트워크
{{< /callout >}}
