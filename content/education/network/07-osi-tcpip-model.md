## 1. OSI 모델의 배경

1980년대 초까지 각 제조사가 독자적인 네트워크 규격을 쓰던 탓에 서로 다른 제조사 장비끼리 통신이 어려웠다. ISO는 이를 해결하기 위해 통신을 7개 계층으로 나눈 **OSI (Open Systems Interconnection)** 참조 모델을 제정했다.

### OSI 7계층

```
┌────────────────────────┐
│ 7 응용 (Application)    │
├────────────────────────┤
│ 6 표현 (Presentation)   │
├────────────────────────┤
│ 5 세션 (Session)        │
├────────────────────────┤
│ 4 전송 (Transport)      │
├────────────────────────┤
│ 3 네트워크 (Network)    │
├────────────────────────┤
│ 2 데이터링크 (Data Link)│
├────────────────────────┤
│ 1 물리 (Physical)       │
└────────────────────────┘
```

암기: "응표세전네데물"

## 2. OSI 각 계층

### 7. 응용 계층

사용자가 네트워크에 접근하는 창구. 애플리케이션 서비스를 제공한다.

대표 프로토콜: HTTP, HTTPS, FTP, SMTP, POP3, IMAP, DNS, SSH, Telnet

### 6. 표현 계층

데이터 형식 변환, 암호화/복호화, 압축/해제를 담당한다.

관련 기술: SSL/TLS, JPEG/PNG, UTF-8, GZIP

### 5. 세션 계층

통신 세션을 설정·유지·종료한다.

| 통신 방식 | 설명 | 예 |
|:---|:---|:---|
| 단방향 (Simplex) | 한 방향 | 라디오·TV |
| 반이중 (Half-Duplex) | 양방향 교대 | 무전기 |
| 전이중 (Full-Duplex) | 양방향 동시 | 전화 |

### 4. 전송 계층

종단 간 신뢰성 있는 데이터 전송을 책임진다. 포트 번호로 애플리케이션을 식별한다.

PDU: 세그먼트 (TCP) / 데이터그램 (UDP)

### 3. 네트워크 계층

논리 주소(IP)를 부여하고, 목적지까지 경로를 결정(라우팅)한다.

PDU: 패킷 / 장비: 라우터 / 프로토콜: IP, ICMP, ARP

### 2. 데이터 링크 계층

같은 네트워크 안에서 물리 주소(MAC)로 프레임을 전달하고 오류를 검출한다.

```
프레임 구조
┌──────┬──────┬────┬────┬────┐
│목적지│출발지│타입│데이 │FCS │
│ MAC │ MAC │    │터  │    │
│ 6B  │ 6B  │ 2B │    │ 4B │
└──────┴──────┴────┴────┴────┘
```

| 하위 계층 | 역할 |
|:---|:---|
| LLC | 흐름·오류 제어 |
| MAC | 물리 주소·매체 접근 제어 |

### 1. 물리 계층

비트를 전기/광/전파 신호로 변환하고 물리 매체를 통해 전달한다.

PDU: 비트 / 장비: 허브, 리피터, 케이블

## 3. TCP/IP 모델

TCP/IP 모델은 실제 인터넷에서 사용하는 4계층 구조다. OSI 모델이 이론적 참조라면 TCP/IP는 구현 표준이다.

### OSI ↔ TCP/IP 매핑

| OSI 7계층 | TCP/IP 4계층 | 대표 프로토콜 |
|:---|:---|:---|
| 7 응용 | 4 응용 | HTTP, DNS, FTP, SMTP, SSH |
| 6 표현 | 4 응용 | TLS, 인코딩 |
| 5 세션 | 4 응용 | 세션 관리 |
| 4 전송 | 3 전송 | TCP, UDP |
| 3 네트워크 | 2 인터넷 | IP, ICMP, ARP |
| 2 데이터링크 | 1 네트워크 접속 | Ethernet, Wi-Fi |
| 1 물리 | 1 네트워크 접속 | 신호·매체 규격 |

### TCP/IP 프로토콜 스위트

```
┌─────────────────────────┐
│ 응용: HTTP DNS FTP SMTP │
├─────────────────────────┤
│ 전송:   TCP    UDP      │
├─────────────────────────┤
│ 인터넷: IP ICMP ARP     │
├─────────────────────────┤
│ 접속: Ethernet Wi-Fi    │
└─────────────────────────┘
```

{{< callout type="info" >}}
**계층별 대표 프로토콜 요약**
- 응용: HTTP, HTTPS, DNS, FTP, SMTP, SSH
- 전송: TCP (신뢰성), UDP (속도)
- 인터넷: IP, ICMP, ARP
- 접속: Ethernet (IEEE 802.3), Wi-Fi (IEEE 802.11)
{{< /callout >}}

## 4. OSI vs TCP/IP

| 항목 | OSI | TCP/IP |
|:---|:---|:---|
| 계층 수 | 7 | 4 |
| 개발 시기 | 1984 | 1970s |
| 주체 | ISO | DARPA |
| 성격 | 이론·참조 | 실용·구현 |
| 현 사용 | 교육·설계 참조 | 인터넷 표준 |

### TCP/IP가 표준이 된 이유

1. 먼저 구현되어 ARPANET에서 실제 동작하고 있었다
2. 하드웨어·운영체제에 독립적
3. 계층이 단순해 구현이 쉬움
4. 인터넷의 성장과 함께 사실상 표준화

## 5. 계층별 PDU

| 계층 | PDU | 구성 |
|:---|:---|:---|
| 응용 | 데이터 (Data) | 순수 페이로드 |
| 전송 | 세그먼트 / 데이터그램 | TCP/UDP 헤더 + 데이터 |
| 네트워크 | 패킷 (Packet) | IP 헤더 + 세그먼트 |
| 데이터링크 | 프레임 (Frame) | ETH 헤더 + 패킷 + FCS |
| 물리 | 비트 (Bit) | 0/1 신호 |

### 캡슐화 흐름

```
응용     [데이터]
             ▼
전송     [TCP][데이터]
             ▼
네트워크 [IP][TCP][데이터]
             ▼
링크    [ETH][IP][TCP][데이터][FCS]
             ▼
물리    10101010...
```

---

## 6. 계층화의 장점

1. **독립성**: 한 계층의 변경이 다른 계층에 영향 없음
2. **표준화**: 계층마다 표준 프로토콜 정의
3. **모듈화**: 문제 발생 시 해당 계층만 진단
4. **역할 분담**: 복잡한 통신 과정을 단순화

### 문제 진단 예

| 증상 | 점검 계층 | 명령 |
|:---|:---|:---|
| 케이블 불량 | L1 | LED·케이블 교체 |
| IP 설정 문제 | L3 | `ipconfig`, ping 게이트웨이 |
| 포트 차단 | L4 | `netstat`, `telnet` |
| 브라우저 오류 | L7 | 개발자 도구 |

## 7. 계층별 장비

| 계층 | 장비 |
|:---|:---|
| L1 | 허브, 리피터 |
| L2 | 스위치, 브리지 |
| L3 | 라우터, L3 스위치 |
| L4~L7 | 방화벽, 로드밸런서, WAF |

## 핵심 정리

| 개념 | 설명 |
|:---|:---|
| OSI 모델 | 이론적 7계층 참조 모델 |
| TCP/IP 모델 | 실용적 4계층 구현 모델 |
| 계층화 | 독립성·표준화·모듈화 |
| PDU | 각 계층의 데이터 단위 |
| 캡슐화 | 헤더 추가 과정 |
| 역캡슐화 | 헤더 제거 과정 |

### OSI 7계층 요약

| 계층 | 이름 | PDU | 장비 |
|:---|:---|:---|:---|
| 7 | 응용 | 데이터 | - |
| 6 | 표현 | 데이터 | - |
| 5 | 세션 | 데이터 | - |
| 4 | 전송 | 세그먼트 | - |
| 3 | 네트워크 | 패킷 | 라우터 |
| 2 | 데이터링크 | 프레임 | 스위치 |
| 1 | 물리 | 비트 | 허브 |

{{< callout type="info" >}}
**용어 정리**
- **OSI**: Open Systems Interconnection
- **ISO**: International Organization for Standardization
- **TCP/IP**: Transmission Control Protocol / Internet Protocol
- **PDU**: Protocol Data Unit
- **캡슐화**: 상위 계층 데이터에 헤더를 추가하는 과정
{{< /callout >}}
