## 1. 라우터의 역할

라우터는 OSI 3계층(네트워크 계층)에서 동작하며 서로 다른 IP 네트워크를 연결하는 장비다. 패킷의 목적지 IP 주소를 확인하고, 라우팅 테이블을 참조해 다음 전달 경로를 결정한다.

주요 기능은 다음과 같다.

- 네트워크 간 패킷 전달
- 최적 경로 선택(라우팅)
- 브로드캐스트 도메인 분리
- NAT, 방화벽 기능

스위치가 있어도 서로 다른 네트워크 간에는 통신할 수 없다. 스위치는 L2 장비라 IP 주소를 해석하지 않기 때문이다. 라우터가 있어야 192.168.1.0/24와 192.168.2.0/24 같은 서로 다른 네트워크가 연결된다.

## 2. 허브·스위치·라우터 비교

| 항목 | 허브 | 스위치 | 라우터 |
|:---|:---|:---|:---|
| OSI 계층 | 1 (물리) | 2 (데이터링크) | 3 (네트워크) |
| 식별자 | 없음 | MAC 주소 | IP 주소 |
| 전송 방식 | 모든 포트로 복사 | 목적지 포트로 전달 | 경로 선택 후 전달 |
| 브로드캐스트 | 분리 불가 | 분리 불가 (VLAN 예외) | 분리 가능 |
| 주 용도 | 신호 증폭 | LAN 내부 연결 | 네트워크 간 연결 |

## 3. 기본 게이트웨이

기본 게이트웨이는 호스트가 자신의 서브넷 밖으로 나갈 때 패킷을 맡기는 라우터 IP 주소다. 호스트는 목적지가 같은 서브넷이 아닐 경우 게이트웨이로 패킷을 보낸다.

```text
    PC                같은 서브넷?
192.168.1.10   ┌──→  예 → 직접 전송
               │
 목적지 확인 ──┤
               │
               └──→  아니오 → 게이트웨이로 전송
                               192.168.1.1
```

### 설정 확인

```bash
# Windows
ipconfig
# IPv4 주소 : 192.168.1.10
# 서브넷 마스크 : 255.255.255.0
# 기본 게이트웨이 : 192.168.1.1

# Linux
ip route show
# default via 192.168.1.1 dev eth0
```

## 4. 라우팅과 라우팅 테이블

라우팅은 목적지까지 패킷을 전달할 경로를 선택하는 과정이다. 라우터는 라우팅 테이블을 보고 전달할 인터페이스와 다음 홉을 결정한다.

라우팅 테이블의 주요 열은 다음과 같다.

| 항목 | 설명 |
|:---|:---|
| Destination | 목적지 네트워크 |
| Netmask | 서브넷 마스크 |
| Gateway | 다음 홉 라우터 (직접 연결은 on-link) |
| Interface | 송출 인터페이스 |
| Metric | 경로 비용 (작을수록 우선) |

### 라우팅 테이블 예시

```bash
# Linux
$ ip route show
default via 192.168.1.1 dev eth0 metric 100
192.168.1.0/24 dev eth0 proto kernel scope link src 192.168.1.10
192.168.2.0/24 via 192.168.1.254 dev eth0 metric 20
```

### Longest Prefix Match

라우터는 여러 경로가 매칭될 때 **프리픽스가 가장 긴 경로**를 선택한다.

| 목적지 후보 | 프리픽스 | 우선순위 |
|:---|:---|:---|
| 192.168.1.50/32 | /32 | 1 (선택) |
| 192.168.1.0/24 | /24 | 2 |
| 192.168.0.0/16 | /16 | 3 |
| 0.0.0.0/0 | /0 | 4 (기본 경로) |

192.168.1.50으로 향하는 패킷은 /32 호스트 경로가 있으면 그 쪽으로, 없으면 /24, /16 순으로 탐색하며 매칭되는 것이 없으면 기본 경로(0.0.0.0/0)로 보낸다.

## 5. 정적 라우팅 vs 동적 라우팅

| 구분 | 정적 라우팅 | 동적 라우팅 |
|:---|:---|:---|
| 경로 설정 | 관리자 수동 설정 | 프로토콜이 자동 학습 |
| 장애 대응 | 수동 변경 필요 | 자동 우회 |
| 리소스 | 부하 적음 | CPU·대역폭 사용 |
| 관리 범위 | 소규모 | 중·대규모 |
| 예측 가능성 | 높음 | 중간 |

### 정적 경로 추가 예시

```bash
# Linux
sudo ip route add 192.168.2.0/24 via 192.168.1.254 dev eth0

# Windows (관리자)
route add 192.168.2.0 mask 255.255.255.0 192.168.1.254 -p

# Cisco IOS
Router(config)# ip route 192.168.2.0 255.255.255.0 192.168.1.254
```

## 6. 주요 동적 라우팅 프로토콜

### 분류

| 구분 | 방식 | 대표 프로토콜 |
|:---|:---|:---|
| IGP 거리 벡터 | 이웃과 경로 교환 | RIP |
| IGP 링크 상태 | 전체 토폴로지 계산 | OSPF, IS-IS |
| IGP 하이브리드 | 두 방식 혼합 | EIGRP |
| EGP | AS 간 경로 | BGP |

### RIP

- 거리 벡터, 홉 카운트 사용
- 최대 15홉, 16은 도달 불가
- 30초마다 테이블 갱신
- 설정 단순, 소규모에 적합, 수렴 느림

### OSPF

- 링크 상태, 다익스트라 알고리즘
- Cost(대역폭 기반) 메트릭
- Area 계층으로 확장성 확보
- 빠른 수렴, 중·대규모에 적합

### BGP

- 경로 벡터, AS 간 라우팅
- TCP 179 포트 사용
- 정책 기반 경로 선택 (Local Preference, AS Path, MED 등)
- 인터넷 백본의 표준

### 프로토콜 비교

| 항목 | RIP | OSPF | EIGRP | BGP |
|:---|:---|:---|:---|:---|
| 방식 | DV | LS | 하이브리드 | PV |
| 메트릭 | 홉 수 | Cost | 복합 | Path |
| 최대 홉 | 15 | 무제한 | 255 | 무제한 |
| 수렴 속도 | 느림 | 빠름 | 빠름 | 느림 |
| VLSM | v2만 | 지원 | 지원 | 지원 |
| 용도 | 소규모 | 중·대규모 | 중·대규모 | AS 간 |

### 규모별 선택 가이드

| 네트워크 규모 | 권장 |
|:---|:---|
| 소규모 (라우터 10대 이하) | 정적 라우팅, RIPv2 |
| 중규모 (10~100대) | OSPF, EIGRP |
| 대규모 (100대 이상) | OSPF 계층 구조, IS-IS |
| 인터넷 연결 | BGP |

## 7. 트러블슈팅 절차

연결 문제를 진단할 때는 다음 순서로 확인한다.

1. **로컬 네트워크 확인**: `ping 게이트웨이`
2. **라우팅 테이블 확인**: `route print` / `ip route show`
3. **경로 추적**: `tracert` / `traceroute`
4. **라우터 설정 확인**: 인터페이스 상태, 라우팅 프로토콜 이웃 관계

```bash
# 경로 추적
traceroute 192.168.3.10
 1  192.168.1.1     1ms   ← 기본 게이트웨이
 2  192.168.2.2     5ms   ← 중간 라우터
 3  * * *                 ← 여기서 막힘
```

`* * *`이 나타나는 홉의 라우터에 해당 목적지 경로가 설정되어 있는지 점검한다.

{{< callout type="warning" >}}
트러블슈팅은 "가까운 곳부터 먼 곳으로" 확인한다. 자기 IP → 게이트웨이 → 라우터 간 링크 → 목적지 순이다.
{{< /callout >}}

## 8. 자주 쓰는 명령어

```bash
# Windows
ipconfig                              # 인터페이스 정보
route print                           # 라우팅 테이블
tracert <host>                        # 경로 추적

# Linux
ip addr show                          # 인터페이스 정보
ip route show                         # 라우팅 테이블
traceroute <host>                     # 경로 추적

# Cisco IOS
show ip interface brief
show ip route
show ip protocols
show ip ospf neighbor
```

## 핵심 정리

| 개념 | 설명 |
|:---|:---|
| 라우터 | 서로 다른 네트워크를 연결하는 3계층 장비 |
| 라우팅 | 최적 경로를 선택해 패킷을 전달하는 과정 |
| 기본 게이트웨이 | 호스트가 서브넷 밖으로 나갈 때 사용하는 라우터 IP |
| 라우팅 테이블 | 목적지 네트워크와 경로 정보의 집합 |
| Longest Prefix Match | 가장 긴 프리픽스 매칭을 우선 |
| 정적 라우팅 | 관리자가 수동 설정 |
| 동적 라우팅 | 프로토콜이 자동 학습 |
| RIP / OSPF / BGP | 거리 벡터 / 링크 상태 / AS 간 프로토콜 |

{{< callout type="info" >}}
**용어 정리**
- **라우터 (Router)**: L3에서 서로 다른 네트워크를 연결하는 장비
- **라우팅 (Routing)**: 목적지까지의 경로를 선택하는 과정
- **기본 게이트웨이 (Default Gateway)**: 외부 네트워크로 나가는 기본 경로
- **라우팅 테이블**: 경로 정보를 저장한 테이블
- **홉 (Hop)**: 경유하는 라우터의 수
- **메트릭 (Metric)**: 경로 비용, 작을수록 우선
- **AS (Autonomous System)**: 단일 관리 주체의 네트워크 집합
- **IGP / EGP**: AS 내부용 / AS 간 라우팅 프로토콜
{{< /callout >}}
