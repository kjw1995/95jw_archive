## 1. IP 주소 클래스의 개념

IP 주소 클래스는 32비트 주소 공간을 네트워크 규모에 따라 A~E 다섯 등급으로 분류하는 방식이다. 네트워크 ID와 호스트 ID가 차지하는 비트 수가 클래스마다 고정되어 있어, 첫 옥텟만으로도 어떤 용도의 네트워크인지 판별할 수 있다.

| 구분 | 역할 |
|:---|:---|
| 네트워크 ID | 어느 네트워크에 속하는지 식별 |
| 호스트 ID | 네트워크 내에서 개별 장치를 식별 |

{{< callout type="info" >}}
A~C는 일반 통신용, D는 멀티캐스트, E는 연구·예약이다. 공인 IP 할당은 A~C에서만 이루어진다.
{{< /callout >}}

## 2. 클래스 식별 비트 패턴

각 클래스는 첫 번째 옥텟의 상위 비트 패턴으로 구분된다. 상위 비트가 고정되기 때문에 첫 옥텟의 10진수 값만 확인해도 클래스를 알 수 있다.

```text
A: 0xxxxxxx    →  1 ~ 126
B: 10xxxxxx    →  128 ~ 191
C: 110xxxxx    →  192 ~ 223
D: 1110xxxx    →  224 ~ 239
E: 1111xxxx    →  240 ~ 255
```

| 첫 옥텟 | 클래스 | 예시 |
|:---|:---|:---|
| 1 ~ 126 | A | 10.0.0.1, 126.1.2.3 |
| 128 ~ 191 | B | 172.16.0.1, 150.10.5.7 |
| 192 ~ 223 | C | 192.168.1.1, 220.100.50.1 |
| 224 ~ 239 | D | 224.0.0.1 (멀티캐스트) |
| 240 ~ 255 | E | 240.0.0.1 (예약) |

0.x.x.x는 현재 네트워크 표기, 127.x.x.x는 루프백으로 예약되어 있다.

## 3. 클래스별 주소 구조

### A 클래스

- 구조: 네트워크 8비트 + 호스트 24비트
- 범위: 1.0.0.0 ~ 126.255.255.255
- 기본 마스크: 255.0.0.0 (/8)
- 네트워크 수: 126개 (0, 127 제외)
- 호스트 수: 16,777,214개 (2^24 - 2)

```text
 0xxxxxxx . xxxxxxxx . xxxxxxxx . xxxxxxxx
 ├──NET──┤ ├──────── HOST ──────────────┤
```

### B 클래스

- 구조: 네트워크 16비트 + 호스트 16비트
- 범위: 128.0.0.0 ~ 191.255.255.255
- 기본 마스크: 255.255.0.0 (/16)
- 네트워크 수: 16,384개 (2^14)
- 호스트 수: 65,534개 (2^16 - 2)

```text
 10xxxxxx . xxxxxxxx . xxxxxxxx . xxxxxxxx
 ├──── NETWORK ─────┤ ├──── HOST ───────┤
```

### C 클래스

- 구조: 네트워크 24비트 + 호스트 8비트
- 범위: 192.0.0.0 ~ 223.255.255.255
- 기본 마스크: 255.255.255.0 (/24)
- 네트워크 수: 2,097,152개 (2^21)
- 호스트 수: 254개 (2^8 - 2)

```text
 110xxxxx . xxxxxxxx . xxxxxxxx . xxxxxxxx
 ├─────── NETWORK ───────────┤ ├── HOST ─┤
```

### D 클래스와 E 클래스

- D: 224.0.0.0 ~ 239.255.255.255, 멀티캐스트 전용
- E: 240.0.0.0 ~ 255.255.255.255, 실험·예약

주요 멀티캐스트 주소는 다음과 같다.

| 주소 | 용도 |
|:---|:---|
| 224.0.0.1 | 모든 호스트 |
| 224.0.0.2 | 모든 라우터 |
| 224.0.0.5 | OSPF 라우터 |
| 224.0.0.9 | RIPv2 라우터 |
| 239.x.x.x | 조직 내부 멀티캐스트 |

## 4. 클래스 비교와 호스트 수 계산

### 전체 비교

| 클래스 | 첫 옥텟 | NET 비트 | HOST 비트 | 기본 마스크 |
|:---|:---|:---|:---|:---|
| A | 1 ~ 126 | 8 | 24 | 255.0.0.0 (/8) |
| B | 128 ~ 191 | 16 | 16 | 255.255.0.0 (/16) |
| C | 192 ~ 223 | 24 | 8 | 255.255.255.0 (/24) |
| D | 224 ~ 239 | - | - | 멀티캐스트 |
| E | 240 ~ 255 | - | - | 예약 |

### 호스트 수 계산 원리

```text
사용 가능 호스트 = 2^(HOST 비트) - 2
```

호스트 ID가 모두 0인 주소는 네트워크 주소, 모두 1인 주소는 브로드캐스트 주소로 예약되어 호스트에 할당할 수 없다. 그래서 항상 2를 뺀다.

## 5. 사설 IP 주소 (RFC 1918)

외부 인터넷에 노출되지 않는 내부용 주소 대역이다. 같은 사설 IP를 여러 조직이 중복해 쓸 수 있으며, 인터넷과 통신할 때는 NAT를 거친다.

| 대역 | CIDR | 용도 |
|:---|:---|:---|
| 10.0.0.0 ~ 10.255.255.255 | 10.0.0.0/8 | 대규모 기업 |
| 172.16.0.0 ~ 172.31.255.255 | 172.16.0.0/12 | 중규모 조직 |
| 192.168.0.0 ~ 192.168.255.255 | 192.168.0.0/16 | 가정, SOHO |

## 6. 특수 IP 주소

| 주소 | 용도 |
|:---|:---|
| 0.0.0.0 | 현재 네트워크, 기본 경로 |
| 127.0.0.0/8 | 루프백 (자기 자신, localhost) |
| 169.254.0.0/16 | 링크-로컬 (APIPA) |
| 255.255.255.255 | 제한된 브로드캐스트 |

## 7. 클래스풀에서 클래스리스(CIDR)로

클래스풀 방식은 IP 주소를 크게 낭비한다. 예를 들어 500대가 필요한 조직에 C 클래스는 부족하고 B 클래스를 주면 약 99%가 남는다.

이를 해결하기 위해 등장한 것이 CIDR(Classless Inter-Domain Routing)이다. CIDR는 클래스 경계를 버리고 `IP/프리픽스` 형식으로 마스크 길이를 자유롭게 지정한다. 500대에는 /23 (510개)로 할당하여 약 98% 효율을 얻을 수 있다.

{{< callout type="info" >}}
현대 인터넷은 전적으로 CIDR 기반이다. 클래스 개념은 기본 마스크 판단이나 오래된 문서를 읽을 때 참고 지식으로만 쓰인다.
{{< /callout >}}

## 핵심 정리

| 개념 | 설명 |
|:---|:---|
| A 클래스 | 1~126, 대규모 네트워크, 호스트 약 1,677만 대 |
| B 클래스 | 128~191, 중규모 네트워크, 호스트 약 6만 5천 대 |
| C 클래스 | 192~223, 소규모 네트워크, 호스트 254대 |
| D 클래스 | 224~239, 멀티캐스트용 |
| E 클래스 | 240~255, 연구·예약용 |
| 사설 IP | 10.x, 172.16~31.x, 192.168.x |
| 루프백 | 127.0.0.1, localhost |
| CIDR | 클래스 없이 프리픽스 길이로 네트워크 크기 지정 |

{{< callout type="info" >}}
**용어 정리**
- **클래스 (Class)**: IP 주소를 네트워크 규모별로 A~E로 분류한 체계
- **네트워크 ID**: 네트워크를 식별하는 비트
- **호스트 ID**: 네트워크 내 개별 장치를 식별하는 비트
- **사설 IP**: 내부 네트워크 전용 주소 (RFC 1918)
- **루프백**: 자기 자신을 가리키는 주소 (127.0.0.1)
- **CIDR**: 클래스 경계를 없앤 프리픽스 기반 표기법
{{< /callout >}}
