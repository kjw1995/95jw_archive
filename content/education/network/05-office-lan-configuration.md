## 1. 가정과 회사 네트워크의 차이

회사 네트워크는 가정과 달리 외부에 공개하는 서비스와 내부 전용 자원을 함께 운영하기 때문에 구조가 복잡하다.

| 구분 | 가정 | 회사 |
|:---|:---|:---|
| 규모 | ~10대 | 수십~수천 대 |
| 서버 | 없음 | 웹·메일·DNS 등 |
| 보안 영역 | 단일 | DMZ·내부망 분리 |
| 관리 | 개인 | 전담 인력 |
| 장비 | 공유기 1대 | 라우터·스위치·방화벽 |

### 전체 구조 개요

```
     인터넷
       │
   ┌───┴───┐
   │ 방화벽 │
   └───┬───┘
  ┌────┼────┐
  ▼    ▼    ▼
┌───┐┌───┐┌────┐
│DMZ││서버││내부│
│   ││ 팜 ││ 망 │
└───┘└───┘└────┘
```

## 2. DMZ (Demilitarized Zone)

외부 인터넷과 내부 LAN 사이의 완충 영역이다. 외부 공개가 필요한 서버를 두고, 내부망과 분리해 공격 피해 확산을 막는다.

### 이중 방화벽 구성

```
   인터넷
     │
 ┌───┴───┐
 │외부 FW │  1차 방어
 └───┬───┘
     │
   ┌─┴─┐
   │DMZ│
   └─┬─┘
 ┌───┴───┐
 │내부 FW │  2차 방어
 └───┬───┘
     │
   ┌─┴─┐
   │LAN│
   └───┘
```

### 방화벽 정책 원칙

| 방향 | 허용 여부 |
|:---|:---|
| 인터넷 → DMZ | 지정 포트만 허용 (80, 443 등) |
| DMZ → 내부망 | 원칙적 거부 |
| 내부망 → DMZ | 관리 목적 허용 |
| 내부망 → 인터넷 | 업무용 포트 허용 |

{{< callout type="warning" >}}
DMZ의 서버가 침해되어도 내부망으로 직접 들어오지 못하도록 **DMZ→내부망 트래픽은 기본 차단**해야 한다.
{{< /callout >}}

## 3. DMZ에 배치하는 서버

### 대표 서버와 포트

| 서버 | 역할 | 주요 포트 |
|:---|:---|:---|
| 웹 서버 | HTTP/HTTPS 서비스 | 80, 443 |
| 메일 서버 | SMTP/POP3/IMAP | 25, 110, 143, 587 |
| DNS 서버 | 도메인 해석 | 53 |
| FTP 서버 | 파일 전송 | 20, 21 |
| SSH 서버 | 원격 접속 | 22 |

### 메일 전송 흐름

```
발신자                           수신자
  │  SMTP   ┌──────┐   SMTP    │
  ├────────▶│ 메일 │────────▶  │
  │         │ 서버 │             │
  │         └──┬───┘             │
  │            │ POP3/IMAP       │
  │            └───────────────▶ ▼
```

## 4. 서버 운영 방식

### 온프레미스

자체 시설에 서버를 두고 직접 운영한다.

| 장점 | 단점 |
|:---|:---|
| 데이터 직접 통제 | 초기 비용 큼 |
| 커스터마이징 자유 | 유지보수 인력 필요 |
| 네트워크 지연 최소 | 확장·재해복구 어려움 |

### 클라우드

인터넷으로 IT 리소스를 임대해 사용한다.

| 모델 | 관리 범위 | 예시 |
|:---|:---|:---|
| IaaS | 인프라 대여 | AWS EC2, Azure VM |
| PaaS | 플랫폼 제공 | Elastic Beanstalk, Heroku |
| SaaS | 완성 앱 제공 | M365, Salesforce |

| 장점 | 단점 |
|:---|:---|
| 초기 비용 낮음 | 장기 비용 상승 가능 |
| 탄력적 확장 | 인터넷 의존 |
| 글로벌 배포 | 데이터 주권·벤더 종속 |

### 하이브리드 클라우드

```
 ┌────────┐        ┌────────┐
 │온프레미스│◀─VPN─▶│ 클라우드 │
 │        │        │        │
 │핵심 DB  │        │웹서버   │
 │레거시   │        │백업     │
 └────────┘        └────────┘
```

## 5. 회사 네트워크 장비

### 엔터프라이즈 장비 vs 가정용

| 구분 | 가정용 | 엔터프라이즈 |
|:---|:---|:---|
| 라우터 | 공유기 내장 | 전용 라우터 (Cisco 등) |
| 스위치 | 4포트 내장 | 24/48포트 |
| 방화벽 | 기본 | UTM/NGFW |
| 무선 | 공유기 AP | 컨트롤러 + AP |
| 관리 | 웹 GUI | CLI·중앙 관리 |

### 주요 벤더

| 벤더 | 분야 |
|:---|:---|
| Cisco | 종합 네트워크 |
| Juniper | 라우터·스위치 |
| Fortinet | 보안 (UTM) |
| Aruba (HPE) | 무선 AP |
| Palo Alto | NGFW |

## 6. 사내 네트워크 설계

### VLAN

물리적 스위치는 하나여도 논리적으로 네트워크를 분리해 브로드캐스트 도메인과 보안 경계를 나눈다.

```
┌──────────────────────────┐
│        L2 스위치          │
│ ┌────┐  ┌────┐  ┌────┐  │
│ │VLAN│  │VLAN│  │VLAN│  │
│ │ 10 │  │ 20 │  │ 30 │  │
│ │영업 │  │개발 │  │인사 │  │
│ └────┘  └────┘  └────┘  │
└──────────────────────────┘
```

### IP 체계 예시

```
10.0.0.0/8
├─ 10.10.0.0/16  본사
│  ├─ 10.10.1.0/24  1층 (VLAN 10)
│  ├─ 10.10.2.0/24  2층 (VLAN 20)
│  ├─ 10.10.100.0/24  서버팜
│  └─ 10.10.200.0/24  DMZ
├─ 10.20.0.0/16  지사 A
└─ 10.30.0.0/16  지사 B
```

### 이중화 (High Availability)

라우터는 HSRP/VRRP, 스위치는 링크 집합, 서버는 이중 NIC로 단일 장애점을 없앤다.

## 7. 실제 구성 예시

### 소규모 (SOHO)

```
인터넷─[FW/RT]─[스위치]─┬─ PC 5대
                      ├─ 프린터
                      └─ Wi-Fi AP
```

### 중규모 (50~100인)

```
인터넷
  │
[방화벽]
  ├─[DMZ]─ 웹·메일 서버
  └─[코어 스위치]
     ├─ 서버팜 (DB·파일)
     ├─ 층별 L2 스위치
     └─ 무선 컨트롤러·AP
```

## 핵심 정리

| 구성 요소 | 역할 |
|:---|:---|
| DMZ | 외부 공개 서버용 완충 영역 |
| 방화벽 | 트래픽 정책·침입 차단 |
| VLAN | 논리적 네트워크 분리 |
| 온프레미스 | 자체 시설 직접 운영 |
| 클라우드 | 임대형 IT 리소스 |
| 이중화 | 단일 장애점 제거 |

{{< callout type="info" >}}
**용어 정리**
- **SOHO**: Small Office/Home Office (소규모 사업장)
- **DMZ**: Demilitarized Zone (비무장 지대)
- **데이터 센터**: 서버·네트워크 장비를 집중 운영하는 시설
- **클라우드**: 인터넷으로 제공되는 컴퓨팅 서비스
- **온프레미스**: 자체 시설에 직접 구축한 시스템
- **VLAN**: Virtual LAN, 논리적 LAN 분할
{{< /callout >}}
