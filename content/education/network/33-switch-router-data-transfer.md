## 1. 역할 구분

스위치와 라우터는 동작하는 계층과 참조하는 주소가 다르다. 스위치는 같은 LAN 안에서 MAC 주소 기반으로 프레임을 전달하고, 라우터는 서로 다른 네트워크 사이에서 IP 주소 기반으로 패킷을 라우팅한다.

| 항목 | 스위치(L2) | 라우터(L3) |
|:---|:---|:---|
| OSI 계층 | 물리 + 데이터링크 | 물리 + 데이터링크 + 네트워크 |
| 주소 기반 | MAC | IP |
| 참조 테이블 | MAC 주소 테이블 | 라우팅 테이블 |
| PDU | 프레임 | 패킷 |
| 사용 범위 | 같은 LAN | 서로 다른 네트워크 |
| 브로드캐스트 도메인 | 공유 | 분리 |

## 2. 네트워크 구성 예시

```
[PC A]          [라우터 A]       [라우터 B]       [웹 서버]
192.168.1.10    WAN:172.16.0.1   WAN:172.16.0.2   192.168.10.100
     │               │               │               │
  [스위치A]──LAN1──┤               ├──LAN2──[스위치B]
                   192.168.1.0/24  192.168.10.0/24
```

## 3. 스위치의 동작

### 처리 단계

```
포트 1 → [물리]
           │ 전기 신호 → 비트
           ▼
         [데이터링크]
           │
           ▼
  ┌─────────────────────┐
  │ 1. 출발지 MAC 학습  │
  ├─────────────────────┤
  │ 2. 목적지 MAC 조회  │
  ├─────────────────────┤
  │ 3. 포워딩 결정      │
  └─────────────────────┘
           │
           ▼
         [물리]
           │ 전기 신호
           ▼
         포트 3
```

### MAC 주소 테이블

스위치는 프레임을 수신할 때마다 출발지 MAC과 수신 포트를 기록한다. 이후 목적지 MAC을 이 테이블에서 찾아 해당 포트로만 전달한다.

| 포트 | MAC 주소 | 에이징 |
|:---|:---|:---|
| 1 | AA:BB:CC:DD:EE:FF | 300s |
| 2 | 11:22:33:44:55:66 | 300s |
| 3 | 77:88:99:AA:BB:CC | 300s |

### 전달 방식

| 방식 | 조건 | 동작 |
|:---|:---|:---|
| 유니캐스트 | 목적지 MAC이 테이블에 있음 | 해당 포트로만 전달 |
| 플러딩 | 목적지 MAC이 테이블에 없음 | 출발지 제외 모든 포트로 전달 |
| 브로드캐스트 | 목적지 MAC = FF:FF:FF:FF:FF:FF | 모든 포트로 전달 |

{{< callout type="info" >}}
3계층(L3) 스위치는 L2 스위칭에 라우팅 기능을 합친 장비다. VLAN 간 라우팅을 하드웨어로 수행하여 일반 라우터보다 훨씬 빠르며, 내부 네트워크 백본으로 널리 쓰인다.
{{< /callout >}}

## 4. 라우터의 동작

라우터는 이더넷 프레임을 역캡슐화해 IP 헤더를 확인한 뒤, 라우팅 테이블을 조회해 다음 홉을 결정한다. TTL을 1 감소시키고, NAT가 필요하면 주소를 변환한 후 새 이더넷 헤더로 재캡슐화하여 내보낸다.

### 처리 단계

```
입력 포트
   │ 전기 신호 → 비트
   ▼
[데이터링크]
  - 목적지 MAC 확인
  - 이더넷 헤더 제거
   ▼
[네트워크]
  - 목적지 IP 확인
  - 라우팅 테이블 조회
  - TTL -1
  - NAT 처리 (필요 시)
   ▼
[데이터링크]
  - 다음 홉 MAC 조회(ARP)
  - 새 이더넷 헤더 추가
   ▼
출력 포트
   │ 전기 신호
   ▼
```

### 라우팅 테이블

| 목적지 | 넷마스크 | 게이트웨이 | 인터페이스 |
|:---|:---|:---|:---|
| 192.168.1.0 | /24 | 직접 연결 | eth0 |
| 192.168.10.0 | /24 | 172.16.0.2 | eth1 |
| 172.16.0.0 | /16 | 직접 연결 | eth1 |
| 0.0.0.0 | /0 | 172.16.0.254 | eth1 |

가장 긴 넷마스크가 일치하는 항목(Longest Prefix Match)이 선택된다. 일치하는 항목이 없으면 기본 경로(`0.0.0.0/0`)가 사용된다.

### TTL이 감소하는 이유

라우팅 루프가 발생해도 TTL이 0에 도달하면 패킷이 폐기되므로 네트워크가 마비되지 않는다. 라우터는 TTL을 1 감소시키고, 0이 된 패킷은 ICMP "Time Exceeded"로 알린다. traceroute는 이를 역이용해 경로상의 라우터를 찾는다.

## 5. 구간별 헤더 변화

### 구간 1: PC A → 라우터 A (LAN1)

```
Ethernet  src MAC: PC A
          dst MAC: 라우터 A (LAN)
IP        src IP : 192.168.1.10
          dst IP : 192.168.10.100
```

### 구간 2: 라우터 A → 라우터 B (WAN)

```
Ethernet  src MAC: 라우터 A (WAN)
          dst MAC: 라우터 B (WAN)
IP        src IP : 172.16.0.1  ← NAT
          dst IP : 192.168.10.100
```

### 구간 3: 라우터 B → 웹 서버 (LAN2)

```
Ethernet  src MAC: 라우터 B (LAN)
          dst MAC: 웹 서버
IP        src IP : 172.16.0.1
          dst IP : 192.168.10.100
```

핵심 패턴:
- **MAC 주소는 매 홉마다 교체된다** (항상 다음 홉으로 향하는 주소).
- **IP 주소는 종단 간 유지된다** (NAT가 출발지 IP를 변경할 수 있음).
- TTL은 라우터를 통과할 때마다 1씩 감소한다.

## 6. NAT

### 필요성

IPv4 주소가 한정적이어서, 집이나 사무실 내부는 사설 IP(192.168.x.x, 10.x.x.x, 172.16.x.x)를 사용하고 외부로 나갈 때 공인 IP로 변환한다.

### PAT (포트 주소 변환)

가장 일반적인 NAT 방식으로, 하나의 공인 IP에 내부 여러 단말을 포트 번호로 구분해 매핑한다.

| 내부 | 외부 |
|:---|:---|
| 192.168.1.10:54321 | 203.0.113.10:54321 |
| 192.168.1.20:54322 | 203.0.113.10:54322 |
| 192.168.1.30:54323 | 203.0.113.10:54323 |

```bash
# Linux iptables로 PAT 설정
sudo sysctl -w net.ipv4.ip_forward=1
sudo iptables -t nat -A POSTROUTING \
  -o eth1 -j MASQUERADE
```

ip route show

## 핵심 정리

| 항목 | 내용 |
|:---|:---|
| 스위치 | L2, MAC 기반 프레임 스위칭, 학습/플러딩/브로드캐스트 |
| 라우터 | L3, IP 기반 패킷 포워딩, 라우팅 테이블·TTL·NAT |
| L3 스위치 | L2 + L3 하드웨어 라우팅, VLAN 간 통신 |
| MAC vs IP | MAC은 매 홉, IP는 종단 간 |

{{< callout type="info" >}}
**용어 정리**
- **MAC 주소 테이블**: 스위치가 포트와 MAC의 대응을 저장하는 테이블
- **라우팅 테이블**: 목적지 네트워크별 다음 홉 정보를 저장하는 테이블
- **NAT / PAT**: 사설 IP와 공인 IP 간 주소(와 포트) 변환
- **TTL**: 라우터를 지나며 감소하는 패킷 수명 카운터
- **Longest Prefix Match**: 가장 구체적인(긴 마스크) 경로를 선택하는 규칙
{{< /callout >}}
