## 1. 셸(Shell)이란?

> 사용자와 리눅스 커널 사이에서 **명령어를 해석하고 전달하는 인터페이스 프로그램**

### 셸의 동작 과정

```
사용자 ──① "date" 입력──▶ 셸
                          │ ② 문자열 분석
                          │ ③ 명령어 위치 탐색
                          │ ④ 실행 요청
                          ▼
                        커널
                          │ ⑤ 프로세스 생성·실행
                          │ ⑥ 결과 반환
                          ▼
                         셸 ──⑦ 화면 출력──▶ 사용자
```

### 셸의 주요 기능

| 기능 | 설명 | 예시 |
|:-----|:-----|:-----|
| **명령어 해석** | 입력 분석 후 실행 | `ls -la` 파싱 |
| **프로그램 실행** | 커널에 실행 요청 | 명령어 → 프로세스 |
| **입출력 리다이렉션** | 입출력 방향 변경 | `>`, `<`, `>>` |
| **파이프라인** | 출력을 다음 입력으로 | `ls \| grep txt` |
| **환경 변수 관리** | 변수 설정 | `$PATH`, `$HOME` |
| **스크립트 실행** | 명령어 파일 실행 | `.sh` 파일 |
| **작업 제어** | 백그라운드/포그라운드 | `&`, `jobs`, `fg` |

## 2. 셸과 커널의 분리

### 왜 분리되어 있을까?

셸(교체 가능)과 커널(핵심 유지)은 인터페이스로만 연결된다.

```
 bash / zsh / fish ─[인터페이스]─ 리눅스 커널
   (교체 가능)                     (핵심 유지)
```

- 셸만 교체하여 다른 사용 경험 가능
- 셸 오류가 커널에 영향을 주지 않음 (안정성)
- 다른 OS에서도 같은 셸 사용 가능 (이식성)
- 사용자별로 다른 셸 선택 가능

### 셸 이름의 유래

> **Shell(껍데기)**: 커널(핵심)을 감싸고 보호하는 껍질

```
┌─────────────────────┐
│   ┌─────────────┐   │
│   │    커널     │   │ ← Shell
│   │  (Kernel)   │   │   (껍데기)
│   └─────────────┘   │   사용자-커널
└─────────────────────┘   중재자
```

## 3. 프롬프트 (Prompt)

> 셸이 사용자에게 명령어 입력을 기다리고 있음을 나타내는 표시

### 프롬프트 구조

```bash
user@hostname:~/Documents$
│    │        │          │
│    │        │          └ $ : 일반 사용자
│    │        └── 현재 디렉터리 (~ = 홈)
│    └── 호스트명
└── 사용자명
```

### 프롬프트 기호

| 기호 | 의미 | 권한 수준 |
|:----:|:-----|:-----|
| `$` | 일반 사용자 | 제한된 권한 |
| `#` | 루트(슈퍼유저) | 모든 권한 |
| `%` | C셸 계열 일반 사용자 | 제한된 권한 |

### 프롬프트 환경변수 (PS1)

```bash
$ echo $PS1
\u@\h:\w\$
```

| 코드 | 의미 |
|:----:|:-----|
| `\u` | 사용자명 |
| `\h` | 호스트명 (첫 점까지) |
| `\w` | 현재 디렉터리 (전체 경로) |
| `\W` | 현재 디렉터리 (마지막만) |
| `\t` | 시간 (24시간) |
| `\$` | 일반 `$`, 루트 `#` |

```bash
# 시간 추가 예시
$ export PS1="[\t] \u@\h:\w\$ "
[14:30:45] user@host:~$

# 영구 적용
$ echo 'export PS1="[\t] \u@\h:\w\$ "' >> ~/.bashrc
```

## 4. 로그인 셸과 비로그인 셸

### 로그인 셸

> 사용자가 시스템에 로그인할 때 자동으로 시작되는 첫 번째 셸

```bash
# 현재 셸이 로그인 셸인지 확인
$ echo $0
-bash    # 앞에 '-'가 있으면 로그인 셸

# 사용자별 로그인 셸 (마지막 필드)
$ grep username /etc/passwd
username:x:1000:1000::/home/username:/bin/bash
```

### 비로그인 셸

> 로그인 후 추가로 시작한 셸 (터미널 열기, 셸 명령 실행 등)

```bash
$ bash          # 새 bash 시작 (비로그인 셸)
$ exit          # 현재 셸 종료 → 이전 셸로

# logout은 로그인 셸에서만 동작
$ logout
bash: logout: not login shell: use `exit'
```

### 셸 설정 파일 로딩 순서

```
로그인 셸:
  /etc/profile          (시스템 전체)
        ▼
  ~/.bash_profile       (개인, 우선순위 1)
  └ 없으면 ~/.bash_login → ~/.profile

비로그인 셸:
  ~/.bashrc             (개인 설정)

로그아웃:
  ~/.bash_logout        (정리 작업)
```

### 설정 파일 용도

| 파일 | 용도 | 예시 설정 |
|:-----|:-----|:-----|
| `/etc/profile` | 시스템 전체 환경변수 | PATH 기본값 |
| `~/.bash_profile` | 로그인 시 개인 설정 | 환경변수 export |
| `~/.bashrc` | 셸 시작마다 실행 | alias, 프롬프트, 함수 |
| `~/.bash_logout` | 로그아웃 시 실행 | 임시 파일 정리 |

{{< callout type="info" >}}
**`~/.bash_profile` vs `~/.bashrc`** 헷갈리기 쉽다. 환경변수(`export PATH` 등)는 로그인 시 한 번 읽히는 `~/.bash_profile`에, alias·프롬프트·함수처럼 셸을 열 때마다 필요한 설정은 `~/.bashrc`에 둔다. 많은 배포판은 `~/.bash_profile`에서 `~/.bashrc`를 불러오도록 구성해 둔다.
{{< /callout >}}

## 5. 대화형 vs 셸 스크립트

### 대화형(Interactive) 모드

> 명령어를 입력하고 결과를 즉시 확인하는 방식

```bash
$ date
2024년 12월 28일 토요일 ...
$ pwd
/home/user
```

### 셸 스크립트

> 실행할 명령어들을 파일에 미리 작성해 자동 실행하는 방식

```bash
#!/bin/bash
# backup.sh - 간단한 백업 스크립트

BACKUP_DIR="/backup"
DATE=$(date +%Y%m%d)
SOURCE="/home/user/Documents"

echo "백업 시작: $SOURCE → $BACKUP_DIR"
tar -czf "$BACKUP_DIR/backup_$DATE.tar.gz" "$SOURCE"
echo "백업 완료!"
```

```bash
$ chmod +x backup.sh     # 실행 권한 부여
$ ./backup.sh            # 실행
```

**셸 스크립트의 장점**

- **자동화** — 반복 작업 실행, cron 연동 예약
- **재사용** — 복잡한 명령 조합을 저장·공유
- **문서화** — 작업 절차가 코드로 기록됨

> 리눅스 철학: "작은 프로그램을 조합해 복잡한 처리를 수행한다."

## 6. 셸의 종류

### 셸 계보

```
1977  Bourne Shell (sh)
        │
   ┌────┴────┬──────────┐
 C Shell  Korn Shell   Bash
 (csh)     (ksh)      (1989)
   │
 tcsh

이후: zsh(1990), fish(2005)
```

### 주요 셸 비교

| 셸 | 전체 이름 | 특징 | 스크립트 |
|:-----|:-----|:-----|:-----|
| **sh** | Bourne Shell | 최초 Unix 셸, 기본 기능 | 높음 (표준) |
| **bash** | Bourne Again Shell | sh 호환 + 확장, 리눅스 기본 | 높음 |
| **csh** | C Shell | C언어 문법, 대화형 편리 | 낮음 |
| **tcsh** | TENEX C Shell | csh 개선판, 자동완성 | 낮음 |
| **ksh** | Korn Shell | sh + csh 장점 결합 | 높음 |
| **zsh** | Z Shell | bash + tcsh + 독자 기능 | 높음 |
| **fish** | Friendly Interactive Shell | 친화적, 구문 강조 | 중간 |

### 셸 기능 비교

| 기능 | sh | bash | csh | zsh | fish |
|:-----|:--:|:--:|:--:|:--:|:--:|
| Tab 완성 | ✗ | ✓ | ✓ | ◎ | ◎ |
| 히스토리 | ✗ | ✓ | ✓ | ✓ | ✓ |
| 에일리어스 | ✗ | ✓ | ✓ | ✓ | ✓ |
| 작업 제어 | ✗ | ✓ | ✓ | ✓ | ✓ |
| 구문 강조 | ✗ | ✗ | ✗ | △ | ✓ |
| 자동 제안 | ✗ | ✗ | ✗ | △ | ✓ |
| sh 호환 | ✓ | ✓ | ✗ | ✓ | ✗ |

> ◎ 매우 강력, ✓ 지원, △ 플러그인, ✗ 미지원

### bash를 추천하는 이유

- 대부분의 리눅스 배포판 **기본 셸** (별도 설치 불필요)
- sh와 **호환** (기존 sh 스크립트 그대로 실행)
- **대화형 + 스크립트** 양쪽에 적합
- Linux, macOS, WSL, FreeBSD 등 **다양한 OS** 지원
- 풍부한 문서와 커뮤니티

## 7. 셸 전환하기

### 일시적 셸 전환

```bash
$ sh            # sh로 중첩 실행
$ bash          # 다시 bash 중첩

# 셸 스택 (현재 위치는 맨 아래)
# bash (로그인) → sh → bash ← 현재
$ exit          # 한 단계씩 빠져나감
```

### 영구적 셸 변경

```bash
# 사용 가능한 셸 목록
$ cat /etc/shells

# 로그인 셸 변경 (재로그인 필요)
$ chsh -s /bin/zsh

# 다른 사용자 셸 변경 (root)
$ sudo chsh -s /bin/zsh username
```

## 8. 터미널 에뮬레이터

### 터미널이란?

> 원래는 입출력 전용 하드웨어. 현재는 이를 소프트웨어로 구현한 **터미널 에뮬레이터**를 의미

```
┌──────────────────────────┐
│  터미널 에뮬레이터       │
│  user@host:~$ ls -la     │ ← 화면 표시
│  ...                     │   키보드 입력
└────────────┬─────────────┘
             │ 입출력 전달
             ▼
        셸 (bash)
        명령어 해석·실행
```

- **터미널**: 화면 표시, 키보드 입력 전달 (입출력 담당)
- **셸**: 명령어 해석, 커널과 통신 (처리 담당)

### 주요 터미널 에뮬레이터

| OS | 터미널 | 특징 |
|:-----|:-----|:-----|
| **Windows** | Windows Terminal | MS 공식, 탭, WSL 연동 |
| | PuTTY | 가벼움, SSH 클라이언트 |
| **macOS** | Terminal.app | 기본 내장 |
| | iTerm2 | 분할 화면, 강력 |
| **Linux** | GNOME Terminal | GNOME 기본 |
| | Alacritty | GPU 가속, 빠름 |

### 터미널 vs 콘솔 vs 셸

| 용어 | 의미 |
|:-----|:-----|
| **터미널** | 입출력 인터페이스 (에뮬레이터) |
| **콘솔** | 시스템에 직접 연결된 입출력 장치 |
| **셸** | 명령어 해석기 프로그램 |
| **TTY** | TeleTYpewriter, 가상 터미널 장치 |

## 9. 셸 편의 기능

### 히스토리 (History)

```bash
$ history             # 명령어 이력 보기
$ !503               # 503번 명령어 재실행
$ !!                 # 마지막 명령어 재실행
$ !tail              # tail로 시작하는 마지막 명령어
Ctrl + R             # 히스토리 역방향 검색

# 이력 크기 설정 (~/.bashrc)
export HISTSIZE=10000       # 메모리 저장 개수
export HISTFILESIZE=20000   # 파일 저장 개수
```

### 자동 완성 (Tab Completion)

```bash
$ sys<Tab>           # 명령어 완성
$ cd /ho<Tab>        # 경로 완성 → /home/
$ cd /home/<Tab><Tab># 후보 목록 표시
```

### Alias (별칭)

```bash
$ alias ll='ls -la'
$ alias ..='cd ..'
$ alias              # 별칭 목록
$ unalias ll         # 별칭 삭제

# 영구 저장
$ echo "alias ll='ls -la'" >> ~/.bashrc
$ source ~/.bashrc   # 즉시 적용
```

{{< callout type="info" >}}
`source ~/.bashrc`(또는 `. ~/.bashrc`)는 설정 파일을 **현재 셸에 다시 적용**한다. `~/.bashrc`를 수정한 뒤 터미널을 닫았다 열 필요 없이 이 명령으로 바로 반영할 수 있다.
{{< /callout >}}

## 10. 환경 변수

### 환경 변수란?

> 셸과 프로그램이 참조하는 **시스템 설정값을 저장하는 변수**

```bash
$ env                  # 모든 환경 변수
$ echo $HOME           # 홈 디렉터리
$ echo $PATH           # 명령어 검색 경로
```

### 환경 변수 설정

```bash
# 셸 변수 (현재 셸에서만 유효)
$ MY_VAR="hello"

# 환경 변수로 내보내기 (자식 프로세스에도 전달)
$ export MY_VAR="hello"

# 영구 설정
$ echo 'export MY_VAR="hello"' >> ~/.bashrc

# PATH에 디렉터리 추가
$ export PATH="$PATH:/home/user/bin"
```

{{< callout type="warning" >}}
PATH를 수정할 때는 반드시 기존 값 `$PATH`를 포함해야 한다. `export PATH="/home/user/bin"`처럼 덮어쓰면 `ls`, `cd` 등 기본 명령어 경로가 사라져 셸이 정상 동작하지 않는다. 항상 `export PATH="$PATH:추가경로"` 형태로 추가하자.
{{< /callout >}}

### 주요 환경 변수

| 변수 | 설명 | 예시 값 |
|:-----|:-----|:-----|
| `HOME` | 홈 디렉터리 | `/home/user` |
| `USER` | 현재 사용자명 | `user` |
| `SHELL` | 로그인 셸 | `/bin/bash` |
| `PATH` | 명령어 검색 경로 | `/usr/bin:/bin` |
| `PWD` | 현재 작업 디렉터리 | `/home/user` |
| `LANG` | 시스템 언어 | `ko_KR.UTF-8` |
| `EDITOR` | 기본 편집기 | `vim` |
| `PS1` | 프롬프트 형식 | `\u@\h:\w\$` |

## 요약

| 개념 | 핵심 |
|:-----|:-----|
| **셸** | 사용자-커널 인터페이스, 명령어 해석기 |
| **커널 분리 이유** | 유연성, 안정성, 이식성 |
| **프롬프트** | 입력 대기 표시 (`$` 일반, `#` 루트) |
| **로그인 셸** | 로그인 시 시작되는 첫 셸 |
| **셸 스크립트** | 명령어 자동화 파일 |
| **bash** | 리눅스 기본 셸, sh 호환, 범용적 |
| **터미널** | 입출력 담당 (에뮬레이터) |
| **환경 변수** | 시스템/셸 설정값 ($PATH, $HOME) |

### 빠른 참조

```bash
# 셸 정보
echo $SHELL          # 로그인 셸
cat /etc/shells      # 사용 가능한 셸

# 셸 전환
bash / sh / zsh      # 일시 전환
chsh -s /bin/zsh     # 영구 변경

# 설정 파일
~/.bashrc            # bash 설정 (비로그인)
~/.bash_profile      # bash 설정 (로그인)

# 편의 기능
history              # 명령어 히스토리
Ctrl + R             # 히스토리 검색
Tab                  # 자동 완성
alias name='cmd'     # 별칭 설정
```
